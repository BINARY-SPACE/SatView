TC On-Board Memory Images Settings Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

Many aspects of the TC On-Board Memory Images Display are configurable. Layout, dump management aspects, logging options and print jobs can be modified as required.

How do I...

Modify the Layout

  1. Open the TC On-Board Memory Images Settings Dialog Box if it is not open yet:
    Activate the TC On-Board Memory Images display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Layout page.
  3. Specifying the Appearance:

    Creating a new Column

    1. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Name combo box.
    2. Click at the position on the column items bar below the Name combo box where the selected column should be inserted. A '*' indicating the selected column appears.
    3. Push the Insert Before or Insert After button.
      Note:
      The Processor column must always be the left most one and it cannot be deleted.

    Deleting a Column

    1. Select the column to be deleted by clicking on it in the column items bar below the Name combo box. A '*' indicating the selected column appears.
    2. Push the Remove button.
      Note:
      The Processor column cannot be deleted.

    Customizing the Tables

    1. Push the Tables... button. This will open the TC On-Board Memory Images Table Properties Dialog Box.

    2. Changing the Fonts

      1. Check either the Title or Items radio button depending on which font of the tables should be changed.
      2. Choose the desired font face in the Name combo box and then complement the properties by selecting the style, size and color in the corresponding Style, Size and Color combo boxes. A sample of the chosen font is always displayed in the Sample area.

      Changing the Background Colors

      • The background color of all table title bars can be changed in the Title combo box.
      • The same can be applied for any key or value table columns by selecting the desired color in the Keys combo box.
    3. Push the OK button.

    Changing the Fonts

    1. Inside the Fonts group box select the Title or Items radio button depending on which font should be changed for the on-board processor list. Mark the Pages check box in addition to alter the fonts used to display the contents of a selected on-board processor.
    2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.
    Controlling the Operating Flags:

    Tool-tips

    A small tool-tip containing details about an on-board processor or a specific dump can be displayed on request. When this option is enabled and the cursor is moved over an area where additional information is available, a small hand occurs indicating the existence of a tool-tip. Press the right mouse button to make it pop-up. The tool-tip disappears automatically when the mouse button is released.

    • Mark the Tooltips check box in order to enable this feature.
  4. Push the OK button.

Manage Dumps

In order to synchronize the representation of the on-board memory images, occurring dumps have to be archived on-ground. This can be achieved by specifying how that data is processed and maintained.

  1. Open the TC On-Board Memory Images Settings Dialog Box if it is not open yet:
    Activate the TC On-Board Memory Images display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Dumps page.

  3. Updating Database Components

    SatView™ maintains two levels of databases: A Database Management System (DBMS) hosting the database and a binary file for fast access which is a compiled excerpt of the DBMS contents.

    1. Choose the appropriate options:

      Option

      Description

      Update Database Management System (DBMS)

      Every dump is stored in the DBMS according to strategies defined by the specified versioning.
      Note:
      It is recommended to enable this option because it serves as a backup for the binary file.

      Update Binary File

      Downloaded on-board memory data is inserted into the binary database file.
      Note:
      If this option is not enabled and only the DBMS is updated, the DBMS has to be re-compiled to make the new dumps accessible.

      Note:
      If neither of the above options is selected, the most recent dumps will be available only.

    2. Specify the delay n between the time when a dump is received and the update of the selected database components by adjusting the Use an update delay of n seconds spin box.
      Note:
      The delay of n seconds should not be too short in order to enable bulk transfers to the databases which is more efficient than updating each dump individually.

    Customizing the Dump Versioning

    SatView™ implements a sophisticated versioning mechanism in order to simplify the management of on-board memory image dumps.

    1. Inside the Versioning group box select the on-board processor for which the dump management should be modified in the Processor combo box.
    2. Choose the appropriate versioning strategy:

      Strategy

      Description

      None

      Only the most recent dump is stored. Its name is identical to the one specified for the memory image template required to manage the corresponding dump.

      Use a simple increasing number

      Enumerates the dumps in a way that can be further specified:

      1. Enter a prefix x into the With Prefix: x edit field. The prefix x will be precluded to the sample number.
      2. Limit the number of samples n to be maintained in a cyclic way by specifying it in the Limit to: n cyclic dumps spin box.

      Time interval is bigger than n minutes

      Uses a major.minor numbering where the major number is increased when the time interval between the dumps is bigger than n minutes. The version number can be precluded with a prefix x and limited to n samples:

      1. Enter a prefix x into the With Prefix: x edit field. The prefix x will be precluded to the version number.
      2. Limit the number of samples n to be maintained in a cyclic way by specifying it in the Limit to: n cyclic dumps spin box.

      Data changes more than n %

      Uses a major.minor numbering where the major number is increased whenever the data changes (on a byte basis) between the dumps are bigger than n percent. The version number can be precluded with a prefix x and limited to n samples:

      1. Enter a prefix x into the With Prefix: x edit field. The prefix x will be precluded to the version number.
      2. Limit the number of samples n to be maintained in a cyclic way by specifying it in the Limit to: n cyclic dumps spin box.
  4. Push the OK button.

Change the Logging Options

Every change with respect to the operational configuration of the on-board memory images management can be logged and reviewed at a later time. This allows the reconstruction of any configuration changes.

  1. Open the TC On-Board Memory Images Settings Dialog Box if it is not open yet:
    Activate the TC On-Board Memory Images display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Logging page.

  3. Configuring the Options

    1. Push the Options... button. The TC On-Board Memory Images Logging Options Dialog Box will appear.
    2. Type the name for the log file into the Filename edit field. Use the Browse... button in order to locate the file if necessary.
      Note:
      The file name can be identical to those of the other log sources like, for example, command sources. This may be interesting if only one single log file should be generated.
    3. Check one of the available file policy options:

      Option

      Description

      Overwrite logs as needed

      Enables the cyclic overwrite mode.
      Note:
      As soon as the log file reaches the specified size or no more space is available on the storage media new logs overwrite the oldest ones.

      Overwrite logs older than n days

      Enables the cyclic overwrite mode.
      Note:
      Logs are overwritten by new ones as soon as they are older than n days. This can happen even before the specified file size is reached.

      Do not overwrite any logs

      Disables the cyclic overwrite mode.
      Note:
      Stops logging when either the specified size is reached or no more space is available on the storage media.
      Optional:
      • Mark the Disabled check box in order to prevent the log file from being updated.
    4. Check one of the available file size options:

      Option

      Description

      Unlimited

      Archives logs as long as space is available on the storage media.

      Limited to n KBytes

      Limits the file size to n KB.
      Optional:
      • Mark the Reset check box in order to reset the log file.
    5. Push the Retrievals... button which opens the TC On-Board Memory Images Log Retrieval Options Dialog Box and type the name of the log file to be used for retrievals into the Filename edit field. Use the Browse... button in order to locate the file if necessary.
      Push the OK button.
      Note:
      The file name is usually identical to the one specified for logging but can also be identical to those of the other log sources like, for example, command sources. This may be interesting if only one single file is used for the logging of all command sources.
    6. Push the OK button.

    Viewing Real-time Log Entries

    1. Check the Realtime radio button.
    2. Mark the Reload the last n entries check box, enter the number n into the associated edit field and push the Retrieve button in order to immediately retrieve the last n log entries.
      Note:
      By enabling this option the last n log entries will always be listed whenever the dialog box is opened.

    Retrieving Log Entries

    1. Check the Historical radio button and enter the start time for the retrieval into the associated edit field.
    2. Specify the stop condition by checking either:
      • Stop at t and enter the desired stop time t
      • Limit to n entries and enter the number n of entries to retrieve
      • Retrieve up to the end of the log file radio button
    3. Push the Retrieve button.

    Redirecting Log Entries

    • Mark the Show Event check box in order to redirect each log entry as an event to the Global Eventbox.
  4. Push the OK button.

Use Print Job Profiles

  1. Open the TC On-Board Memory Images Settings Dialog Box if it is not open yet:
    Activate the TC On-Board Memory Images display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Printing page.

  3. Creating a new Print Job Profile

    1. Type the name of the print job profile to be created into the Name combo box.
      Note:
      The Default print job profile is used when printing with the quick printing method and can only be modified but not removed.
    2. In the Properties group box specify the print profile to be used with this print job profile by selecting the corresponding profile in the Print Profile combo box. The print profile specifies the printer and paper related properties whereas the print job profile adds some, more TC On-Board Memory Images display specific, printing directives.
    3. Select the print profile to be used inside the Print Profile combo box. The print profile specifies the printer and paper related properties.
    4. Specify the format of the any dump printed by entering the desired number of columns into the Columns spin box inside the Layout group box.
    5. The number of bits per column n can be varied by changing it in the Data size: n bits spin box inside the same group box.
    6. Inside the Options group box choose how much of a dump should be printed:

      Option

      Description

      Print all rows

      Prints the complete dump.

      Print the first/last n rows

      Prints the first or last n rows of a dump.

      Print from l1 to l2 row

      Prints the dump rows from l1 to l2 only.
    7. Type a comment into the Text edit field if something should be added to the title of the printout. Mark the Clear after use check box if this text should be used on one printout only.
    8. Push the Add button.

    Modifying a Print Job Profile

    • Select the name of the print job profile to be modified in the Name combo box and perform the necessary changes (see 'Creating a new Print Job Profile').

    Deleting a Print Job Profile

    1. Select the name of the print job profile to be deleted in the Name combo box.
    2. Push the Remove button to remove the selected print job profile. Use the Remove All button to delete all existing profiles.
      Note:
      The Default print job profile cannot be deleted.
  4. Push the OK button.

Schedule Print Jobs

  1. Open the TC On-Board Memory Images Settings Dialog Box if it is not open yet:
    Activate the TC On-Board Memory Images display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Jobs page.
  3. Select the print job to be scheduled in the Defined list box.
  4. Choose the on-board processor to be attached to the selected print job in the Processor combo box which is located inside the Behavior group box.
  5. Hilight those memory image templates inside the Print list box that should be part of the print-out.
    Note:
    Multiple selections are allowed.
  6. Specify when the print job should be executed by marking one or more of the available options:

    Option

    Description

    Print at name event

    Prints the dump attached to this print job when the event name is in a signaled state.
    Select the event in the associated combo box.
    Note:
    Mark the Complete dumps only check box if a dump should only be printed when it is complete.

    Each time when downloaded

    Prints the dump whenever it is received.
    Note:
    Mark the Complete dumps only check box if a dump should only be printed when it is complete.
  7. Specify a stop condition, if required, by choosing either of these options:

    Option

    Description

    At time t

    Stops the specified print behavior after time t.

    After n times

    Inhibits any further print-outs after n times.
  8. Push the > button to activate the print job. The name of the print job is copied from the Defined list box into the Active one in order to indicate the activation.
    Whenever a print job needs to be deactivated but not deleted select it in the Active list box and then push the < button.
    Note:
    Deactivated print jobs are not scheduled.
  9. Push the OK button.

Prerequisites:
The privilege 'Changing the telecommand on-board software services' must be enabled.
