TC History Display

SUMMARY

Saturday, October 10, 2015
Version 1.0

The TC History Display provides the ability to view the global commanding history archives and has advanced retrieval and filtering options: The Global TC History Display lists all released or failed telecommands in real-time mode whereas those with a local scope allow manual retrievals.

How do I...

Open a TC History Display

  1. Push the Historical Telecommands toggle-button inside the History panel of the Telecommand ribbon bar category.
    This opens the TC History Open Dialog Box.
  2. Choose between the available display types by checking the appropriate radio button in the dialog box:

    Type

    Description

    Normal

    Opens a TC History display.
    Enter a unique name into the Display edit field.
  3. Choose the data mode by either checking the Realtime or History radio button.
    Note:
    Requesting real-time data opens the Global TC History Display whereas in the other case, when asking for historical telecommand data, follow the instructions about how to retrieve with a TC History display.
  4. Optionally configure the TC History display.
  5. Push the OK button.

Note:
The TC History Display shows released telecommands only.

Configure the Settings

Many aspects of the TC History display are configurable. Layout, filtering & logging options and the print profile can be modified as required.

  1. Open the TC History Settings Dialog Box if it is not open yet:
    Activate the TC History display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.

  2. Read more about:

    Modifying the Layout

    Specifying the Appearance:

    1. Select the Layout page.

    2. Creating a new Column

      1. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Name combo box.
      2. Click at the position on the column items bar below the Name combo box where the selected column should be inserted. A '*' indicating the selected column appears.
      3. Push the Insert Before or Insert After button.
        Note:
        The No. column must always be the left most one and it cannot be deleted.

      Deleting a Column

      1. Select the column to be deleted by clicking on it in the column items bar below the Name combo box. A '*' indicating the selected column appears.
      2. Push the Remove button.
        Note:
        The No. column cannot be deleted.

      Specifying the Background Grid & Color

      • Mark the Grid check box in order to enable a background grid and select the desired color inside the associated color combo box.

      Changing the Fonts

      1. Inside the Fonts group box select the Title or Items radio button depending on which font should be changed for the area where the historical telecommands are displayed.
      2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.

    Controlling the Operating Flags:

    1. Select the Layout page.

    2. Tool-tips

      A small tool-tip containing telecommand related details can be displayed on request. When this option is enabled and the cursor is moved over a telecommand inside the corresponding area, a small hand occurs indicating that a tool-tip is available for that part of the telecommand. Press the right mouse button to make it pop-up. The tool-tip disappears automatically when the mouse button is released.

      • Mark the Tooltips check box in order to enable this feature.

    Changing the Filtering Options

    Filters with respect to various criteria can be set in place in order to restrict the telecommands displayed.

    1. Select the Filtering page.
    2. Choose the desired filtering criteria:

      Filter Option

      Description

      Telecommand

      Applies a filter with respect to properties of telecommands:

      Property

      Description

      Name

      Filters all telecommands with a specified name i.e. all telecommands with a different name are not displayed.

      Sequence

      Filters all telecommand sequences with a specified name.

      Type

      Filters all telecommands with either an Immediate, Time-Tagged, 1st Step or 2nd Step type.

      Sub-system

      Filters all telecommands related to a certain sub-system:

      1. Select the desired sub-system in the Sub-system combo box.
      2. Push the < button. This adds it to the list of sub-systems.
        Repeat the previous steps to add more or use the > button to remove a selected entry.

      Verification Status

      Filters all telecommands those stage verification check ended in a certain result. The following stage verification checks can be compared with these predefined results:

      Stage

      Result

      Transmission
      Acceptance
      Start
      Progress
      Completion

      Passed
      Failed
      Unknown
      Unverified
      1. Select the stage verification check in the Stage combo box.
      2. Mark the State check box and choose the required check result in the associated combo box.
        Repeat the previous steps if more that one stage check should be included in the filter.

      Note:
      Telecommands not implementing a stage check that is included in the filter are not listed.

      Limited to maximal n entries

      Limits the number of telecommands listed to n entries.

      Note:
      Choosing multiple filtering options results in combining them with a logical 'AND'.

    3. Choose the desired sorting order:

      Sorting Option

      Description

      By Transmission Time

      The telecommands are listed according to their transmission time.

      By Execution Time

      The telecommands are listed according to their on-board execution time. If there is no on-board execution time attached to the telecommand the transmission time is used instead.

      Show the latest entry as first one

      Determines if the latest telecommands are inserted at the top or appended at the bottom.
    4. If required, add an alert behavior concerning the telecommand stage verification checks or for requested acknowledgments by pushing the Alerts... button.
      This opens the TC History Alert Options Dialog Box:

      Specifying the Alert Behavior

      1. Select the type of incident that should be alerted inside the Type combo box:

        Incident

        Description

        Any telecommand verification failure

        Generates an alert whenever a stage verification check failed for a telecommand.

        Telecommand acceptance failure

        An alert is raised when the acceptance stage verification check failed for a telecommand.

        Telecommand start failure

        An alert is raised when the start stage verification check failed for a telecommand.

        Telecommand progress failure

        An alert is raised when the progress stage verification check failed for a telecommand.

        Telecommand completion failure

        An alert is raised when the completion stage verification check failed for a telecommand.

        Acknowledgeable telecommand acceptance success

        An alert is raised when a telecommand requires the acknowledgment for its successful acceptance on-board.

        Acknowledgeable telecommand start success

        An alert is raised when a telecommand requires the acknowledgment for its successful execution start on-board.

        Acknowledgeable telecommand progress success

        An alert is raised when a telecommand requires the acknowledgment for its successful execution progress on-board.

        Acknowledgeable telecommand completion success

        An alert is raised when a telecommand requires the acknowledgment for its successful execution completion on-board.
      2. Inside the Action group box select the desired audition through the Audition combo box that should be performed when the above selected incident occurs. It will turn up as an event in the Global Eventbox with the color specified in the associated Color combo box. Mark the Blinking & audible check box if the event should also blink (incl. audible sound if chosen) and require an acknowledgment.
      3. Add all notifications to be performed (if any) when an incident of the selected type occurs by choosing them first in the Notifications combo box and then by pressing the Add button.
        Use the Remove button to delete the selected notification profile or Remove All to delete all of them at once.
      4. Push the OK button.

    Specifying the Retrieval Source

    1. Select the Retrieving page.
    2. Choose if either the transmission time or the on-board execution time should be used as reference time by checking the Transmission Time or Execution Time check box.
      Note:
      If the on-board execution time is selected as reference time for retrievals and an archived telecommand does not have an on-board execution time (immediate on-board execution) then the transmission time is used instead.
    3. Select the retrieval file to become the source for any retrieval requests by highlighting it in the Data Source combo box.
      Push the Inspect button to check a potential retrieval file. The details about the inspected retrieval file are shown in the Source Information group box.
      Note:
      The set of retrieval files listed in the Data Source combo box can be specified in the Retrievals page of the System Settings Dialog Box.

    Modifying the Print Profile

    1. Select the Printing page.
    2. Select the print profile to be used inside the Print Profile combo box. The print profile specifies the printer and paper related properties.
    3. Inside the Options group box choose those telecommands that should be printed:

      Range

      Description

      No. from n1 to n2

      Prints all telecommands between number n1 and n2.

      Release Time from t1 to t2

      Prints all telecommands between time t1 and t2.

      Execution Time from t1 to t2

      Prints all telecommands with an on-board execution time between time t1 and t2.
      Note:
      If this option is selected and an archived telecommand does not have an on-board execution time (immediate on-board execution) then the transmission time is used instead.

      Overall Verification:
      Success
      Failure

      Prints all telecommands with a verification stage check result of either 'Success' or 'Failure'.
      Note:
      If a telecommand has no execution verification check attached, a 'Success' result is assumed.

      Note:
      If several filter options are selected they are combined through a logical 'AND'.

    4. Type a comment into the Text edit field if something should be added to the title of the printout. Mark the Clear after use check box if this text should be used on one printout only.
  3. Push the OK button.

Perform Retrievals

SatView™ provides a convenient way to retrieve telecommand related data. Navigating through historical telecommands or command source log files, for example, is a simple process thanks to the various playback modes and high capacity random access storage devices.

  1. Open the TC History Retrievals Dialog Box if it is not open yet:
    Select the Retrievals... item on the application button's menu when the display is activated. Alternatively, push the Retrievals button on the display's toolbar.
  2. In the Retrieve from Time edit field inside the Historical Data group box specify the time at which the retrieval should start.
    The syntax for the specification of the time is: YYYY.DDD.hh:mm:ss.
  3. Specify when the retrieval should be stopped by checking one of the following options:

    Option

    Description

    Stop at YYYY.DDD.hh:mm:ss

    Stops the retrieval at the specified time. Type that time into the associated time edit field.

    Stop when no more data is available

    Stops retrieving telecommands when no more data is available in the retrieval file.
  4. Determine the playback mode by checking one of the following possibilities:

    Option

    Description

    Automatic

    Replays the telecommands with a specified speed.
    Adjust the speed with the Speed slider.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through historical telecommands.

    Pseudo Realtime

    The telecommands are replayed at a speed synchronized with their past release times.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through historical telecommands.

    Manual

    The telecommand closest to the specified starting time is monitored and then the retrieval is stopped.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through historical telecommands.
    Note:

    When retrieving historical telecommands all state changes and related check results are displayed together with the time when they happened or were evaluated.

  5. Push the OK button.
Note:

Use the Global TC History Display to view the most recent telecommands released.

Explore Features

  • Read more about:

    Command History List

    There are several ways how to customize the amount of information displayed within the command history list.

    • Specifying the Display Options

      Changing the columns of the command history list:

      1. Open the TC History Settings Dialog Box if it is not open yet:
        Activate the TC History display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
      2. Select the Layout page.
      3. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Name combo box.
      4. Click at the position on the column items bar below the Name combo box where the selected column should be inserted. A '*' indicating the selected column appears.
      5. Push the Insert Before, Insert After button in order to add a column or Remove to delete a selected column.
        Note:
        The No. column must always be the left most one and it cannot be deleted.
      6. Push the OK button.

      Another way of controlling the amount of arguments displayed is the so called 'full view mode':

      • Select the Full View Mode item on the application button's menu when the display is activated or, alternatively, push the Full View Mode button on the display's toolbar.

      By enabling this option a telecommand is split into as many lines on the stack as are required to display all arguments involved.

    Telecommands & Telecommand Sequences

    Several functions are available to check the execution verification state of telecommands, to search, acknowledge and to browse released telecommands.

    • Browsing the Properties

      1. Open the TC History Telecommand Properties Dialog Box or TC History Telecommand Sequence Properties Dialog Box if it is not open yet:
        Activate the TC History display, select the telecommand ot telecommand sequence those properties need to be viewed and select the Details... item on the application button's menu. Alternatively, push the Details button on the display's toolbar.
        When a telecommand sequence is browsed, select the required member inside the Entry combo box.
      2. Check for the requested telecommand properties.
        Notes:
        • Double-click onto one of the parameters in the list box inside the Parameters group box to view its associated calibration table (if any).
        • Examine the data inside the Raw Data group box when looking for the encoded telecommand data.
      3. Push the OK button.

      Copying to the Clipboard

      This procedure is a convenient way to copy released telecommands back to a command source. The telecommands are automatically reset to an on-stack status when pasted.

      • How to Copy:
        Choose the telecommand(s) to be copied by clicking onto it (them) and then select:
        • The Edit è Copy item on the application button's menu.
          Copies the telecommand(s) to the clipboard.
          Note:
          The same action can also be performed by the floating menu that pops-up when right-clicking on the selected (non-dispatched) telecommand(s).

      Acknowledging

      There are many reasons that might require a telecommand to be acknowledged: A failure during release, confirmation of a stage verification check result and others.
      All telecommands requiring a certain attention are marked with a small icon inside the No. column of the command history list. A description of the reason can be viewed through the tool-tip attached to that icon.

      • Choose the telecommand to be acknowledged by clicking onto it and selecting the Acknowledge item on the application button's menu when the display is activated. Alternatively, push the Acknowledge button on the display's toolbar.

      Note:
      Sometimes, it might be necessary to acknowledge a telecommand multiple times. This is because the command history list maintains a 'Last-In-First-Out' (LIFO) stack for each telecommand, able to keep an unlimited amount of messages for viewing.

      Searching

      The command history list provides help when searching for a telecommand. Thereby, every column can serve as a search key.

      1. Open the TC History Search Telecommand Dialog Box if it is not open yet:
        Click onto the column header representing the requested search key.
      2. Enter the searched value into the Find edit field or select it inside the combo box (for predefined or recent values).
        Note:
        Some search keys require different ways to specify a searched value.
      3. Determine the direction the search should take place by checking either the Up or Down radio button.
        Note:
        The search always starts with the first or last telecommand available depending on the search direction if the Search all check box is marked.
      4. Choose one or more of the available search options:

        Option

        Description

        Search all

        Searches all telecommands listed. Starts the search either from the first or last parameter (depending on the search direction) but not from the current one.

        Match case

        Searches for a word or phrase with matching case.

        Match whole word

        Searches for the specified word which may not be part of another word.
      5. Push the Find Next button to start or continue the search.
        The current telecommand complying with the searched value is always marked with a '*' within the 'No.' column.
      6. Push the Close button to terminate the search.

Print Data

  1. Activate the TC History display to be printed.
  2. Choose one of the following printing methods:

    Method

    Description

    How to do it

    Quick Printing

    This method is the most simple one because it does not need any further interaction of the user in order to print the TC History display. The way how the display is printed is defined by the Default print job profile.

    Push the Print button on the display's toolbar.

    Interactive Printing

    Whenever the TC History display needs to be printed in a different way than defined by the Default print job profile this method is the one to be used. It allows the modification of the print properties without changing the profile.
    • Select the Print... item on the application button's menu when the display is activated.
    • Change the settings in the Print Dialog Box as required.
    • Push the OK button.

Close it

  • Push the Historical Telecommands toggle-button inside the History panel of the Telecommand ribbon bar category or select the Close item on the display's system menu.

Note:
The Global TC History Display will continue to list released telecommands even when it is in a closed (hidden) state.

Prerequisites:
The privilege 'Changing the configuration of telecommand history display windows' must be enabled to configure the TC History display. Having only the 'Using telecommand history display windows' privilege active prevents any changes.
