TC Procedures Monitor

SUMMARY

Saturday, October 10, 2015
Version 1.0

The TC Procedures Monitor controls the execution of telecommand procedures. A telecommand procedure is somewhat similar to a sub-routine that can be called at a certain time or at a specific event with a predefined number of arguments. It can contain one or more threads performing all kind of computations that, in the end, may result in the submission of telecommands to the central command handler. A custom language, the Telecommand Procedure Language (TPL) has been developed to ease the development of such procedures.
The TC Procedures Monitor offers extended scheduling capabilities and also allows the triggering of a telecommand procedure through a telemetry event. In addition, it supports several execution modes (including manual mode) and has extended alert capabilities.

How do I...

Open the TC Procedures Monitor

  1. Push the TC Procedures Monitor toggle-button inside the Command Sources panel of the Telecommand ribbon bar category or, alternatively, open the Windows button's menu inside the Window panel of the System ribbon bar category and select the TC Procedures Monitor item.
  2. Optionally configure the TC Procedures Monitor.

Note:
The TC Procedures Monitor operates in real-time mode only.

Configure the Settings

Many aspects of the TC Procedures Monitor are configurable. Layout, contents, logging options and the print profile can be modified as required. Furthermore, it is also possible to modify the default arguments of the various telecommand procedures, their execution triggers, default release windows and how messages originating from the telecommand procedures should be handled.

  1. Open the TC Procedures Monitor Settings Dialog Box if it is not open yet:
    Activate the TC Procedures Monitor and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.

  2. Read more about:

    Modifying the Layout

    Telecommand Procedures List:

    1. Select the Layout page.

    2. Creating a new Column

      1. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Name combo box.
      2. Click at the position on the column items bar below the Name combo box where the selected column should be inserted. A '*' indicating the selected column appears.
      3. Push the Insert Before or Insert After button.
        Note:
        The Name column must always be the left most one and it cannot be deleted.

      Deleting a Column

      1. Select the column to be deleted by clicking on it in the column items bar below the Name combo box. A '*' indicating the selected column appears.
      2. Push the Remove button.
        Note:
        The Name column cannot be deleted.

      Changing the Fonts

      1. Inside the Fonts group box select the Title or Items radio button depending on which font should be changed for the telecommand procedures list.
      2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.

    Flowchart Pane:

    1. Select the Layout page.

    2. Changing the Background Color

      • Select the color to be used as background for the flowchart pane inside the Flowchart combo box.

      1. Inside the Fonts group box check the Labels radio button in order to change the font for the labels of the flowchart pane.
        Note:
        The color of the flowchart labels can be changed by selecting it in the color combo box right to the Labels radio button.
      2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.

    Changing the Contents

    Command Source:

    1. Select the Contents page.

    2. Specifying the Operational Checks & Modes

      1. Mark the Command Source check box on top of the Operations group box.
      2. Choose the appropriate telecommand release constraints:

        Checks

        Description

        Override Pre-Execution Verification (PEV)

        Forces any pre-execution verification checks to result in a positive ('Go') result.

        Override Execution Verification (CEV)

        Forces any execution verification checks to result in a successful ('Passed') result.

        Disable PEV

        Does not perform any pre-execution verification checks (implicit go).

        Disable CEV

        Does not perform any execution verification checks.

        Note:
        The above checks apply to the command source only. The telecommands dispatched may have different checks enabled.

      3. Choose the appropriate telecommand release modes:

        Flags

        Description

        Release Wait Mode

        Waits with the release of a telecommand until all enabled pre-execution validation and pre-execution verification checks are in a 'Go' state. If a limited release time window is specified for the telecommand procedure and it expires for a certain telecommand, the release is considered to have failed.

        Disabled

        Disables the release of telecommands.

    2. Adding a Procedure

      Any telecommand procedure that should be active and execute from time to time or even continuously needs to be added to the contents list.

      1. Select the telecommand procedure to be added in the List combo box and push the Add button.
        Note:
        A telecommand procedure may be listed more than once. This makes sense when the procedure should start executing on multiple trigger conditions.
      2. Make sure that the Command Source check box is not selected and configure the telecommand procedure by:

        Specifying the Argument Values

        1. Push the Arguments... button. The TC Procedures Monitor Arguments Values Dialog Box will appear.

This can be achieved by selecting the argument in the list view first, then entering a value and finally by pushing the Set button. A limit check is performed automatically on that value if there is an out-of-limit table associated with the argument or if the argument represents a time.
          Note:
          Only values that match the indicated coding and radix for the selected argument can be entered. Expressions are not allowed.
          When an argument has an attached calibration curve it can be viewed by double-clicking on it.
        3. Push the OK button.

        Specifying the Execution Triggers

        1. Push the Execution... button. The TC Procedures Monitor Execution Options Dialog Box will appear.
        2. Specify the execution trigger:

          Always after start-up

          Starts the execution of the telecommand procedure always after start-up of the application.

          Initiates the execution of the telecommand procedure every day at the time t.

          • Stopped at t

           

          • Stops execution after i runs
          • Stops at time t
            Note:
            A telecommand procedure running at time t will be completed.

        3. Specify the execution mode to be applied:

          External actions

        4. Push the OK button.

    Changing the Logging Options

    Every change with respect to the operational configuration of this command source can be logged and reviewed at a later time. This allows the reconstruction of any configuration changes.

    1. Select the Logging page.

    2. Configuring the Options

      1. Push the Options... button. The TC Procedures Monitor Logging Options Dialog Box will appear.
      2. Type the name for the log file into the Filename edit field. Use the Browse... button in order to locate the file if necessary.
        Note:
        The file name can be identical to those of the other (remote) command sources. This may be interesting if only one single log file should be generated.
      3. Check one of the available file policy options:

        Option

        Description

        Overwrite logs as needed

        Enables the cyclic overwrite mode.
        Note:
        As soon as the log file reaches the specified size or no more space is available on the storage media new logs overwrite the oldest ones.

        Overwrite logs older than n days

        Enables the cyclic overwrite mode.
        Note:
        Logs are overwritten by new ones as soon as they are older than n days. This can happen even before the specified file size is reached.

        Do not overwrite any logs

        Disables the cyclic overwrite mode.
        Note:
        Stops logging when either the specified size is reached or no more space is available on the storage media.
        Optional:
        • Mark the Disabled check box in order to prevent the log file from being updated.
      4. Check one of the available file size options:

        Option

        Description

        Unlimited

        Archives logs as long as space is available on the storage media.

        Limited to n KBytes

        Limits the file size to n KB.
        Optional:
        • Mark the Reset check box in order to reset the log file.
      5. Push the Retrievals... button which opens the TC Procedures Monitor Log Retrieval Options Dialog Box and type the name of the log file to be used for retrievals into the Filename edit field. Use the Browse... button in order to locate the file if necessary.
        Push the OK button.
        Note:
        The file name is usually identical to the one specified for logging but can also be identical to those of the other (remote) command sources. This may be interesting if only one single file is used for the logging of all command sources.
      6. Push the OK button.

      Viewing Real-time Log Entries

      1. Check the Realtime radio button.
      2. Mark the Reload the last n entries check box, enter the number n into the associated edit field and push the Retrieve button in order to immediately retrieve the last n log entries.
        Note:
        By enabling this option the last n log entries will always be listed whenever the dialog box is opened.

      Retrieving Log Entries

      1. Check the Historical radio button and enter the start time for the retrieval into the associated edit field.
      2. Specify the stop condition by checking either:
        • Stop at t and enter the desired stop time t
        • Limit to n entries and enter the number n of entries to retrieve
        • Retrieve up to the end of the log file radio button
      3. Push the Retrieve button.

      Redirecting Log Entries

      • Mark the Show Event check box in order to redirect each log entry as an event to the Global Eventbox.

  3. Push the OK button.

    Telecommand Procedures

    The execution of telecommand procedures can also be initiated manually and controlled by breakpoints.
    Read how to handle them in more detail:

    1. Select the telecommand procedure to be controlled in the list of the upper summary pane.

    2. Starting the Execution manually

      • Select the Start/Resume item on the application button's menu when the display is activated. Alternatively, push the Start/Resume button on the display's toolbar.
        Note:
        This starts a telecommand procedure manually. The option Allow manual execution must be enabled to do this.

      Suspending the Execution manually

      • Select the Suspend item on the application button's menu when the display is activated. Alternatively, push the Suspend button on the display's toolbar.
        Note:
        This suspends the execution of a telecommand procedure. The option Allow manual breaks must be enabled to do this.

      Resuming the Execution manually

      • Select the Start/Resume item on the application button's menu when the display is activated. Alternatively, push the Start/Resume button on the display's toolbar.
        Note:
        This resumes a telecommand procedure. The option Allow manual execution must be enabled to do this.

      Executing Step by Step

      1. Suspend the execution of the telecommand procedure either manually or through the use of breakpoints inside the flowchart pane.
      2. Use the following stepping possibilities:

        Action

        Menu Item

        Description

        Step into

        Select the Step Into item on the application button's menu when the display is activated. Alternatively, push the Step into button on the display's toolbar.

        Continues the execution until the first telecommand procedure step of the next lower block level is encountered.

        Step over

        Select the Step Over item on the application button's menu when the display is activated. Alternatively, push the Step over button on the display's toolbar.

        Allows the execution to continue until the next step on the same code block level.

        Step out

        Select the Step Out item on the application button's menu when the display is activated. Alternatively, push the Step out button on the display's toolbar.

        Does not stop the execution until the first step of the next higher block level is reached.

        Note:
        The step commands apply only to the telecommand procedure currently selected in the monitoring list and, more precisely, only to the thread visible in the flowchart pane.

      Executing Line by Line

      1. Suspend the execution of the telecommand procedure either manually or through the use of breakpoints inside the source code pane.
      2. Use the following stepping possibilities:

        Action

        Menu Item

        Description

        Step over

        Select the Step Over item on the application button's menu when the display is activated. Alternatively, push the Step over button on the display's toolbar.

        Allows the execution to continue until the next source code line is reached in one of the threads i.e. whenever one of the threads has continued to the next line the execution is suspended.

        Note:
        The step commands apply only to the telecommand procedure currently selected in the monitoring list.

      Aborting the Execution

      • Select the Abort item on the application button's menu when the display is activated. Alternatively, push the Abort button on the display's toolbar.
        Note:
        This aborts the execution of a telecommand procedure. The option Allow manual breaks must be enabled to do this.

      Setting Breakpoints

      Inside the flowchart pane:

      1. Select the thread to host the breakpoint by selecting the corresponding tab inside the flowchart pane.
      2. Activate the telecommand procedure step to be attached with a breakpoint.
        Note:
        Activating a telecommand procedure step is possible only when the procedure is not running (and not suspended).
      3. Click on the left margin of the flowchart pane at the same vertical position as the activated step. A circle symbolizing the breakpoint will occur.
        Note:
        The option Allow manual breaks must be enabled to set breakpoints.

      Inside the source code pane:

      1. Search for the location to set the breakpoint inside the source code pane.
        Note:
        Only source code lines containing executable code can receive a breakpoint.
      2. Click on the left margin of the source code pane at the same height as the line to receive the breakpoint. A circle symbolizing the breakpoint will occur.
        Note:
        The option Allow manual breaks must be enabled to set breakpoints.

      Changing Breakpoint Properties

      1. Search for the breakpoint those properties need to be changed in either the flowchart panes or the source code pane.
      2. Right-click on the breakpoint found on the left margin of the panes. A floating menu will occur.
        Choose one of the available options:

        Option

        Description

        Break always

        Suspends the execution of the telecommand procedure at the location of the breakpoints.

        Break at failed trigger

        Suspends the execution of the telecommand procedure only when the trigger of the associated step fails.
        Note:
        This option is available only for breakpoints inside the flowchart panes.

        Break at failed body

        Suspends the execution of the telecommand procedure only when the body of the associated step fails.
        Note:
        This option is available only for breakpoints inside the flowchart panes.

        Break at failed confirmation

        Suspends the execution of the telecommand procedure only when the confirmation of the associated step fails.
        Note:
        This option is available only for breakpoints inside the flowchart panes.

        Disable

        Disables the breakpoint i.e. the execution is not suspended.

      Removing Breakpoints

      1. Search for the breakpoint to be removed in either the flowchart panes or the source code pane.
      2. Click on the symbol for the breakpoint and it will disappear.

      Acknowledging Alerts

      Telecommand procedures that are in an alerted state blink inside the summary pane and indicate the problem with a small icon on the left side. An attached tool-tip can be activated on that symbol to view more details.

        Either the TC Procedures Monitor Dialog Box or TC Procedures Monitor Acknowledgments Dialog Box will appear.
      2. Push the Acknowledge button.
        Repeat the previous steps if more than one alert needs to be acknowledged.
        Note:
        Each telecommand procedure maintains a stack of alerts which might require multiple acknowledgments.
      3. Push the OK button.

      Note:
      This resets the execution state of a telecommand procedure i.e. the flowchart pane is reset to a non-executed state.

      Browsing Details

      1. Select the Details... item on the application button's menu when the telecommand procedure to be browsed is selected. Alternatively, push the Details button on the display's toolbar.
        This will open the TC Procedures Monitor Dialog Box where some selected details about the telecommand procedures is available.
      2. Push the OK button.

      Note:
      This dialog box can also be used to acknowledge an alerted telecommand procedure.

    Telecommands

    The release of telecommands can be controlled by interlocks and other release constraints.

    • Specifying Release Constraints

      Telecommand procedures may use interlocks to control the release of selected telecommands and also support release time windows.

      1. Select the Settings... item on the application button's menu when the display is activated. Alternatively, push the Settings button on the display's toolbar.
        This will open the TC Procedures Monitor Settings Dialog Box.
      2. Choose the Contents page.
      3. Select the telecommand procedure those telecommand release constraints should be changed in the contents list box.
      4. Make sure that the Command Source check box is not marked and push the Commands... button. This will open the TC Procedures Monitor Release Options Dialog Box.
      5. Choose the step for which the release constraints need to be specified or modified in the Steps list view.
      6. Mark the Interlock check box in order to connect an interlock with the telecommand of the selected step, if required, and choose the desired interlock in the associated combo box.
      7. Specify the desired release time window to be used, if required, by checking either the Unlimited or Limited to n seconds radio button. Enter the number of seconds n into the related edit field if a limited time window is chosen.
      8. Push all necessary OK buttons.

      Note:
      Any specified release constraints will be valid for all instances of a telecommand procedure.

      Authorizing before Release

      Telecommand procedures may issue telecommands which require an authorization before being released. Such a situation is indicated by alerting that telecommand procedure.

      1. Select the telecommand procedure that waits for an authorization to release one or more telecommands inside the summary pane.
        Note:
        A telecommand that was issued by a procedure that was called as a sub-routine shows-up on a separate line in the summary pane.
      2. Select theAuthorizations... item on the application button's menu. Alternatively, push the Authorizations button on the display's toolbar.
        This will open the TC Procedures Monitor Authorizations Dialog Box.
      3. Push the Authorize button in order to agree the dispatch of the telecommand or the Deny button to prevent that.
        This can be repeated as many times as telecommands need to be authorized.
      4. Push the OK button.

    Interlocks

    Interlocks are essential when it comes to coordinate the release of telecommands from a command source. They can allow or prevent the release of telecommands depending on the success/failure of previous telecommands. Global interlocks apply to all command sources whereas local ones affect only those command sources hosting them.

    • Creating, modifying and deleting Interlocks

      1. Select the Settings... item on the application button's menu when the display is activated. Alternatively, push the Settings button on the display's toolbar.
        This will open the TC Procedures Monitor Settings Dialog Box.
      2. Choose the Contents page.
      3. Push the Commands... button. This will open the TC Procedures Monitor Release Options Dialog Box.
      4. Push the Interlocks... button in order to create, modify or delete interlocks with the TC Procedures Monitor Release Interlocks Dialog Box.
      5. Type the name of the new interlock into the Name edit field or select an existing one in the associated combo box.
      6. Select the scope of the interlock inside the Properties group box:

        Scope

        Description

        Global (all command sources)

        Blocks all command sources and prevents the release of telecommands as long as the interlock is in a pending or failed state.

        Local (same command source only)

        Blocks the local command source as long as the interlock is in a pending or failed state.
        Note:
        Enable the No wait state option if the interlock should not block when in a pending state.

        Local on specified subsystem

        Blocks the release of all telecommands belonging to the specified sub-system of the local command source as long as the interlock is in a pending or failed state.
        Note:
        Enable the No wait state option if the interlock should not block when in a pending state.

        Local on specified telecommands

        Blocks the release of all telecommands associated with this interlock of the local command source as long as the interlocking telecommand has not been verified successfully.
      7. Choose the stage the interlock applies to:

        Stage

        Description

        Reception (at ground station)

        The interlock is set to Success if the telecommand was successfully received by the ground station command uplink infrastructure or to 'Failure' otherwise.

        Transmission

        The interlock is set depending on the outcome of the transmission of a telecommand. It is set to 'Success' if a telecommand was successfully uplinked or to 'Failure' otherwise.

        Acceptance

        The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing an 'Acceptance' stage. Otherwise it remains in an idle state.

        Start

        The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Start' stage. Otherwise it remains in an idle state.

        Progress

        The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Progress' stage. Otherwise it remains in an idle state.

        Completion

        The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Completion' stage. Otherwise it remains in an idle state.
      8. Push the Add button in order to add a new interlock or Remove to delete an existing one. Pushing Remove All clears all at once.
        Notes:
        • Global interlocks can only be created, modified or deleted when the privilege 'Performing privileged tasks on the telecommand procedures monitor' is enabled.
        • Only interlocks in an idle, successful or failed state can be modified or deleted.
      9. Push all necessary OK buttons.

      Resetting Interlocks

      1. Select the Reset Interlocks... item on the application button's menu when the display is activated. Alternatively, push the Reset Interlocks button on the display's toolbar.
        This will open the TC Procedures Monitor Reset Interlocks Dialog Box.
      2. Choose the interlock to be reset to idle state by selecting it in the list and by pushing the Reset button. Use the Reset All button to reset all of them at once.
        Note:
        Interlocks in a pending state can not be reset manually.
      3. Push the OK button.

      Note:
      This option is not available if there are no interlocks to be reset.

Print Data

  1. Activate the TC Procedures Monitor to be printed.
  2. Choose one of the following printing methods:

    Method

    Description

    How to do it

    Quick Printing

    This method is the most simple one because it does not need any further interaction of the user in order to print the TC Procedures Monitor. The way how the display is printed is defined by the Default print job profile.

    Push the Print button on the display's toolbar.

    Interactive Printing

    Whenever the TC Procedures Monitor needs to be printed in a different way than defined by the Default print job profile this method is the one to be used. It allows the modification of the print properties without changing the profile.
    • Select the Print... item on the application button's menu when the display is activated.
    • Change the settings in the Print Dialog Box as required.
    • Push the OK button.

Close it

  • Push the TC Procedures Monitor toggle-button inside the Command Sources panel of the Telecommand ribbon bar category or select the Close item on the display's system menu.

Note:
The TC Procedures Monitor will continue to schedule and execute telecommand procedures even when it is in a closed state. In order to completely stop its activities either disable all telecommand procedures or delete all of them.

Prerequisites:
The privilege 'Changing the contents of the telecommand procedures monitor' must be enabled to manipulate the procedure monitor and to release telecommands from this command source. Having only the 'Browsing the contents of the telecommand procedures monitor' privilege active prevents any changes. Some critical actions and configuration changes require the 'Performing privileged tasks on the telecommand procedures monitor' privilege.
