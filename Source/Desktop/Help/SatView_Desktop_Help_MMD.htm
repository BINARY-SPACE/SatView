Mimics Display (MMD)

SUMMARY

Saturday, October 10, 2015
Version 1.0

A Mimics Display (MMD) is a set of dynamic graphic objects that are animated by telemetry data. A custom language, the Mimics Description Language (MDL) has been developed to control the mimics objects. Individual areas of the display can be linked with other MMDs allowing a kind of zoom-in functionality.

How do I...

Open a MMD

  1. Push the MMD button inside the Telemetry Displays panel of the Displays ribbon bar category.
    This opens the MMD Open Dialog Box.
  2. Choose between the available display types by checking the appropriate radio button in the dialog box:

    Type

    Description

    Normal

    Lists all MMDs defined in the database.
  3. Choose the data mode by either checking the Realtime or History radio button.
    Follow the instructions about how to retrieve with a MMD when requesting historical telemetry data.
  4. Optionally configure the MMD.
  5. Push the OK button.

Configure the Settings

Many aspects of a MMD are configurable. Layout, retrieval source and print jobs can be configured and modified as required. Furthermore, it is also possible to view the source code determining the behavior of the various mimics objects.

  1. Open the MMD Settings Dialog Box if it is not open yet:
    Activate the MMD and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.

  2. Read more about:

    Modifying the Layout

    Specifying the Image Properties:

    1. Select the Layout page.

    2. Image Size

      • Inside the Format group box check either the Default or Fixed radio button:

        Option

        Description

        Default

        Shows the complete image of the MMD.

        Fixed

        Clips the image to the specified size.
        Enter the desired horizontal and vertical size (in pixels) into the associated edit fields.

      Image Margins

      • Fill in the desired margin sizes (in pixels) into the Left, Right, Top and Bottom edit fields.
        Note:
        When a default image size is specified only the right and bottom margins can be specified.

      Background Color

      • Choose the desired background color in the Background combo box.

      Background Grid

      1. Choose the kind of background grid to be created by checking either the Points or Lines radio buttons.
      2. Enter the horizontal and vertical interval (in pixels) between the points or lines into the Horizontal Interval or Vertical edit fields.
      3. Choose the color of the points or lines by selecting the desired color in the Color combo box.

    Controlling the Operating Flags:

    1. Select the Layout page.

    2. Invalid Data Flag

      This flag controls the visibility behavior of the telemetry parameter values with respect to their validity.

      • Mark the Invalid Data check box if telemetry parameter values should be displayed even when their associated validity flag is false.

      Bad Data Flag

      Every telemetry parameter has an associated data quality flag referring to its value. By default, a parameter value with a bad quality flag does not drive any mimics objects.

      • Mark the Bad Data check box if the telemetry parameter values should be displayed even if their data quality flag is set to bad.

      Tool-tips

      A small tool-tip containing details about a mimics object can be displayed on request. When this option is enabled and the cursor is moved over a mimics object inside the MMD that has an associated tool-tip text, a small hand occurs indicating that a tool-tip is available. Press the right mouse button to make it pop-up. The tool-tip disappears automatically when the mouse button is released.

      • Mark the Tooltips check box in order to enable this feature.

    Viewing the Source Code:

    1. Select the Layout page.

    2. MDL Code

      • The source code controlling the behavior of the image can be viewed in the Source Code list box.

    Specifying the Retrieval Source

    1. Select the Retrieving page.
    2. Select the retrieval file to become the source for any retrieval requests by highlighting it in the Data Source combo box.
      Push the Inspect button to check a potential retrieval file. The details about the inspected retrieval file are shown in the Source Information group box.
      Note:
      The set of retrieval files listed in the Data Source combo box can be specified in the Retrievals page of the System Settings Dialog Box.

    Using Print Job Profiles

    1. Select the Printing page.

    2. Creating a new Print Job Profile

      1. Type the name of the print job profile to be created into the Name combo box.
        Note:
        The Default print job profile is used when printing with the quick printing method and can only be modified but not removed.
      2. In the Properties group box specify the print profile to be used with this print job profile by selecting the corresponding profile in the Print Profile combo box. The print profile specifies the printer and paper related properties whereas the print job profile adds some, more MMD specific, printing directives.
      3. Specify the alignment of the print-out with respect to the page by selecting the desired print directive in the Alignment combo box.
        Note:
        This option does not affect the paper orientation.
      4. Mark the Underline OOLs check box if parameter values should be underlined automatically when in an out-of-limit condition.
        Note:
        This option has an effect for those mimics objects only that directly show parameter values.
      5. Mark the No Colors check box if the print-out should be in monochrome.
        Note:
        This option might have no effect with some color printers because of a non-standard color setup. However, in such cases printing with a grayscale is possible through the definition of an appropriate print profile.
      6. Determine the scaling of the print-out by checking either the Scale automatically to paper size or Use a fixed scaling factor of n% check box whereby the factor n needs to be filled in the corresponding edit field. Scaling is always done in an isotropic way.
      7. Type a comment into the Text edit field if additional information should be added to the title of the print-out. Mark the Clear after use check box if this text should be used in a single print-out only.
      8. Push the Add button.

      Modifying a Print Job Profile

      • Select the name of the print job profile to be modified in the Name combo box and perform the necessary changes (see 'Creating a new Print Job Profile').

      Deleting a Print Job Profile

      1. Select the name of the print job profile to be deleted in the Name combo box.
      2. Push the Remove button to remove the selected print job profile. Use the Remove All button to delete all existing profiles.
        Note:
        The Default print job profile cannot be deleted.

    Scheduling Print Jobs

    1. Select the Jobs page.
    2. Select the print job to be scheduled in the Defined list box.
    3. Specify when the print job should be executed by marking one or more of the available options:

      Option

      Description

      Print at YYYY.DDD.hh:mm:ss

      Prints the MMD at a specified time.
      Type the time into the associated time edit field.

      Print at hh:mm:ss every day

      Prints the MMD every day at a specified time.
      Type the time into the associated time edit field.

      Print at name event

      Prints the MMD when the event name is in a signaled state.
      Select the event in the associated combo box.
    4. When printing at a certain time is enabled it is possible to specify how many times the printing should be repeated and with what interval. In the For repetition type combo box select the repetition characteristics for either daily or non-daily printing. In the case of non-daily printing specify the interval in minutes after which another printing should occur by checking the Repeat every n minutes and by typing the interval n into the associated edit field.
      For both kinds of printing stop conditions can be specified by selecting either the Stop after n copies or Stop at time YYYY.DDD.mm:hh:ss radio button and by filling in the appropriate data into the edit fields.
    5. Push the > button to activate the print job. The name of the print job is copied from the Defined list box into the Active one in order to indicate the activation.
      Whenever a print job needs to be deactivated but not deleted select it in the Active list box and then push the < button.
      Note:
      Deactivated print jobs are not scheduled.
  3. Push the OK button.

Perform Retrievals

SatView™ provides a convenient way to retrieve telemetry data. Navigating through historical telemetry data is a simple process thanks to the various playback modes and high capacity random access storage devices.

  1. Open the MMD Retrievals Dialog Box by selecting the Retrievals... item on the application button's menu when the display is activated. Alternatively, push the Retrievals button on the display's toolbar.
  2. In the Retrieve from Time edit field inside the Historical Data group box specify the time at which the retrieval should start.
    The syntax for the specification of the time is: YYYY.DDD.hh:mm:ss.
    Note:
    Depending on the telemetry data standard used it is not always possible to start the retrieval process exactly at the specified time. This is true for incremental telemetry data streams where fast retrievals can only start at positions containing a summary of all telemetry parameters. Mark the Exact Time check box in order to force the retrieval to start exactly at the specified time. However, this may take some time before the retrieval can take place.
  3. Specify when the retrieval should be stopped by checking one of the following options:

    Option

    Description

    Stop at YYYY.DDD.hh:mm:ss

    Stops the retrieval at the specified time. Type that time into the associated time edit field.

    Stop when no more data is available

    Stops retrieving telemetry data when no more data is available in the retrieval file.

    Switch to realtime

    Automatically switches to real-time when no more telemetry data is available.
  4. Determine the playback mode by checking one of the following possibilities:

    Option

    Description

    Automatic

    Replays the telemetry data with a specified speed.
    Adjust the speed with the Speed slider.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.

    Pseudo Realtime

    The telemetry data is replayed at a speed close to real-time.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.

    Manual

    The telemetry data closest to the specified starting time is monitored and then the retrieval is stopped.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.
  5. Specify an interval in minutes if a sequential retrieval is not required. This option is useful when navigating through the retrieval file and searching for a certain incident.
    Note:
    Depending on the telemetry data standard used it is not always possible to retrieve with the interval specified i.e. the de-facto interval may be smaller or bigger than the one specified.
  6. Push the OK button.
Note:

Push the Realtime Mode check box to terminate any retrievals and to switch back to real-time.

Find Displays

SatView™ provides a convenient way to search for a MMD that has a title containing a certain word or words. The search criteria can contain boolean operators.

  1. Open the MMD Find Dialog Box if it is not open yet:
    Push the Find... button on the MMD Open Dialog Box.
  2. Select the search key in the Key combo box.
  3. Select a search value expression in the Value combo box or enter one into the same edit field (if no choices are available).
    Note:
    The search values may also contain the boolean operators '+' (AND) and '|' (OR). Whenever one or more of these operators are specified the operands need to be set into double quotes. There is no operator precedence; the search value expression is just evaluated from the left to the right.
    Examples:

    Key

    Search Value

    Action

    Description

    "AOCS" + "DYNAMICS"

    Searches for all MMDs that contain the words 'AOCS' and 'DYNAMICS' in their title.
  4. Push the Search button.
  5. Select the MMD to be opened in the Results list box and push the Open button or double-click on the display to be opened.
    Note:
    Push the Close button in order to discontinue the search for a MMD.

Link Displays

In some cases it might be of interest to retrieve the same historical telemetry data in multiple display windows simultaneously. This can be achieved by linking them together.

  1. Open the MMD Linkages Dialog Box if it is not open yet:
    Activate the MMD to be linked with other display windows and select the Linkages... item on the application button's menu. Alternatively, push the Linkages button on the display's toolbar.
  2. Select the displays to be linked to the current MMD in the Displays list box.
  3. Push the > button to add the selected one to the list of already linked displays.
    Use the < button to remove a single linked display selected in the Links list box or << to remove all of them at once.
    Note:
    The unlinked display windows will be changed to stopped historical mode or will remain in real-time mode.
  4. Push the OK button.
Note:
  • If the current MMD is processing real-time telemetry data, any linked display window will also be opened in that mode. When operating in historical mode any linked one will be opened with telemetry data being processed at the same (historical) time.
  • The retrieval navigation keys are always disabled on a linked display window.
  • More than one display can be linked to a mimics display.
    In order to have several displays monitoring the same historical telemetry data perform the following steps:
    1. Start a retrieval request on the mimics display (see 'Retrieving with a MMD').
    2. Link one or several display windows to it as described above.
    3. Use the retrieval navigation keys on the initial mimics display to stop or continue the monitoring.
  • Initiating a retrieval request on a display window linked to this one will automatically unlink the other one.

Print Data

  1. Activate the MMD to be printed.
  2. Choose one of the following printing methods:

    Method

    Description

    How to do it

    Quick Printing

    This method is the most simple one because it does not need any further interaction of the user in order to print a MMD. The way how the display is printed is defined by the Default print job profile.

    Push the Print button on the display's toolbar.

    Interactive Printing

    Whenever a MMD needs to be printed in a different way than defined by the Default print job profile this method is the one to be used. It allows the modification of the print properties without changing the profile.
    • Select the Print... item on the application button's menu when the display is activated.
    • Change the settings in the Print Dialog Box as required.
    • Push the OK button.

    Scheduled Printing
    Event-driven Printing

    Sometimes it may be advantageous to print a MMD automatically at a certain time or when a predefined telemetry event occurs. This can be achieved by scheduling a print job.
    • Select the Settings... item on the application button's menu when the display is activated or push the Settings button on the display's toolbar.
    • Select the Print page in the MMD Settings Dialog Box.
    • Add a print profile as required.
    • Push the OK button.

Prerequisites:
The privilege 'Using mimics display windows' must be enabled in order to use any MMDs.
