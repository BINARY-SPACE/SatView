Out-of-Limit Display (OOL)

SUMMARY

Saturday, October 10, 2015
Version 1.0

An Out-of-Limit Display (OOL) lists all parameters that are in an out-of-limit condition. Sophisticated filters allow to specify how selected parameters are alerted. Auditions and notifications can be linked to individual parameters as well causing the related action to be executed when it slips into an out-of-limit condition.

How do I...

Open an OOL Display

  1. Push the OOL button inside the Telemetry Displays panel of the Displays ribbon bar category.
    This opens the OOL Open Dialog Box.
  2. Choose between the available display types by checking the appropriate radio button in the dialog box:

    Type

    Description

    Normal

    Opens an OOL display. Enter a unique name into the Display edit field.
    Note:
    Use this type of display in order to retrieve historical out-of-limit parameters.
  3. Choose the data mode by either checking the Realtime or History radio button.
    Note:
    Requesting real-time data opens the Global OOL Display whereas in the other case, when asking for historical telemetry data, follow the instructions about how to retrieve with an OOL display.
  4. Optionally configure the OOL display.
  5. Push the OK button.

Configure the Settings

Many aspects of an OOL display are configurable. Layout, filtering options, retrieval source and print jobs can be configured and modified as required. The sophisticated filtering capabilities provide a convenient way to specify how an individual out-of-limit event should be handled. Tool-tips, providing details about out-of-limit parameters and, in particular, their values (including the active limit table) can be enabled or disabled as well.

  1. Open the OOL Settings Dialog Box if it is not open yet:
    Activate the OOL display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.

  2. Read more about:

    Modifying the Layout

    Specifying the Appearance:

    1. Select the Layout page.

    2. Creating a new Column

      1. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Column combo box.
      2. Click at the position on the column items bar below the Column combo box where the selected column should be inserted. A '*' indicating the selected column appears.
      3. Push the Insert Before or Insert After button.
        Note:
        The ID column must always be the left most one of a group.

      Deleting a Column

      1. Select the column to be deleted by clicking on it in the column items bar below the Name combo box.
        A '*' indicating the selected column appears.
      2. Push the Remove button.
        Note:
        At least one ID column must be defined.

      Changing the Fonts

      1. Inside the Fonts group box select the Title or OOLs radio button depending on which font should be changed. The Title radio button refers to title bar on top of each display and the OOLs to the listed out-of-limit parameters.
      2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.

    Controlling the Operating Flags:

    1. Select the Layout page.

    2. Invalid Data Flag

      This flag controls the visibility behavior of the out-of-limit parameters with respect to their validity. By default, an invalid parameter, currently in an out-of-limit condition, is not displayed.

      • Mark the Invalid Data check box if out-of-limit parameter values should be displayed even when their associated validity flag is false.

      Bad Data Flag

      Every telemetry parameter has an associated data quality flag referring to its value. By default, a parameter value with a bad quality flag is not updated.

      • Mark the Bad Data check box if the out-of-limit parameter values should be displayed even if their data quality flag is set to bad.

      Tool-tips

      A small tool-tip containing parameter related details can be displayed on request. When this option is enabled and the cursor is moved over a parameter inside the OOL display, a small hand occurs indicating that a tool-tip is available. Press the right mouse button to make it pop-up. The tool-tip disappears automatically when the mouse button is released.

      • Mark the Tooltips check box in order to enable this feature.

    Changing the Filtering Options

    1. Select the Filtering page.

    2. Creating a new Filter

      1. Select the parameter to receive a filter in the For Parameter combo box.
        Note:
        Choose All Parameters if the filter should apply to all parameters. This option is mutual exclusive to filters for individual parameters.
      2. Specify if the parameter should be alerted when it is slipping into an out-of-limit condition:

        Option

        Description

        All the time

        An out-of-limit condition is shown all the time.

        Never

        The parameter(s) are not listed even if they are in an out-of-limit condition.

        Between t1 and t2

        The parameter(s) are only listed when they are in an out-of-limit condition between t1 and t2.
      3. Specify the action to be taken when an out-of-limit event occurs:

        Action

        Description

        Alert as Event

        Generates an event in the Global Eventbox whenever the parameter(s) drop into an out-of-limit condition.

        Bring window to top

        Pops-up the OOL display window whenever a new out-of-limit event occurs.

        Autoacknowledge

        Suppresses any visible or audible alert when an out-of-limit event occurs.

        Notify in-limit condition

        Generates an event in the Global Eventbox when the parameter(s) slip back to an in-limit condition.
        Note:
        This option can also be combined with audition and notification profiles.

        Flash

        Flashes the parameter(s) inside the OOL display as long as they are not acknowledged.

        Autoscroll

        Scrolls the parameter(s) into the visible part of the OOL display when they occur.
      4. Select the audition profile inside the Audition combo box to be executed when the parameter(s) are going into (or slip out of) an out-of-limit condition.
      5. Select the notification profile to be attached to the parameter(s) inside the Notification combo box. This profile is executed together with the audition profile (if any).
      6. Push the Add button.

      Modifying a Filter

      • Select the parameter those filter needs to be modified in the For Parameter combo box and perform the necessary changes (see 'Creating a new Filter' for more details).

      Deleting a Filter

      1. Select the parameter those filter needs to be deleted in the For Parameter combo box.
      2. Push the Remove button to delete the filter. Use the Remove All button in order to delete all existing filters.

      Specifying the out-of-limit Colors

      • Choose the out-of-limit status for which the color should be changed in the For the status OOL combo box and select the desired color in the use the color x combo box.
        Note:
        This out-of-limit color association applies to all parameters regardless of any defined filters.

    Specifying the Retrieval Source

    1. Select the Retrieving page.
    2. Select the retrieval file to become the source for any retrieval requests by highlighting it in the Data Source combo box.
      Push the Inspect button to check a potential retrieval file. The details about the inspected retrieval file are shown in the Source Information group box.
      Note:
      The set of retrieval files listed in the Data Source combo box can be specified in the Retrievals page of the System Settings Dialog Box.

    Using Print Job Profiles

    1. Select the Printing page.

    2. Creating a new Print Job Profile

      1. Type the name of the print job profile to be created into the Name combo box.
        Note:
        The Default print job profile is used when printing with the quick printing method and can only be modified but not removed.
      2. In the Properties group box specify the print profile to be used with this print job profile by selecting the corresponding profile in the Print Profile combo box. The print profile specifies the printer and paper related properties whereas the print job profile adds some, more OOL display specific, printing directives.
      3. Specify how many out-of-limit parameters should be printed by checking either the Print all OOLs, Print the first/last n OOLs or Print from t1 to t2 radio button and filling in the corresponding edit fields.
      4. Mark the Print OOL Legend check box if the out-of-limit status color legend should be printed as well.
      5. Type a comment into the Text edit field if additional information should be added to the title of the print-out. Mark the Clear after use check box if this text should be used in a single print-out only.
      6. Push the Add button.

      Modifying a Print Job Profile

      • Select the name of the print job profile to be modified in the Name combo box and perform the necessary changes (see 'Creating a new Print Job Profile').

      Deleting a Print Job Profile

      1. Select the name of the print job profile to be deleted in the Name combo box.
      2. Push the Remove button to remove the selected print job profile. Use the Remove All button to delete all existing profiles.
        Note:
        The Default print job profile cannot be deleted.

    Scheduling Print Jobs

    1. Select the Jobs page.
    2. Select the print job to be scheduled in the Defined list box.
    3. Specify when the print job should be executed by marking one or more of the available options:

      Option

      Description

      Print at YYYY.DDD.hh:mm:ss

      Prints the OOL display at a specified time.
      Type the time into the associated time edit field.

      Print at hh:mm:ss every day

      Prints the OOL display every day at a specified time.
      Type the time into the associated time edit field.

      Print at name event

      Prints the OOL display when the event name is in a signaled state.
      Select the event in the associated combo box.
    4. When printing at a certain time is enabled it is possible to specify how many times the printing should be repeated and with what interval. In the For repetition type combo box select the repetition characteristics for either daily or non-daily printing. In the case of non-daily printing specify the interval in minutes after which another printing should occur by checking the Repeat every n minutes and by typing the interval n into the associated edit field.
      For both kinds of printing stop conditions can be specified by selecting either the Stop after n copies or Stop at time YYYY.DDD.mm:hh:ss radio button and by filling in the appropriate data into the edit fields.
    5. Push the > button to activate the print job. The name of the print job is copied from the Defined list box into the Active one in order to indicate the activation.
      Whenever a print job needs to be deactivated but not deleted select it in the Active list box and then push the < button.
      Note:
      Deactivated print jobs are not scheduled.
  3. Push the OK button.

Perform Retrievals

SatView™ provides a convenient way to retrieve telemetry data. Navigating through historical telemetry data is a simple process thanks to the various playback modes and high capacity random access storage devices.

  1. Open the OOL Retrievals Dialog Box by selecting the Retrievals... item on the application button's menu when the display is activated. Alternatively, push the Retrievals button on the display's toolbar.
  2. In the Retrieve from Time edit field inside the Historical Data group box specify the time at which the retrieval should start.
    The syntax for the specification of the time is: YYYY.DDD.hh:mm:ss.
    Note:
    Depending on the telemetry data standard used it is not always possible to start the retrieval process exactly at the specified time. This is true for incremental telemetry data streams where fast retrievals can only start at positions containing a summary of all telemetry parameters. Mark the Exact Time check box in order to force the retrieval to start exactly at the specified time. However, this may take some time before the retrieval can take place.
  3. Specify when the retrieval should be stopped by checking one of the following options:

    Option

    Description

    Stop at YYYY.DDD.hh:mm:ss

    Stops the retrieval at the specified time. Type that time into the associated time edit field.

    Stop when no more data is available

    Stops retrieving telemetry data when no more data is available in the retrieval file.
  4. Determine the playback mode by checking one of the following possibilities:

    Option

    Description

    Automatic

    Replays the telemetry data with a specified speed.
    Adjust the speed with the Speed slider.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.

    Pseudo Realtime

    The telemetry data is replayed at a speed close to real-time.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.

    Manual

    The telemetry data closest to the specified starting time is monitored and then the retrieval is stopped.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.
  5. Specify an interval in minutes if a sequential retrieval is not required. This option is useful when navigating through the retrieval file and searching for a certain incident.
    Note:
    Depending on the telemetry data standard used it is not always possible to retrieve with the interval specified i.e. the de-facto interval may be smaller or bigger than the one specified.
  6. Push the OK button.
Note:

Use the Global OOL Display to view the current real-time out-of-limit parameters.

Explore Features

  • Read more about:

    Finding an out-of-limit Parameter

    SatView™ can search for out-of-limit parameters by various criteria represented by the columns of an OOL display.

    1. Open the OOL Search Dialog Box if it is not open yet: Activate the OOL display and then click onto the column header representing the requested search key.
    2. Enter the searched value into the Find edit field or select it inside the combo box (for predefined or recent values).
    3. Determine the direction the search should take place by checking either the Up or Down radio button.
      Note:
      The search always starts with the first or last event available depending on the search direction if the Search all check box is marked.
    4. Choose one or more of the available search options:

      Option

      Description

      Search all

      Searches all parameters listed. Starts the search either from the first or last parameter (depending on the search direction) but not from the current one.

      Match case

      Searches for a word or phrase with matching case.

      Match whole word

      Searches for the specified word which may not be part of another word.
    5. Push the Find Next button to start or continue the search.
    6. Push the Close button to terminate the search.

    Exploring an out-of-limit Parameter

    OOL displays provide the capability to display additional details about the telemetry parameters they contain.

    1. Open the OOL Parameters Dialog Box by double-clicking on the parameter inside the OOL display to be viewed or by selecting the Parameters... item on the application button's menu when the display is activated. Alternatively, push the Out-of-Limit Parameters button on the display's toolbar.
    2. Select the parameter those properties should be shown in the Parameter combo box of the dialog box (if necessary).
    3. Choose the appropriate property page in order to view the desired information.
    4. Push the OK button.

    Acknowledging an out-of-limit Parameter

    An OOL display lists all parameters that are in an out-of-limit condition. Depending on the chosen filtering options it might be necessary to acknowledge certain or all of those out-of-limit parameters.

    1. Open the OOL Acknowledgement Dialog Box by selecting the Acknowledge... item on the application button's menu when the display is activated. Alternatively, push the Acknowledge button on the display's toolbar.
    2. The parameters that are going to be acknowledged are listed in the Parameters list box.
      All those that should not be acknowledged can be deleted from this list by selecting them and then pushing the Remove button. The Remove All clears the complete list.
      Note:
      The audible alerts associated to the out-of-limit parameters can be muted separately i.e. all auditions listed in the Auditions list box will be muted as long as they are not removed by the Remove or Remove All button.
    3. Push the OK button.

Link Displays

In some cases it might be of interest to retrieve the same historical telemetry data in multiple display windows simultaneously. This can be achieved by linking them together.

  1. Open the OOL Linkages Dialog Box if it is not open yet:
    Activate the OOL display to be linked with other display windows and select the Linkages... item on the application button's menu. Alternatively, push the Linkages button on the display's toolbar.
  2. Select the displays to be linked to the current OOL display in the Displays list box.
  3. Push the > button to add the selected one to the list of already linked displays.
    Use the < button to remove a single linked display selected in the Links list box or << to remove all of them at once.
    Note:
    The unlinked display windows will be changed to stopped historical mode or will remain in real-time mode.
  4. Push the OK button.
Note:
  • If the current OOL display is processing real-time telemetry data, any linked display window will also be opened in that mode. When operating in historical mode any linked one will be opened with telemetry data being processed at the same (historical) time.
  • The retrieval navigation keys are always disabled on a linked display window.
  • More than one display can be linked to an out-of-limit display.
    In order to have several displays monitoring the same historical telemetry data perform the following steps:
    1. Start a retrieval request on the out-of-limit display (see 'Retrieving with an OOL display').
    2. Link one or several display windows to it as described above.
    3. Use the retrieval navigation keys on the initial out-of-limit display to stop or continue the monitoring.
  • Initiating a retrieval request on a display window linked to this one will automatically unlink the other one.

Print Data

  1. Activate the OOL display to be printed.
  2. Choose one of the following printing methods:

    Method

    Description

    How to do it

    Quick Printing

    This method is the most simple one because it does not need any further interaction of the user in order to print an OOL display. The way how the display is printed is defined by the Default print job profile.

    Push the Print button on the display's toolbar.

    Interactive Printing

    Whenever an OOL display needs to be printed in a different way than defined by the Default print job profile this method is the one to be used. It allows the modification of the print properties without changing the profile.
    • Select the Print... item on the application button's menu when the display is activated.
    • Change the settings in the Print Dialog Box as required.
    • Push the OK button.

    Scheduled Printing
    Event-driven Printing

    Sometimes it may be advantageous to print an OOL display automatically at a certain time or when a predefined telemetry event occurs. This can be achieved by scheduling a print job.
    • Select the Settings... item on the application button's menu when the display is activated.
    • Select the Print page in the OOL Settings Dialog Box.
    • Add a print profile as required.
    • Push the OK button.

Prerequisites:
The privilege 'Using out-of-limit display windows' must be enabled in order to use any OOL displays.
