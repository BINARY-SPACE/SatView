TC Diagnostics Oversampling Dump Display

SUMMARY

Saturday, October 10, 2015
Version 1.0

The TC Diagnostics Oversampling Dump Display provides the capability to monitor oversampled telemetry parameters and assists in generating the telecommands necessary to control this kind of diagnostic mode.

How do I...

Open a TC Diagnostics Oversampling Dump Display

  1. Push the Oversampling Dumps toggle-button inside the Diagnostics Services panel of the Telecommand ribbon bar category.
    This opens the TC Diagnostics Oversampling Dump Open Dialog Box.
  2. Choose between the available display types by checking the appropriate radio button in the dialog box:

    Type

    Description

    Normal

    Opens a TC Diagnostics Oversampling Dump display.
    Enter a unique name into the Display edit field.
  3. Choose the data mode by either checking the Realtime or History radio button.
    Note:
    Requesting real-time data opens the Global TC Diagnostics Oversampling Dump Display whereas in the other case, when asking for historical telemetry data, follow the instructions about how to retrieve with a TC Diagnostics Oversampling Dump display.
  4. Optionally configure the TC Diagnostics Oversampling Dump display.
  5. Push the OK button.

Configure the Settings

Many aspects of the TC Diagnostics Oversampling Dump display are configurable. Layout, the oversampling characteristics, retrieval source and the print profile can be modified as required.

  1. Open the TC Diagnostics Oversampling Dump Settings Dialog Box if it is not open yet:
    Activate the TC Diagnostics Oversampling Dump display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.

  2. Read more about:

    Modifying the Layout

    Specifying the Appearance:

    1. Select the Layout page.

    2. Showing the raw Telemetry Data Pane

      • Inside the Format group box indicate if a pane with the associated raw telemetry data should be made visible by marking the Raw Data Pane check box.

      Choosing the Columns

      • Choose the desired columns to be displayed by marking the corresponding check boxes:

        Column

        Description

        Numbers

        Enumerates the number of samples processed.

        Time

        Shows time at which the corresponding telemetry data was received.

        Values

        Displays the values for the oversampled telemetry parameters.

        Units

        Appends the units (if any) to the values.

      Note:
      At least one of the columns must be selected.

      Choosing the Background Color for a Column

      In order to differentiate between the numbering & timing information and the telemetry parameter values the background of the corresponding columns can be colored differently.

      • Mark the Background check box below the columns to be affected and select the desired color.

      Changing the Fonts

      1. Inside the Fonts group box select the Title, Items or Values radio button depending on which font should be changed for the pane where the oversampled telemetry parameter values are displayed.
        Repeat this step for the Raw Data pane where the fonts for the title or raw data can be changed by checking either the Title or Data radio button.
      2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.

    Controlling the Operating Flags:

    1. Select the Layout page.

    2. Invalid Data Flag

      This flag controls the visibility behavior of the telemetry parameter values with respect to their validity.

      • Mark the Invalid Data check box if telemetry parameter values should be displayed even when their associated validity flag is false.

      Bad Data Flag

      Every telemetry parameter has an associated data quality flag referring to its value. By default, a parameter value with a bad quality flag is not updated.

      • Mark the Bad Data check box if the telemetry parameter values should be displayed even if their data quality flag is set to bad.

      Tool-tips

      A small tool-tip containing telemetry related details can be displayed on request. When this option is enabled and the cursor is moved over an entry the corresponding area, a small hand occurs indicating that a tool-tip is available for that part of the telemetry. Press the right mouse button to make it pop-up. The tool-tip disappears automatically when the mouse button is released.

      • Mark the Tooltips check box in order to enable this feature.

    Changing the Oversampling Characteristics

    The telemetry parameters are going to be oversampled can be defined manually or automatically.

    1. Select the Contents page.
    2. Automatic Detection of the oversampled Telemetry Parameters:
      • Make sure that the Disable automatic detection check box is not selected.
      • The List list box must not contain any telemetry parameters.

      Note:
      An automatic detection is only possible when a corresponding telecommand has been issued by this display window.

      Manual Specification of the oversampling Characteristics:
      1. Select the telemetry parameter to be part of the oversampling mask in the Parameter combo box.
      2. Push the Add button.
        Repeat the previous steps to add more telemetry parameters or use the Remove button to delete one from the list. Remove All empties the collection at once.
        Note:
        New telemetry parameters are always inserted after the current selection.
      3. Choose a telemetry parameter to be used as a validity flag, overriding those of all members of the oversampling mask by marking the Validity check box and selecting the required validity telemetry parameters.
      4. If desired, limit the number of samples displayed by marking the Limit check box and entering the desired count into the associated edit box.
      5. Select the expected oversampling rate by choosing the appropriate number n inside the Oversampling rate: n seconds combo box.
        Note:
        The bandwidth required by the specified oversampling mask is displayed at Requested bandwidth: n bps and the bandwidth currently available (can only be computed when receiving telemetry data) is indicated at Available bandwidth: n bps.

      Once the desired oversampling mask is specified, the related telecommand(s) to generate this diagnostic mode can be prepared by pushing the Dumps... button. This opens the TC Diagnostics Oversampling Dump Properties Dialog Box allowing to customize the telecommand:

      Customizing Telecommands to initiate the Diagnostic Oversampling Mode

      1. Specify the on-board execution mode by checking either the Immediately or At Time radio button. In the latter case the future on-board execution time has to be entered into the associated time spin-box.
      2. Choose the command source from which the prepared telecommand should be dispatched from by checking either the Manual Stack or Automatic Stack radio button.
      3. Decide if the telecommand has to be released immediately or at a specific time t by selecting the Immediately or At Time radio button and eventually entering the time t.
      4. Mark the Pre-Authorized check box in order to force an automatic dispatch from the chosen command source.
      5. Push the OK button.

      Note:
      This feature is available for the Global TC Diagnostics Oversampling Dump Display only.

    Specifying the Retrieval Source

    1. Select the Retrieving page.
    2. Select the retrieval file to become the source for any retrieval requests by highlighting it in the Data Source combo box.
      Push the Inspect button to check a potential retrieval file. The details about the inspected retrieval file are shown in the Source Information group box.
      Note:
      The set of retrieval files listed in the Data Source combo box can be specified in the Retrievals page of the System Settings Dialog Box.

    Modifying the Print Profile

    1. Select the Printing page.
    2. Select the print profile to be used inside the Print Profile combo box. The print profile specifies the printer and paper related properties.
    3. Inside the Options group box choose which oversampled telemetry parameters should be printed in the Parameter Values combo box.
    4. Push the > button to add the telemetry parameter to the list of those to be printed.
      Use the < button remove a selected one from the list.
    5. Mark the Raw Data check box if the contents of the raw data pane needs to be printed as well.
    6. Decide if the samples printed should be restricted:

      Range

      Description

      No. from n1 to n2

      Prints all samples between number n1 and n2.

      Time from t1 to t2

      Prints all samples between time t1 and t2.

      Note:
      If several filter options are selected they are combined through a logical 'AND'.

    7. Type a comment into the Text edit field if something should be added to the title of the printout. Mark the Clear after use check box if this text should be used on one printout only.
  3. Push the OK button.

Perform Retrievals

SatView™ provides a convenient way to retrieve telemetry data. Navigating through historical telemetry data is a simple process thanks to the various playback modes and high capacity random access storage devices.

  1. Open the TC Diagnostics Oversampling Dump Retrievals Dialog Box by selecting the Retrievals... item on the application button's menu when the display is activated. Alternatively, push the Retrievals button on the display's toolbar.
  2. In the Retrieve from Time edit field inside the Historical Data group box specify the time at which the retrieval should start.
    The syntax for the specification of the time is: YYYY.DDD.hh:mm:ss.
    Note:
    Depending on the telemetry data standard used it is not always possible to start the retrieval process exactly at the specified time. This is true for incremental telemetry data streams where fast retrievals can only start at positions containing a summary of all telemetry parameters. Mark the Exact Time check box in order to force the retrieval to start exactly at the specified time. However, this may take some time before the retrieval can take place.
  3. Specify when the retrieval should be stopped by checking one of the following options:

    Option

    Description

    Stop at YYYY.DDD.hh:mm:ss

    Stops the retrieval at the specified time. Type that time into the associated time edit field.

    Stop when no more data is available

    Stops retrieving telemetry data when no more data is available in the retrieval file.
  4. Determine the playback mode by checking one of the following possibilities:

    Option

    Description

    Automatic

    Replays the telemetry data with a specified speed.
    Adjust the speed with the Speed slider.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.

    Pseudo Realtime

    The telemetry data is replayed at a speed close to real-time.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.

    Manual

    The telemetry data closest to the specified starting time is monitored and then the retrieval is stopped.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.
  5. Specify an interval in minutes if a sequential retrieval is not required. This option is useful when navigating through the retrieval file and searching for a certain incident.
    Note:
    Depending on the telemetry data standard used it is not always possible to retrieve with the interval specified i.e. the de-facto interval may be smaller or bigger than the one specified.
  6. Push the OK button.
Note:

Use the Global TC Diagnostics Oversampling Dump Display to view the current real-time oversampling activities.

    Customizing Telecommands to initiate the Diagnostic Oversampling Mode

    The Global TC Diagnostics Oversampling Dump Display can help to generate and customize the telecommands necessary to initiate an oversampling of the specified telemetry parameters.

    1. Open the TC Diagnostics Oversampling Dump Properties Dialog Box if it is not open yet:
      Select the Settings... item on the application button's menu when the display is activated or, alternatively, push the Settings button on the display's toolbar, choose the Contents page in the dialog box, specify the oversampling dump properties and push the Dumps... button.
    2. Specify the on-board execution mode by checking either the Immediately or At Time radio button. In the latter case the future on-board execution time has to be entered into the associated time spin-box.
    3. Choose the command source from which the prepared telecommand should be dispatched from by checking either the Manual Stack or Automatic Stack radio button.
    4. Decide if the telecommand has to be released immediately or at a specific time t by selecting the Immediately or At Time radio button and eventually entering the time t.
    5. Mark the Pre-Authorized check box in order to force an automatic dispatch from the chosen command source.
    6. Push the OK button.

    Exploring an oversampled Parameter

    TC Diagnostics Oversampling Dump displays provide the capability to display additional details about the telemetry parameters they contain.

    1. Open the TC Diagnostics Oversampling Parameters Dialog Box by double-clicking on the parameter inside the TC Diagnostics Oversampling Dump display to be viewed or by selecting the Parameters... item on the application button's menu when the display is activated. Alternatively, push the Oversampled Parameters button on the display's toolbar.
    2. Select the parameter those properties should be shown in the Parameter combo box of the dialog box (if necessary).
    3. Choose the appropriate property page in order to view the desired information.
    4. Push the OK button.

    Resetting all oversampled Parameter Values

    • Activate the TC Diagnostics Oversampling Dump display and select the Clear All item on the application button's menu. Alternatively, push the Clear All button on the display's toolbar.
      Note:
      This action resets the contents of a TC Diagnostics Oversampling Dump display.

Print Data

  1. Activate the TC Diagnostics Oversampling Dump display to be printed.
  2. Choose one of the following printing methods:

    Method

    Description

    How to do it

    Quick Printing

    This method is the most simple one because it does not need any further interaction of the user in order to print the TC Diagnostics Oversampling Dump display. The way how the display is printed is defined by the Default print job profile.

    Push the Print button on the display's toolbar.

    Interactive Printing

    Whenever the TC Diagnostics Oversampling Dump display needs to be printed in a different way than defined by the Default print job profile this method is the one to be used. It allows the modification of the print properties without changing the profile.
    • Select the Print... item on the application button's menu when the display is activated.
    • Change the settings in the Print Dialog Box as required.
    • Push the OK button.

Close it

  • Push the Oversampling Dumps toggle-button inside the Diagnostics Services panel of the Telecommand ribbon bar category or select the Close item on the display's system menu.

Note:
The Global TC Diagnostics Oversampling Dump Display will continue to list oversampled telemetry parameter values even when it is in a closed (hidden) state.

Prerequisites:
The privilege 'Changing the telecommand diagnostic services' must be enabled to configure the TC Diagnostics Oversampling Dump display. Having only the 'Browsing the telecommand diagnostic services' privilege active prevents any changes.
