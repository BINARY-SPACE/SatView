Eventbox

SUMMARY

Saturday, October 10, 2015
Version 1.0

An Eventbox lists events and messages that can be generated due to many possible reasons.
The Global Eventbox has a global scope and lists all real-time events. Normal Eventboxes are available mainly for retrieving historical events.
All events and messages belong to either the System, Spacecraft or User category, depending from where they originate. Their type gives more information about the severity level and is one out of the following list: Success, Informational, Warning, Error or Scheduled or can have a custom type for further differentiation. The Global Eventbox can also be used as a reminder through its capability to schedule events and to combine them with an audition profile and/or a notification profile. Being able to retrieve events even from remote workstations makes the Eventbox a useful tool to investigate problems.

How do I...

Open an Eventbox

  1. Push the Eventbox button inside the Miscellaneous panel of the Displays ribbon bar category.
    This opens the Eventbox Open Dialog Box.
  2. Select the computer those events and messages should be retrieved by choosing its name in the Computer combo box.
    Note:
    Only computers currently available on the LAN are listed. Connect to one not listed there by typing its name into the Computer edit field and by pushing the Connect button.
  3. Choose the spacecraft of interest through the Spacecraft combo box. Mark the Use global event log check box if a global (unique) spacecraft event log is available and should be used instead of the local one.
  4. Select the user by choosing its name in the User combo box.
  5. Choose the operating mode by either checking the Realtime or History radio button. Follow the instructions about how to retrieve with an Eventbox when requesting historical events.
    Note:
    Local Eventboxes can only operate in real-time mode when the Use global event log option is enabled.
  6. Optionally configure the Eventbox.
  7. Push the OK button.

Configure the Settings

Many aspects of an Eventbox are configurable. Layout, event filtering and print jobs can be modified as required.

  1. Open the Eventbox Settings Dialog Box if it is not open yet:
    Activate the Eventbox and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.

  2. Read more about:

    Modifying the Layout

    Specifying the Appearance:

    1. Select the Layout page.

    2. Changing the Columns

      • Inside the Format group box mark columns that should be displayed:

        Column

        Description

        Time

        Shows the time at which the event occurred or the message was displayed.

        Category

        Indicates the category of the event or message.

        Type

        Indicates the type.

        Message

        Displays the text associated with the event or message.

        Comments

        Displays any attached comment.

        Recipients

        Lists all recipients of an event or message.

        User

        Shows the name of the user that was logged-in when the event occurred or the message was displayed.

      Changing the Fonts

      1. Inside the Fonts group box select the Title Items or List Items radio button depending on which font should be changed. The Title Items radio button refers to title bar on top of each display, the List Items to the items listed.
      2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.

    Using a standard Setup:

    1. Select the Layout page.

    2. Defining a default Layout

      1. Specify the columns to be displayed and the fonts to be used as described above.
      2. Mark the Use as Default check box.

    Changing the Filtering Options

    1. Select the Filtering page.

    2. Filtering by Category

      • Mark the System, Spacecraft and/or User check box if events belonging to that category should be displayed.
      Note:

      This option has an effect on future events only.

      Filtering by Type or Subtype

      1. In the Types list box add all event types that should be listed.
        Note:
        If all filters are removed, the standard types 'Success', 'Informational', 'Warning', 'Error' and 'Scheduled' are filtered.
      2. In the Subtypes list box add all event sub-types that should be listed.
        Note:
        If no sub-types are added, no filtering with respect to the sub-type will take place.
      Note:

      This option has an effect on future events only.

      Filtering by Time

      • Mark the Limited from t1 to t2 check box if only events within a certain time period should be displayed and fill the appropriate time values t1, t2 into the associated edit fields.
      Note:

      This option has an effect on future events only.

      Limiting the Number of Events

      • Mark the Limited to maximal n events check box if maximal n events should be displayed and enter the number n into the associated edit field.
      Note:

      The number of events will be adjusted the next time an event occurs.

      Changing the Sort Order

      • Mark the List the latest event as the first one check box if the latest event should always be listed at the top.

      Changing the Behaviors

      At Startup:

      • Automatically load the last Events

        • Mark the At startup load the last n events check box if the last n events should be loaded automatically at startup and enter the number n into the associated edit field.

      At Event Occurrence:

      • Acknowledge an Event automatically

        • Mark the Autoacknowledge check box if every event should be acknowledged automatically (no blinking and no sound).
        Note:

        This option is available in real-time mode only.

        Pop-up the Eventbox

        • Mark the Bring window to top at an event occurrence check box if the window should pop-up when an event occurs.
        Note:

        This option is available in real-time mode only.

        Send a Notification

        • Mark the Send Notification check box if the Global Eventbox should be notified by a message when an event occurs in another Eventbox.
        Note:

        This option is available for local Eventboxes operating in real-time mode only.

      Changing the Colors

      1. Select the category and type of those events that should be displayed in a different color through the Category and Type combo box.
      2. Choose the desired color in the Color combo box.
      Note:

      This option has an effect on future events only.

      Using Sounds

      • Mark the Use Sound check box if events should produce audible sound when attached to an audition profile.
      Note:

      This option has no effect when the autoacknowledge option (see 'Changing the Behaviors') is enabled and is available for the Global Eventbox only.

    Use Print Job Profiles

    1. Select the Printing page.

    2. Creating a new Print Job Profile

      1. Type the name of the print job profile to be created into the Name combo box.
        Note:
        The Default print job profile is used when printing with the quick printing method and can only be modified but not removed.
      2. In the Properties group box specify the print profile to be used with this print job profile by selecting the corresponding profile in the Print Profile combo box. The print profile specifies the printer and paper related properties whereas the print job profile adds some, more Eventbox specific, printing directives.
      3. Select one of the available printing options by checking the corresponding radio button:

        Option

        Description

        Print all events

        Prints all events listed in the Eventbox.

        Print the first/last n events

        Prints the first or last n events.
        Type 'first' or 'last' and the number n into the corresponding edit fields.

        Print the events between t1 and t2

        Prints all events between t1 and t2 only.
        Type t1 and t2 into the corresponding edit fields.
      4. Type a comment into the Text edit field if additional information should be added to the title of the print-out. Mark the Clear after use check box if this text should be used in a single print-out only.
      5. Push the Add button.

      Modifying a Print Job Profile

      • Select the name of the print job profile to be modified in the Name combo box and perform the necessary changes (see 'Creating a new Print Job Profile').

      Deleting a Print Job Profile

      1. Select the name of the print job profile to be deleted in the Name combo box.
      2. Push the Remove button to remove the selected print job profile. Use the Remove All button to delete all existing profiles.
        Note:
        The Default print job profile cannot be deleted.

    Schedule Print Jobs

    1. Select the Jobs page.
    2. Select the print job to be scheduled in the Defined list box.
    3. Specify when the print job should be executed by marking one or more of the available options:

      Option

      Description

      Print at YYYY.DDD.hh:mm:ss

      Prints the Eventbox at a specified time.
      Type the time into the associated time edit field.

      Print at hh:mm:ss every day

      Prints the Eventbox every day at a specified time.
      Type the time into the associated time edit field.

      Print when n events are listed

      Prints the Eventbox as soon as the number n of events is reached.
      Type the number n into the associated edit field.
    4. When printing at a certain time is enabled it is possible to specify how many times the printing should be repeated and with what interval. In the For repetition type combo box select the repetition characteristics for either daily or non-daily printing. In the case of non-daily printing specify the interval in minutes after which another printing should occur by checking the Repeat every n minutes and by typing the interval n into the associated edit field.
      For both kinds of printing stop conditions can be specified by selecting either the Stop after n copies or Stop at time YYYY.DDD.mm:hh:ss radio button and by filling in the appropriate data into the edit fields.
    5. Push the > button to activate the print job. The name of the print job is copied from the Defined list box into the Active one in order to indicate the activation.
      Whenever a print job needs to be deactivated but not deleted select it in the Active list box and then push the < button.
      Note:
      Deactivated print jobs are not scheduled.
  3. Push the OK button.

Perform Retrievals

SatView provides a convenient way to retrieve events. Navigating through the recorded events is a simple process and very useful to track down any problems with the spacecraft or the workstation.

  1. Open the Eventbox Retrievals Dialog Box:
    Activate the Eventbox and then select the Retrievals... item on the application button's menu. Alternatively, push the Retrievals button on the display's toolbar.
  2. In the Retrieve from Time edit field inside the Historical Data group box specify the time at which the retrieval should start.
    The syntax for the specification of the time is: YYYY.DDD.hh:mm:ss.
  3. Specify when the retrieval should be stopped by checking one of the following options:

    Option

    Description

    Stop at YYYY.DDD.hh:mm:ss

    Stops the retrieval at the specified time. Type that time into the associated time edit field.

    Stop after n events

    Stops retrieving when the specified amount of events n is reached.

    Stop when no more events are available

    Stops retrieving events when no more data is available.
  4. Determine the playback mode by checking one of the following possibilities:

    Option

    Description

    All at once

    Retrieves all the requested events at once.

    Pseudo Realtime

    The events are replayed at a speed similar to the one when they occurred.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical events.

    Manual

    The event closest to the specified starting time is monitored and then the retrieval is stopped.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical events.
  5. Push the OK button.
Note:

Use the Global Eventbox to view the current real-time events.

    Scheduling an Event

    SatView uses events as standard user notification feature. Events can be scheduled to occur at a certain time. Due to the fact that many actions like for example playing sound, voice alerts or notifications by a phone call can be attached to an event almost any kind of action can be initiated at a certain time.

    1. Open the Event Schedule Dialog Box:
      Activate the Global Eventbox and then select the Schedule... item on the application button's menu. Alternatively, push the Schedule button on the display's toolbar.
    2. Type the message text into the Message edit field.
    3. Inside the Characteristics group box select the category to which this event should belong to by selecting it in the Category combo box.
    4. Select or enter the type of the event in the Type combo box.
    5. Attach an audition profile to the event, if required, by selecting one in the Audition combo box.
    6. Choose the color with which the event should be displayed by selecting it in the Color combo box.
    7. Specify the occurrence time of the event in the Initially at Time edit field.
      Note:
      Provide the time in UTC.
    8. Follow these optional steps if a repetitive event should be defined:
      1. Mark the Repeat every n seconds check box if the event should be repeated every n seconds.
      2. Mark the Stop check box if the event should not be repeated endlessly and specify a stop criteria by checking either the At time or After n cycles radio button. Enter time or n into the corresponding edit field.
    9. Select the Autoacknowledge check box if the event should not require an acknowledgment.
      Note:
      When enabling this option the event will not blink when its occurs and will not generate any sound even if an audition profile is attached.
    10. Push the More... button if additional information and directives should be attached to the event. Use the Event Schedule Information Dialog Box to enter that.
    11. Push the OK button.
    Note:

    Scheduling events is available in the Global Eventbox only.

    Prerequisites:
    The privilege 'Modifying the scheduled event list of the global eventbox display window' must be enabled in order to schedule events in the Global Eventbox.

    Finding an Event

    SatView uses events as standard user notification feature. Events can be searched by various criteria represented by the columns of an Eventbox. However, only the current contents of an Eventbox is searched but not the associated event logs.

    1. Open the Event Find Dialog Box:
      Activate the Eventbox and then click onto the column header representing the requested search key.
    2. Enter the searched value into the Find edit field or select it inside the combo box (for predefined or recent values).
    3. Determine the direction the search should take place by checking either the Up or Down radio button.
      Note:
      The search always starts with the first or last event available depending on the search direction if the Search all check box is marked.
    4. Choose one or more of the available search options:

      Option

      Description

      Search all

      Searches all events available. Starts the search either from the first or last event (depending on the search direction) but not from the current one.

      Match time

      Searches for an event with a time matching the specified one.

      Match case

      Searches for a word or phrase with matching case.

      Match whole word

      Searches for the specified word which may not be part of another word.
    5. Push the Find Next button to start or continue the search.
    6. Push the Close button to terminate the search.

    Viewing an Event

    SatView uses events as standard user notification feature. Eventboxes provide the capability to display additional details about the events they contain. At the same occasion it is possible to acknowledge alerted events.

    1. Open the Event Details Dialog Box by double-clicking on the event to be viewed inside the Eventbox or by selecting the Details... after activating the event. Alternatively, push the Details button on the display's toolbar.
    2. Check the Message radio button in order to view the message text associated with the selected event or the Comments radio button to display the comments of the event.
    3. Push the OK button.
    Note:

    An alerted event can also be acknowledged by pushing the Acknowledge button.

    Prerequisites:
    The privilege 'Looking at event details in the global eventbox display window' must be enabled in order to view details about events in the Global Eventbox.

    Acknowledging an Event

    SatView uses events as standard user notification feature. Due to this those events sometimes need to be acknowledged.

    1. Open the Event Acknowledge Dialog Box:
      Activate the Eventbox and then select the Acknowledge... item on the application button's menu. Alternatively, push the Acknowledge button on the display's toolbar.
    2. Push the Update button if not all events that need to be acknowledged are contained in the List list box.
    3. Choose the following filtering options if not all alerted events should be acknowledged:

      Option

      Description

      By Category

      Mark the System, Spacecraft and/or User check box. This will acknowledge only those events belonging to the selected categories.

      By Type

      Check the associated Restricted to Type radio button and select the type of event to be acknowledged.

      By User

      Check the associated Restricted to User radio button and select the user those events should be acknowledged.

      By Time

      Mark the associated From t1 to t2 check box and fill in the time period within which all events should be acknowledged.
    4. Push the Update button in order to verify in the List list box which events are going to be acknowledged.
    5. Push the OK button.
    Note:

    A single event can also be acknowledged with the Eventbox Details Dialog Box.

    Prerequisites:
    The privilege 'Acknowledging events in the global eventbox display window' must be enabled in order to acknowledge events in the Global Eventbox.

    Controlling scheduled Events

    SatView uses events as standard user notification feature. Events can be scheduled to occur at a certain time. Due to the fact that many actions like for example playing sound, voice alerts or notifications by a phone call can be attached to an event almost any kind of action can be initiated at a certain time.

    1. Open the Scheduled Event List Dialog Box:
      Activate the Global Eventbox and then select the Watch... item on the application button's menu. Alternatively, push the Watch button on the display's toolbar.
    2. Select the scheduled event to be viewed, modified or deleted in the List list box.
    3. Perform one of the following actions:
      • Push the Details... button to view a scheduled event. This can be achieved by the Event Details Dialog Box.
      • Push the Change... button to modify a scheduled event. This can be achieved by the Event Schedule Dialog Box.
      • Push the Remove button to delete the selected event from the scheduled list.
        Use the Remove All button to empty the list.

    Prerequisites:
    The privileges 'Watching the scheduled event list of the global eventbox display window' must be enabled to view, modify or delete scheduled events and 'Modifying the scheduled event list of the global eventbox display window' must be active in addition in order to modify or delete scheduled events in the Global Eventbox.

    Copying selected Events

    • Hold-down the right mouse button on the event list of the Eventbox and select one of the available copying options in the appearing floating menu.

    Deleting all Events

    • Activate the Eventbox to be cleared and then select the Clear All item on the application button's menu. Alternatively, push the Clear All button on the display's toolbar.

    Prerequisites:
    The privilege 'Resetting the contents of the global eventbox display window' must be enabled in order to delete events in the Global Eventbox.

Print Data

  1. Activate the Eventbox to be printed.
  2. Choose one of the following printing methods:

    Method

    Description

    How to do it

    Quick Printing

    This method is the most simple one because it does not need any further interaction of the user in order to print an Eventbox. The way how the display is printed is defined by the Default print job profile.

    Push the Print button on the display's toolbar.

    Interactive Printing

    Whenever an Eventbox needs to be printed in a different way than defined by the Default print job profile this method is the one to be used. It allows the modification of the print properties without changing the profile.
    • Select the Print... item on the application button's menu when the display is activated.
    • Change the settings in the Print Dialog Box as required.
    • Push the OK button.

    Scheduled Printing
    Triggered Printing

    Sometimes it may be advantageous to print an Eventbox automatically at a certain time or when certain conditions are met. This can be achieved by scheduling a print job.
    • Push the Settings... item on the application button's menu when the display is activated or push the Settings button on the display's toolbar.
    • Select the Print page in the Eventbox Settings Dialog Box.
    • Add a print profile as required.
    • Push the OK button.

Prerequisites:
The privilege 'Using eventbox display windows' must be enabled in order to use any Eventboxes.
