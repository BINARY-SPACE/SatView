TC Automatic Stack

SUMMARY

Saturday, October 10, 2015
Version 1.0

The TC Automatic Stack is another command source, similar to the previous one but dedicated to the automatic release of telecommands. These telecommands usually are generated by a Mission Planning System (MPS) and imported by schedule or manually. The same processing services as in the TC Manual Stack are supported but the user interface is extended with an additional pane listing the released or failed telecommands.

How do I...

Open the TC Automatic Stack

  1. Push the Automatic TC Stack toggle-button inside the Command Sources panel of the Telecommand ribbon bar category or, alternatively, open the Windows button's menu inside the Window panel of the System ribbon bar category and select the TC Automatic Stack item.
  2. Optionally configure the TC Automatic Stack.

Note:
The TC Automatic Stack is divided into two types of stacks, the Normal Stack handling all telecommands except those having a specific (future) on-board execution time and the Time-Tagged Stack containing all those other telecommands. Both stacks are again split-up into two sub-panes, the upper one, called Scheduled Pane, showing all (dispatched) telecommands not released yet and the lower one, Historical Pane, with those that were released or failed to be released.

Configure the Settings

Many aspects of the TC Automatic Stack are configurable. Layout, operational aspects, filtering & logging options and the print profile can be modified as required. This command source does also have an in-built interface to a Mission Planning System (MPS).

  1. Open the TC Automatic Stack Settings Dialog Box if it is not open yet:
    Activate the TC Automatic Stack and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.

  2. Read more about:

    Specifying the Appearance:

    1. Select the Layout page.
    2. Choose the pane for which the layout should be modified inside the For Pane combo box.

    3. Creating a new Column

      1. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Name combo box.
      2. Click at the position on the column items bar below the Name combo box where the selected column should be inserted. A '*' indicating the selected column appears.
      3. Push the Insert Before or Insert After button.
        Note:
        The No. column must always be the left most one and it cannot be deleted.

      1. Select the column to be deleted by clicking on it in the column items bar below the Name combo box. A '*' indicating the selected column appears.
      2. Push the Remove button.
        Note:

      • Mark the Grid check box in order to enable a background grid and select the desired color inside the associated color combo box.

      1. Inside the Fonts group box select the Title or Items radio button depending on which font should be changed for the telecommand stack.
      2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.

    2. Specifying the Operational Modes & Checks

        Prevents the issue of telecommands from this command source.

        Description

    Telecommands & Telecommand Sequences

    Because this command source is primarily designed to automatically interact with a Mission Planning System (MPS) and Flight Dynamics System (FDS) there is no direct way to load telecommands manually. Nevertheless, in order to retain a certain degree of control over the stack, it is possible to use the copy/paste approach.
    Many other aspects however are identical to those of the TC Manual Stack.

    • Loading manually onto the Stack

      1. Load and configure the telecommand or telecommand sequence by using the TC Manual Stack.
      2. Choose the telecommand(s) to be copied by clicking onto it (them) and then select:
        • The Edit Copy item on the application button's menu.
          Copies the telecommand(s) to the clipboard.
          Note:
          The same action can also be performed through the floating menu that pops-up when right-clicking on the selected (non-dispatched) telecommand(s).
      3. Switch back to the TC Automatic Stack and choose the appropriate stack for the telecommands to be pasted:
        • Normal Stack:
          Use this stack for all those telecommands or telecommand sequences that do have an immediate on-board execution time.
        • Time-Tagged Stack:
          Paste all telecommands with a scheduled on-board execution time into this stack.

        Note:
        Telecommands can only be pasted to the Scheduled Pane of either the Normal Stack or the Time-Tagged Stack.
        If multiple telecommands or a telecommand sequence is transferred that contain(s) telecommands of both types they have to be split-up and copied in two steps.

      4. Paste the telecommands from the clipboard onto the stack by selecting:
        • The Edit Paste item on the application button's menu.
          Automatically inserts the pasted telecommands at the appropriate position on the stack.
          Note:
          The same action can also be performed through the floating menu that pops-up when right-clicking on the (non-dispatched) telecommand(s).
      5. If the pasted telecommands need further configuration push the Properties... button.
        The TC Automatic Stack Telecommand Properties Dialog Box or TC Automatic Stack Telecommand Sequence Properties Dialog Box will pop-up:

              Setting an Interlock

              1. Mark the Interlock check box in order to attach the current telecommand with an interlock i.e. to assign the outcome of a stage verification check to the interlock.
              2. Select the interlock inside the associated combo box or push the Interlocks... button to create a new interlock or to modify an existing one to attach to. The latter case will open the TC Automatic Stack Telecommand Release Interlocks Dialog Box:

                Creating, modifying and deleting Interlocks

                Interlocks are essential when it comes to coordinate the release of telecommands from a command source. They can allow or prevent the release of telecommands depending on the success/failure of previous telecommands. Global interlocks apply to all command sources whereas local ones affect only those command sources hosting them.

                1. Type the name of the new interlock into the Name edit field or select an existing one in the associated combo box.
                2. Select the scope of the interlock inside the Properties group box:

                  Scope

                  Description

                  Global (all command sources)

                  Blocks all command sources and prevents the release of telecommands as long as the interlock is in a pending or failed state.

                  Local (same command source only)

                  Blocks the local command source as long as the interlock is in a pending or failed state.
                  Note:
                  Enable the No wait state option if the interlock should not block when in a pending state.

                  Local on specified subsystem

                  Blocks the release of all telecommands belonging to the specified sub-system of the local command source as long as the interlock is in a pending or failed state.
                  Note:
                  Enable the No wait state option if the interlock should not block when in a pending state.

                  Local on specified telecommands

                  Blocks the release of all telecommands associated with this interlock of the local command source as long as the interlocking telecommand has not been verified successfully.
                3. Choose the stage the interlock applies to:

                  Stage

                  Description

                  Reception (at ground station)

                  The interlock is set to Success if the telecommand was successfully received by the ground station command uplink infrastructure or to 'Failure' otherwise.

                  Transmission

                  The interlock is set depending on the outcome of the transmission of a telecommand. It is set to 'Success' if a telecommand was successfully uplinked or to 'Failure' otherwise.

                  Acceptance

                  The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing an 'Acceptance' stage. Otherwise it remains in an idle state.

                  Start

                  The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Start' stage. Otherwise it remains in an idle state.

                  Progress

                  The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Progress' stage. Otherwise it remains in an idle state.

                  Completion

                  The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Completion' stage. Otherwise it remains in an idle state.
                4. Push the Add button in order to add a new interlock or Remove to delete an existing one. Pushing Remove All clears all at once.
                  Notes:
                  • Global interlocks can only be created, modified or deleted when the privilege 'Performing privileged tasks on the automatic telecommand stack' is enabled.
                  • Only interlocks in an idle, successful or failed state can be modified or deleted.
                5. Push the OK button.

              Depending on an Interlock

              1. Mark the Locked on check box and select the required interlock to depend on in the associated combo box.
                Note:

              2. Choose the expected state (either 'Success' or 'Failure' for the interlock in the right most combo box i.e. if an interlock x and state 'Failure' is selected then the telecommand gets a 'go' as soon as the interlock x changes its state to 'Failure'.
          4. Push the OK button.

    Interlocks

    Interlocks are essential when it comes to coordinate the release of telecommands from a command source. They can allow or prevent the release of telecommands depending on the success/failure of previous telecommands. Global interlocks apply to all command sources whereas local ones affect only those command sources hosting them.

    • Creating, modifying and deleting Interlocks

      Interlocks can be manipulated when telecommands are modified.

      1. Modify the telecommand to interlock or to be interlocked by:
        • Activating the telecommand inside the Scheduled Pane of either the Normal Stack or the Time-Tagged Stack and selecting the Edit Properties... item on the application button's menu. Alternatively, push the Properties button on the display's toolbar.
          Note:
          The same action can also be performed through the floating menu that pops-up when right-clicking on the (non-dispatched) telecommand(s).
        This opens the TC Automatic Stack Telecommand Properties Dialog Box.
      2. Push the Release... button. The TC Automatic Stack Telecommand Release Constraints Dialog Box will appear.
      3. Push the Interlocks... button in order to create, modify or delete interlocks with the TC Automatic Stack Telecommand Release Interlocks Dialog Box.
      4. Type the name of the new interlock into the Name edit field or select an existing one in the associated combo box.
      5. Select the scope of the interlock inside the Properties group box:

        Scope

        Description

        Global (all command sources)

        Blocks all command sources and prevents the release of telecommands as long as the interlock is in a pending or failed state.

        Local (same command source only)

        Blocks the local command source as long as the interlock is in a pending or failed state.
        Note:
        Enable the No wait state option if the interlock should not block when in a pending state.

        Local on specified subsystem

        Blocks the release of all telecommands belonging to the specified sub-system of the local command source as long as the interlock is in a pending or failed state.
        Note:
        Enable the No wait state option if the interlock should not block when in a pending state.

        Local on specified telecommands

        Blocks the release of all telecommands associated with this interlock of the local command source as long as the interlocking telecommand has not been verified successfully.
      6. Choose the stage the interlock applies to:

        Stage

        Description

        Reception (at ground station)

        The interlock is set to Success if the telecommand was successfully received by the ground station command uplink infrastructure or to 'Failure' otherwise.

        Transmission

        The interlock is set depending on the outcome of the transmission of a telecommand. It is set to 'Success' if a telecommand was successfully uplinked or to 'Failure' otherwise.

        Acceptance

        The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing an 'Acceptance' stage. Otherwise it remains in an idle state.

        Start

        The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Start' stage. Otherwise it remains in an idle state.

        Progress

        The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Progress' stage. Otherwise it remains in an idle state.

        Completion

        The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Completion' stage. Otherwise it remains in an idle state.
      7. Push the Add button in order to add a new interlock or Remove to delete an existing one. Pushing Remove All clears all at once.
        Notes:
        • Global interlocks can only be created, modified or deleted when the privilege 'Performing privileged tasks on the automatic telecommand stack' is enabled.
        • Only interlocks in an idle, successful or failed state can be modified or deleted.
      8. Push all necessary OK buttons.

      Prerequisites:
      The command source must be in a 'SUSPENDED' commanding state to perform the above steps.
