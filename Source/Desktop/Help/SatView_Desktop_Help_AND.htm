Alphanumeric Display (AND)

SUMMARY

Saturday, October 10, 2015
Version 1.0

An Alphanumeric Display (AND) presents parameters in an alphanumeric format. Its parameters can be arranged in groups with associated labels and are capable to show values in various number formats. Furthermore, the amount of information displayed for each parameter is configurable by choosing a set of predefined columns to be part of the layout.

How do I...

Open an AND

  1. Push the AND button inside the Telemetry Displays panel of the Displays ribbon bar category.
    This opens the AND Open Dialog Box.
  2. Choose between the available display types by checking the appropriate radio button in the dialog box:

    Type

    Description

    Normal

    Lists all ANDs defined in the database.
    Note:
    The parameter arrangement of this type of AND can be changed interactively. The modifications, however, are temporary only.

    Generic

    Lists all ANDs derived from the available telemetry packets.
    Note:
    This type of AND is not supported when the database is not compatible with the CCSDS Packet Telemetry Standard.

    Temporary

    Lists all ANDs created interactively.
    This type of display can be created by typing a unique name into the Display edit field and by changing the contents of an AND to populate it with telemetry parameters.
    An existing AND of this type can always be deleted from the database by pushing the Remove button.
  3. Choose the data mode by either checking the Realtime or History radio button.
    Follow the instructions about how to retrieve with an AND when requesting historical telemetry data.
  4. Optionally configure the AND.
  5. Push the OK button.

Configure the Settings

Many aspects of an AND are configurable. Layout, parameter arrangement, retrieval source and print jobs can be modified as required. Furthermore, it is also possible to define the way how telemetry data should be displayed with respect to validity and out-of-limit status colors. Tool-tips, providing details about parameters and, in particular, their values can be enabled or disabled as well.

  1. Open the AND Settings Dialog Box if it is not open yet:
    Activate the AND and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.

  2. Read more about:

    Modifying the Layout

    Specifying the Appearance:

    1. Select the Layout page.

    2. Creating a new Column

      1. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Name combo box.
      2. Click at the position on the column items bar below the Name combo box where the selected column should be inserted.
        A '*' indicating the selected column appears.
      3. Push the Insert Before or Insert After button.
        Note:
        The ID column must always be the left most one of a group i.e. all columns (until the next ID column) following this one to the right refer to it.

      Deleting a Column

      1. Select the column to be deleted by clicking on it in the column items bar below the Name combo box.
        A '*' indicating the selected column appears.
      2. Push the Remove button.
        Note:
        At least one ID column must be defined.

      Changing the Fonts

      1. Inside the Fonts group box select the Title, Labels, Parameters or Values radio button depending on which font should be changed. The Title radio button refers to title bar on top of each display, the Labels to any parameter labels, the Parameters to parameters and the Values to their associated values.
      2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.
        Note:
        The Underlined check box is available for labels only.

    Controlling the Operating Flags:

    1. Select the Layout page.

    2. Invalid Data Flag

      This flag controls the visibility behavior of the telemetry parameter values with respect to their validity.

      • Mark the Invalid Data check box if telemetry parameter values should be displayed even when their associated validity flag is false.

      Bad Data Flag

      Every telemetry parameter has an associated data quality flag referring to its value. By default, a parameter value with a bad quality flag is not updated.

      • Mark the Bad Data check box if the telemetry parameter values should be displayed even if their data quality flag is set to bad.

      Tool-tips

      A small tool-tip containing parameter related details can be displayed on request. When this option is enabled and the cursor is moved over a parameter inside the AND, a small hand occurs indicating that a tool-tip is available. Press the right mouse button to make it pop-up. The tool-tip disappears automatically when the mouse button is released.

      • Mark the Tooltips check box in order to enable this feature.

    Changing the Contents

    1. Select the Contents page.

    2. Adding a Parameter or Label

      1. Choose the position where the parameter or label should be inserted by clicking on that location inside the list box at the top of the page. If more than one column group is defined click first on that ID column that should receive the inserted parameter.
        A '*' appears and marks the column group that is affected of any future changes. The current row and column is also indicated below the list box.
      2. Choose the parameter to be inserted by selecting it inside the Name combo box of the Parameters group box or type the name of the label into the corresponding edit field.
      3. Select the color of the parameter or label by choosing one of the colors in the Color combo box.
      4. Determine how the parameter value should be displayed by checking the appropriate radio button:

        Coding & Radix

        Description

        Default

        Shows the parameter value in a calibrated form when possible.

        Decimal

        The coded (non-calibrated) value is displayed with a decimal radix.

        Binary

        The coded (non-calibrated) value is displayed with a binary radix.

        Octal

        The coded (non-calibrated) value is displayed with an octal radix.

        Hexadecimal

        The coded (non-calibrated) value is displayed with a hexadecimal radix.

        Float

        The coded (non-calibrated) value is displayed in a floating point format.
        Note:
        This coding is only available for floating point parameters.

        Inverse

        Displays the one's complement of the parameter value.
        Note:
        This option is only available when displaying a value with a binary, octal or hexadecimal radix.
        Note:
        This option is not applicable for labels.
      5. Determine how the parameter value or label should be aligned by checking the appropriate radio button:

        Alignment

        Description

        Left

        Aligns the parameter value to the left of the column.

        Center

        Centers the parameter value with respect to the column width.

        Right

        Shows the parameter value aligned to the right side of the column.
      6. Specify the occurrence that should be displayed. By default, the first occurrence is always shown. Click on the Past check box if a past sample should be made visible and specify the sample number by typing it into the edit field.
        Note:
        This option is not applicable for labels.
      7. If a non-default number of decimal places (for floating-point numbers) should be used to display a parameter's value then check the radio button to the left of the associated spin box (below the Default one) and enter the desired number.
        Note:
        This option is not applicable for labels.
      8. Add some comments to the parameter when required. This text will be displayed inside the associated tool-tip.
        Note:
        This option is not applicable for labels.
      9. Push the Add button in order to insert the parameter or label at the specified position.

      Note:
      The spin-up/down control (located to the right of the Remove All button) can be used to shift a parameter one or more rows up or down.

      Removing a Parameter or Label

      1. Choose the parameter or label to be deleted by clicking on it inside the list box at the top of the page. If more than one column group is defined click first on that ID column that contains the parameter to be removed. A '*' appears and marks the column group that is affected of any future changes.
        The current row and column is also indicated below the list box.
      2. Push the Remove button in order to delete the selected parameter or label. Use the Remove All button to clear all parameters and labels of the display.

      Specifying the out-of-limit Colors

      • Select the out-of-limit status of which the color association should be changed in the For type of OOL use color combo box and choose the desired color in the other combo box.

      Note:
      The changes to the out-of-limit status color association apply to the complete display and not to a specific parameter.

      Importing an existing AND

      1. Push the Import... button. The AND Directory Dialog Box appears.
      2. Select the AND to be imported and push the OK button.

    Specifying the Retrieval Source

    1. Select the Retrieving page.
    2. Select the retrieval file to become the source for any retrieval requests by highlighting it in the Data Source combo box.
      Push the Inspect button to check a potential retrieval file. The details about the inspected retrieval file are shown in the Source Information group box.
      Note:
      The set of retrieval files listed in the Data Source combo box can be specified in the Retrievals page of the System Settings Dialog Box.

    Using Print Job Profiles

    1. Select the Printing page.

    2. Creating a new Print Job Profile

      1. Type the name of the print job profile to be created into the Name combo box.
        Note:
        The Default print job profile is used when printing with the quick printing method and can only be modified but not removed.
      2. In the Properties group box specify the print profile to be used with this print job profile by selecting the corresponding profile in the Print Profile combo box. The print profile specifies the printer and paper related properties whereas the print job profile adds some, more AND specific, printing directives.
      3. Specify if all parameters or only the first/last n parameters should be printed by checking either the Print all parameters or Print the first/last n rows radio button and by filling-in the number n in the edit field.
      4. Mark the Underline OOLs check box if an out-of-limit value should be underlined.
        Note:
        This option is useful when printing without colors.
      5. Mark the Print OOL Legend check box if a small legend should be included on the print-out indicating the out-of-limit status color association.
        Note:
        This option is useful when printing in colors.
      6. Define the print layout by checking either the Generic Format radio button (uses the same column arrangement as in the display) or the one below it which allows to change the number of columns used when printing.
        Note:
        Changing the print layout is allowed for those displays only that have a single column.
      7. Type a comment into the Text edit field if additional information should be added to the title of the print-out. Mark the Clear after use check box if this text should be used in a single print-out only.
      8. Push the Add button.

      Modifying a Print Job Profile

      • Select the name of the print job profile to be modified in the Name combo box and perform the necessary changes (see 'Creating a new Print Job Profile').

      Deleting a Print Job Profile

      1. Select the name of the print job profile to be deleted in the Name combo box.
      2. Push the Remove button to remove the selected print job profile. Use the Remove All button to delete all existing profiles.
        Note:
        The Default print job profile cannot be deleted.

    Scheduling Print Jobs

    1. Select the Jobs page.
    2. Select the print job to be scheduled in the Defined list box.
    3. Specify when the print job should be executed by marking one or more of the available options:

      Option

      Description

      Print at YYYY.DDD.hh:mm:ss

      Prints the AND at a specified time.
      Type the time into the associated time edit field.

      Print at hh:mm:ss every day

      Prints the AND every day at a specified time.
      Type the time into the associated time edit field.

      Print at name event

      Prints the AND when the event name is in a signaled state.
      Select the event in the associated combo box.
    4. When printing at a certain time is enabled it is possible to specify how many times the printing should be repeated and with what interval. In the For repetition type combo box select the repetition characteristics for either daily or non-daily printing. In the case of non-daily printing specify the interval in minutes after which another printing should occur by checking the Repeat every n minutes and by typing the interval n into the associated edit field.
      For both kinds of printing stop conditions can be specified by selecting either the Stop after n copies or Stop at time YYYY.DDD.mm:hh:ss radio button and by filling in the appropriate data into the edit fields.
    5. Push the > button to activate the print job. The name of the print job is copied from the Defined list box into the Active one in order to indicate the activation.
      Whenever a print job needs to be deactivated but not deleted select it in the Active list box and then push the < button.
      Note:
      Deactivated print jobs are not scheduled.
  3. Push the OK button.

Perform Retrievals

SatView™ provides a convenient way to retrieve telemetry data. Navigating through historical telemetry data is a simple process thanks to the various playback modes and high capacity random access storage devices.

  1. Open the AND Retrievals Dialog Box by selecting the Retrievals... item on the application button's menu when the display is activated. Alternatively, push the Retrievals button on the display's toolbar.
  2. In the Retrieve from Time edit field inside the Historical Data group box specify the time at which the retrieval should start.
    The syntax for the specification of the time is: YYYY.DDD.hh:mm:ss.
    Note:
    Depending on the telemetry data standard used it is not always possible to start the retrieval process exactly at the specified time. This is true for incremental telemetry data streams where fast retrievals can only start at positions containing a summary of all telemetry parameters. Mark the Exact Time check box in order to force the retrieval to start exactly at the specified time. However, this may take some time before the retrieval can take place.
  3. Specify when the retrieval should be stopped by checking one of the following options:

    Option

    Description

    Stop at YYYY.DDD.hh:mm:ss

    Stops the retrieval at the specified time. Type that time into the associated time edit field.

    Stop when no more data is available

    Stops retrieving telemetry data when no more data is available in the retrieval file.

    Switch to realtime

    Automatically switches to real-time when no more telemetry data is available.
  4. Determine the playback mode by checking one of the following possibilities:

    Option

    Description

    Automatic

    Replays the telemetry data with a specified speed.
    Adjust the speed with the Speed slider.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.

    Pseudo Realtime

    The telemetry data is replayed at a speed close to real-time.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.

    Manual

    The telemetry data closest to the specified starting time is monitored and then the retrieval is stopped.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.
  5. Specify an interval in minutes if a sequential retrieval is not required. This option is useful when navigating through the retrieval file and searching for a certain incident.
    Note:
    Depending on the telemetry data standard used it is not always possible to retrieve with the interval specified i.e. the de-facto interval may be smaller or bigger than the one specified.
  6. Push the OK button.
Note:

Push the Realtime Mode check box to terminate any retrievals and to switch back to real-time.

    Viewing Parameter Details

    ANDs provide the capability to display additional details about the telemetry parameters they contain.

    1. Open the AND Parameters Dialog Box by double-clicking on the parameter inside the AND to be viewed or by selecting the Parameters... item on the application button's menu when the display is activated. Alternatively, push the Telemetry Parameters button on the display's toolbar.
    2. Select the parameter those properties should be shown in the Parameter combo box of the dialog box (if necessary).
    3. Choose the appropriate property page in order to view the desired information.
    4. Push the OK button.
    Note:

Find Displays

SatView™ provides a convenient way to search for an AND containing one or more specified telemetry parameters or for a display which has a title containing a certain word or words. The search criteria can contain boolean operators.

  1. Open the AND Find Dialog Box if it is not open yet:
    Push the Find... button on the AND Open Dialog Box.
  2. Select the search key in the Key combo box.
  3. Select a search value expression in the Value combo box or enter one into the same edit field (if no choices are available).
    Note:
    The search values may also contain the boolean operators '+' (AND) and '|' (OR). Whenever one or more of these operators are specified the operands need to be set into double quotes. There is no operator precedence; the search value expression is just evaluated from the left to the right.
    Examples:

    Key

    Search Value

    Action

    Parameter

    "V191" + "V193"

    Searches for all ANDs that contain both telemetry parameters.

    "V191" | "V193" | "V195"

    Searches for all ANDs that contain at least one of the specified telemetry parameters.

    Description

    "AOCS" + "DYNAMICS"

    Searches for all ANDs that contain the words 'AOCS' and 'DYNAMICS' in their title.
  4. Push the Search button.
  5. Select the AND to be opened in the Results list box and push the Open button or double-click on the display to be opened.
    Note:
    Push the Close button in order to discontinue the search for an AND.

Link Displays

In some cases it might be of interest to retrieve the same historical telemetry data in multiple display windows simultaneously. This can be achieved by linking them together.

  1. Open the AND Linkages Dialog Box if it is not open yet:
    Activate the AND to be linked with other display windows and select the Linkages... item on the application button's menu when the display is activated. Alternatively, push the Linkages button on the display's toolbar.
  2. Select the displays to be linked to the current AND in the Displays list box.
  3. Push the > button to add the selected one to the list of already linked displays.
    Use the < button to remove a single linked display selected in the Links list box or << to remove all of them at once.
    Note:
    The unlinked display windows will be changed to stopped historical mode or will remain in real-time mode.
  4. Push the OK button.
Note:
  • If the current AND is processing real-time telemetry data, any linked display window will also be opened in that mode. When operating in historical mode any linked one will be opened with telemetry data being processed at the same (historical) time.
  • The retrieval navigation keys are always disabled on a linked display window.
  • More than one display can be linked to an alphanumeric display.
    In order to have several displays monitoring the same historical telemetry data perform the following steps:
    1. Start a retrieval request on the alphanumeric display (see 'Retrieving with an AND').
    2. Link one or several display windows to it as described above.
    3. Use the retrieval navigation keys on the initial alphanumeric display to stop or continue the monitoring.
  • Initiating a retrieval request on a display window linked to this one will automatically unlink the other one.

Import ANDs

SatView™ supports the interactive definition of temporary ANDs. These displays are not part of the standard database and hence can be exported separately. Through the implemented import/export interfaces, it is possible to synchronize the temporary display databases on multiple workstations in an easy way.

  1. Open the AND Import Dialog Box if it is not open yet:
    Open the Import button's menu inside the Telemetry Displays panel of the Displays ribbon bar category and select the AND... item.
  2. Enter the name of the file that contains the ANDs to be imported into the Filename edit field.
    Use the Browse... button in order to search for the requested file if necessary.
  3. In the Directory combo box select a temporary AND that should be imported.
  4. Push the Add button. Repeat the previous steps if more than one display should be imported or use the Add All button in order to import all available ANDs.
    Use the Remove or Remove All button in order to delete one or several ANDs from the importation list.
  5. Push the OK button.

Prerequisites:
The privilege 'Importing temporary alphanumeric displays' must be enabled in order to import temporary ANDs.

Export ANDs

SatView™ supports the interactive definition of temporary ANDs. These displays are not part of the standard database and hence can be exported separately. Through the implemented import/export interfaces, it is possible to synchronize the temporary display databases on multiple workstations in an easy way.

  1. Open the AND Export Dialog Box if it is not open yet:
    Open the Export button's menu inside the Telemetry Displays panel of the Displays ribbon bar category and select the AND... item.
  2. In the Directory combo box select a temporary AND that should be exported.
  3. Push the Add button. Repeat the previous steps if more than one display should be exported or use the Add All button in order to export all available ANDs.
    Use the Remove or Remove All button in order to delete one or several ANDs from the exportation list.
  4. Enter the name of the file that should contain all exported ANDs into the Filename edit field.
    Use the Browse... button in order to search for an appropriate destination file if necessary.
  5. Push the OK button.

Prerequisites:
The privilege 'Exporting temporary alphanumeric displays' must be enabled in order to export temporary ANDs.

Prerequisites:
The privilege 'Using alphanumeric display windows' must be enabled in order to use any ANDs.
