Parameter Observation Display (POD)

SUMMARY

Saturday, October 10, 2015
Version 1.0

A Parameter Observation Display (POD) presents parameter values or derived expression results in a scrollable, alphanumeric manner. Expressions can be specified 'on-the-fly' and may contain mathematical functions as well as parameter values. In addition, it is possible to define a list of parameters to be added to the display for comparison reasons and a predefined set of statistical functions helps to analyze the resulting parameter values or expression results in more detail.

How do I...

Open a POD

  1. Push the POD button inside the Telemetry Displays panel of the Displays ribbon bar category.
    This opens the POD Open Dialog Box.
  2. Choose between the available display types by checking the appropriate radio button in the dialog box:

    Type

    Description

    Normal

    Lists all PODs defined in the database.
    Note:
    The configuration of this type of POD can be changed interactively. The modifications, however, are temporary only.

    Temporary

    Lists all PODs created interactively.
    This type of display can be created by typing a unique name into the Display edit field and by changing the telemetry parameters or the on-the-fly expressions calculated.
    An existing POD of this type can always be deleted from the database by pushing the Remove button.
  3. Choose the data mode by either checking the Realtime or History radio button.
    Follow the instructions about how to retrieve with a POD when requesting historical telemetry data.
  4. Optionally configure the POD.
  5. Push the OK button.

Configure the Settings

Many aspects of a POD are configurable. The layout, expression evaluation, filtering options, retrieval source and print jobs can be modified as required. Furthermore, it is also possible to define the way how telemetry data should be displayed with respect to validity and out-of-limit status colors. Tool-tips, providing details about parameters and statistical functions, can be enabled or disabled as well.

  1. Open the POD Settings Dialog Box if it is not open yet:
    Activate the POD and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.

  2. Read more about:

    Modifying the Layout

    Specifying the Appearance:

    1. Select the Layout page.

    2. Choosing the Columns

      • Mark the Time, Value, Status and/or Quality check box in order to select the information to be listed.
        Note:
        At least one of the columns must be selected.

      Adding comparable Parameters

      Comparable parameters are useful to check the listed parameter values or expression results against them.

      1. In the Parameters combo box select a parameter to be added for comparison purposes.
      2. Push the Add button. The selected parameter appears in the associated list box.
        Repeat the previous steps in order to add more than one parameter.
      3. Select the color for the comparable parameters in the associated combo box.

      Removing comparable Parameters

      1. In the Parameters list box select the parameter to be removed.
      2. Push the Remove button. The selected parameter disappears from the list box.
        Repeat the previous steps in order to remove more than one parameter.

      Adding statistical Functions

      Statistical functions are useful to perform additional analysis on the listed parameter values or expression results.

      1. In the Functions combo box (the left one) select a statistical function to be added.
      2. Push the > button. The selected function appears in the combo box on the right side.
        Repeat the previous steps in order to add more than one function.
      3. Select the color for the statistical functions in the associated combo box.

      Removing statistical Functions

      1. In the Functions combo box (the right one) select the statistical function to be removed.
      2. Push the < button. The selected function disappears from the right combo box. Push the << button in order to remove all of them at once.
        Repeat the previous steps in order to remove more than one function.

      Changing the Fonts

      1. Choose where the fonts should be changed by selecting the area in the For the area use... combo box.
      2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.

    Controlling the Operating Flags:

    1. Select the Layout page.

    2. Tool-tips

      A small tool-tip containing parameter and statistical function related details can be displayed on request. When this option is enabled and the cursor is moved over a parameter or expression result inside the POD, a small hand occurs indicating that a tool-tip is available. Press the right mouse button to make it pop-up. The tool-tip disappears automatically when the mouse button is released.

      • Mark the Tooltips check box in order to enable this feature.

    Changing the Contents

    1. Select the Contents page.

    2. Monitoring a Parameter

      1. Select the parameter to be monitored in the Parameter combo box.
      2. Check either the Default or Coded radio button depending on whether the parameter values should be displayed in a calibrated (if possible) or coded (raw) way.
      3. Choose the occurrence of the parameter to be monitored by entering that number into the Sample edit field.
        Note:
        A negative occurrence number can be entered in order to monitor a past value of a parameter. A sample number of -1, for example, specifies the value preceeding the current one and -2 even one before.
      4. Push the Add button.

      Note:
      It is allowed to monitor multiple parameters; just add the desired parameters to the stack as described above, without any operators or functions. In such a case the statistical functions apply to all values of all parameters.

      Monitoring an Expression

      On-the-fly expressions are built from parameters, operators, functions or numbers and are defined in the Reverse Polish Notation.

      1. Add a parameter, operator, function or number to the stack by pushing the appropriate buttons and by considering the following points:
        • A small red horizontal bar indicates the current insertion point on the stack.
        • Use the é or ê button to move that point up or down the stack.
        • Use the ç or è button to add an element (i.e. parameter, operator, function or number) onto the stack or to remove it.
          Note:
          Operators and functions may be disabled in some cases because there are not sufficient operands available. Binary operators need at least two operands on the stack, unary ones and most of the functions just one.
        • Past parameters values (negative sample numbers) can also be used inside an expression. The expression, however, is evaluated only when all values (including the past ones) are available.
      2. Repeat the previous step as many times as stack elements are needed in order to compose the complete expression.
        Use the Remove button to delete a selected stack element. The Remove All button clears the complete stack.

      Note:
      If an expression cannot be evaluated due to a missing parameter value or illegal mathematical operation (e.g. division by zero) this is generally indicated by a text message followed by a number (index of the stack element causing the problem) instead of the expression value. In case of a missing parameter value the identification tag of that parameter is shown instead of the index.

      Specifying the default Color

      • Choose the color with which the expression results should be displayed in the Color combo box.
        Note:
        This option is effective only when an expression consists of more than just a single parameter. If just one parameter is observed the color of the parameter value is determined by the specified status colors (see 'Change the Filtering Options').

    Changing the Filtering Options

    1. Select the Filtering page.

    2. Filtering by Time

      • Mark the Between t1 and t2 check box and fill-in the time range t1 < t2 into the appropriate edit fields.
        Note:
        Expression evaluation will take place between t1 and t2 only.

      Filtering by a daily Period

      • Mark the Daily between t1 and t2 check box and fill-in the daily time range into the appropriate edit fields.
        Note:
        Expression evaluation will take place daily between t1 and t2 only.

      Filtering by a periodical Interval

      1. Mark the Periodically every n minutes check box and fill the desired period of n minutes into the appropriate edit field.
      2. Optionally mark the Starting at time t check box and enter the time t into the edit field if the periodical filtering should start at that specific time. The same applies for the Stopping at time t check box which specifies the (optional) stop time for the expression evaluation.

      Filtering by Parameter Status & Validity

      • Mark the Status check box and select the status or validity that determines if a parameter value is listed or not.
        Note:
        This option applies to parameter values only but not to expression results. Any parameter value will be listed if this option is not used.

      Filtering by Parameter Data Quality

      • Mark the Quality check box and determine if only good or bad data should be listed by selecting the appropriate check boxes.
        Note:
        This option applies to parameter values only but not to expression results. Any parameter value regardless of its data quality will be listed if this option is not used.

      Limiting the Parameter Values or Expression Results Count

      • Mark the A maximum of n values check box if a maximum of n parameter values or expression results should be listed and enter the limit n into the corresponding edit field.
        Note:
        The number of listed parameter values or expression results will be adjusted (if necessary) the next time a value or result becomes available.

      Parameter/Expression Value Characteristics:

      Specifying the Number of Digits

      • Mark the Digits check box if parameter values or expression results should be limited to an amount of n digits and enter the limit n into the appropriate edit field.
        Note:
        This option applies to floating point numbers only but not to text. It gets effective for any subsequent parameter values or expression results.

      Specifying the Status Colors

      1. Select the status for which the color should be changed in the For values with status... combo box.
      2. Choose the desired color in the ...use color combo box.
        Note:
        This color association is used only when a single parameter (no expression) is observed.

      Specifying the Triggering Mode

      • Mark the Show check box if a parameter value or expression result should be listed depending on an update or value change and check either the Updates or Changes radio button.
        Note:
        Any parameter value or expression result (even identical ones) will be listed if this option is not used.

    Specifying the Retrieval Source

    1. Select the Retrieving page.
    2. Select the retrieval file to become the source for any retrieval requests by highlighting it in the Data Source combo box.
      Push the Inspect button to check a potential retrieval file. The details about the inspected retrieval file are shown in the Source Information group box.
      Note:
      The set of retrieval files listed in the Data Source combo box can be specified in the Retrievals page of the System Settings Dialog Box.

    Using Print Job Profiles

    1. Select the Printing page.

    2. Creating a new Print Job Profile

      1. Type the name of the print job profile to be created into the Name combo box.
        Note:
        The Default print job profile is used when printing with the quick printing method and can only be modified but not removed.
      2. In the Properties group box specify the print profile to be used with this print job profile by selecting the corresponding profile in the Print Profile combo box. The print profile specifies the printer and paper related properties whereas the print job profile adds some, more POD specific, printing directives.
      3. Specify if all values or only the first/last n values should be printed by checking either the Print all values or Print the first/last n values radio button and by filling the number n in the edit field.
      4. Mark the Underline OOLs check box if an out-of-limit value should be underlined.
        Note:
        This option is useful when printing without colors.
      5. Mark the Print OOL Legend check box if a small legend should be included on the print-out indicating the out-of-limit status color association.
        Note:
        This option is useful when printing in colors and does not require to underline out-of-limit parameter values any more.
      6. Mark the Parameters or Functions check box in order to include the comparing parameters and/or statistical functions in the print-out.
      7. Type a comment into the Text edit field if additional information should be added to the title of the print-out. Mark the Clear after use check box if this text should be used in a single print-out only.
      8. Push the Add button.

      Modifying a Print Job Profile

      • Select the name of the print job profile to be modified in the Name combo box and perform the necessary changes (see 'Creating a new Print Job Profile').

      Deleting a Print Job Profile

      1. Select the name of the print job profile to be deleted in the Name combo box.
      2. Push the Remove button to remove the selected print job profile. Use the Remove All button to delete all existing profiles.
        Note:
        The Default print job profile cannot be deleted.

    Scheduling Print Jobs

    1. Select the Jobs page.
    2. Select the print job to be scheduled in the Defined list box.
    3. Specify when the print job should be executed by marking one or more of the available options:

      Option

      Description

      Print at YYYY.DDD.hh:mm:ss

      Prints the POD at a specified time.
      Type the time into the associated time edit field.

      Print at hh:mm:ss every day

      Prints the POD every day at a specified time.
      Type the time into the associated time edit field.

      Print at name event

      Prints the POD when the event name is in a signaled state.
      Select the event in the associated combo box.
    4. When printing at a certain time is enabled it is possible to specify how many times the printing should be repeated and with what interval. In the For repetition type combo box select the repetition characteristics for either daily or non-daily printing. In the case of non-daily printing specify the interval in minutes after which another printing should occur by checking the Repeat every n minutes and by typing the interval n into the associated edit field.
      For both kinds of printing stop conditions can be specified by selecting either the Stop after n copies or Stop at time YYYY.DDD.mm:hh:ss radio button and by filling in the appropriate data into the edit fields.
    5. Push the > button to activate the print job. The name of the print job is copied from the Defined list box into the Active one in order to indicate the activation.
      Whenever a print job needs to be deactivated but not deleted select it in the Active list box and then push the < button.
      Note:
      Deactivated print jobs are not scheduled.
  3. Push the OK button.

Perform Retrievals

SatView provides a convenient way to retrieve telemetry data. Navigating through historical telemetry data is a simple process thanks to the various playback modes and high capacity random access storage devices.

  1. Open the POD Retrievals Dialog Box by selecting the Retrievals... item on the application button's menu when the display is activated. Alternatively, push the Retrievals button on the display's toolbar.
  2. In the Retrieve from Time edit field inside the Historical Data group box specify the time at which the retrieval should start.
    The syntax for the specification of the time is: YYYY.DDD.hh:mm:ss.
    Note:
    Depending on the telemetry data standard used it is not always possible to start the retrieval process exactly at the specified time. This is true for incremental telemetry data streams where fast retrievals can only start at positions containing a summary of all telemetry parameters. Mark the Exact Time check box in order to force the retrieval to start exactly at the specified time. However, this may take some time before the retrieval can take place.
  3. Specify when the retrieval should be stopped by checking one of the following options:

    Option

    Description

    Stop at YYYY.DDD.hh:mm:ss

    Stops the retrieval at the specified time. Type that time into the associated time edit field.

    Stop when no more data is available

    Stops retrieving telemetry data when no more data is available in the retrieval file.

    Switch to realtime

    Automatically switches to real-time when no more telemetry data is available.
  4. Determine the playback mode by checking one of the following possibilities:

    Option

    Description

    Automatic

    Replays the telemetry data with a specified speed.
    Adjust the speed with the Speed slider.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.

    Pseudo Realtime

    The telemetry data is replayed at a speed close to real-time.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.

    Manual

    The telemetry data closest to the specified starting time is monitored and then the retrieval is stopped.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.
  5. Specify an interval in minutes if a sequential retrieval is not required. This option is useful when navigating through the retrieval file and searching for a certain incident.
    Note:
    Depending on the telemetry data standard used it is not always possible to retrieve with the interval specified i.e. the de-facto interval may be smaller or bigger than the one specified.
  6. Push the OK button.
Note:

Push the Realtime Mode check box to terminate any retrievals and to switch back to real-time.

    Suspending/Resuming Updates

    • Activate the POD and select the Hold item on the application button's menu. Alternatively, push the Hold button on the display's toolbar.

Find Displays

SatView provides a convenient way to search for a POD containing one or more specified telemetry parameters or for a display which has a title containing a certain word or words. The search criteria can contain boolean operators.

  1. Open the POD Find Dialog Box if it is not open yet:
    Push the Find... button on the POD Open Dialog Box.
  2. Select the search key in the Key combo box.
  3. Select a search value expression in the Value combo box or enter one into the same edit field (if no choices are available).
    Note:
    The search values may also contain the boolean operators '+' (AND) and '|' (OR). Whenever one or more of these operators are specified the operands need to be set into double quotes. There is no operator precedence; the search value expression is just evaluated from the left to the right.
    Examples:

    Key

    Search Value

    Action

    Parameter

    "V191" + "V193"

    Searches for all PODs that contain both telemetry parameters.

    "V191" | "V193" | "V195"

    Searches for all PODs that contain at least one of the specified telemetry parameters.

    Description

    "AOCS" + "DYNAMICS"

    Searches for all PODs that contain the words 'AOCS' and 'DYNAMICS' in their title.
  4. Push the Search button.
  5. Select the POD to be opened in the Results list box and push the Open button or double-click on the display to be opened.
    Note:
    Push the Close button in order to discontinue the search for a POD.

Link Displays

In some cases it might be of interest to retrieve the same historical telemetry data in multiple display windows simultaneously. This can be achieved by linking them together.

  1. Open the POD Linkages Dialog Box if it is not open yet:
    Activate the POD to be linked with other display windows and select the Linkages... item on the application button's menu. Alternatively, push the Linkages button on the display's toolbar.
  2. Select the displays to be linked to the current POD in the Displays list box.
  3. Push the > button to add the selected one to the list of already linked displays.
    Use the < button to remove a single linked display selected in the Links list box or << to remove all of them at once.
    Note:
    The unlinked display windows will be changed to stopped historical mode or will remain in real-time mode.
  4. Push the OK button.
Note:
  • If the current POD is processing real-time telemetry data, any linked display window will also be opened in that mode. When operating in historical mode any linked one will be opened with telemetry data being processed at the same (historical) time.
  • The retrieval navigation keys are always disabled on a linked display window.
  • More than one display can be linked to a parameter observation display.
    In order to have several displays monitoring the same historical telemetry data perform the following steps:
    • Start a retrieval request on the parameter observation display (see 'Retrieving with a POD').
    • Link one or several display windows to it as described above.
    • Use the retrieval navigation keys on the initial parameter observation display to stop or continue the monitoring.
  • Initiating a retrieval request on a display window linked to this one will automatically unlink the other one.

Import PODs

SatView supports the interactive definition of temporary PODs. These displays are not part of the standard database and hence can be exported separately. Through the implemented import/export interfaces, it is possible to synchronize the temporary display databases on multiple workstations in an easy way.

  1. Open the POD Import Dialog Box if it is not open yet:
    Open the Import button's menu inside the Telemetry Displays panel of the Displays ribbon bar category and select the POD... item.
  2. Enter the name of the file that contains the PODs to be imported into the Filename edit field.
    Use the Browse... button in order to search for the requested file if necessary.
  3. In the Directory combo box select a temporary POD that should be imported.
  4. Push the Add button. Repeat the previous steps if more than one display should be imported or use the Add All button in order to import all available PODs.
    Use the Remove or Remove All button in order to delete one or several PODs from the importation list.
  5. Push the OK button.

Prerequisites:
The privilege 'Importing temporary parameter observation displays' must be enabled in order to import temporary PODs.

Export PODs

SatView supports the interactive definition of temporary PODs. These displays are not part of the standard database and hence can be exported separately. Through the implemented import/export interfaces, it is possible to synchronize the temporary display databases on multiple workstations in an easy way.

  1. Open the POD Export Dialog Box if it is not open yet:
    Open the Export button's menu inside the Telemetry Displays panel of the Displays ribbon bar category and select the POD... item.
  2. In the Directory combo box select a temporary POD that should be exported.
  3. Push the Add button. Repeat the previous steps if more than one display should be exported or use the Add All button in order to export all available PODs.
    Use the Remove or Remove All button in order to delete one or several PODs from the exportation list.
  4. Enter the name of the file that should contain all exported PODs into the Filename edit field.
    Use the Browse... button in order to search for an appropriate destination file if necessary.
  5. Push the OK button.

Prerequisites:
The privilege 'Exporting temporary parameter observation displays' must be enabled in order to export temporary PODs.

Prerequisites:
The privilege 'Using parameter observation display windows' must be enabled in order to use any PODs.
