Graphic Display (GRD)

SUMMARY

Saturday, October 10, 2015
Version 1.0

A Graphic Display (GRD) presents parameters in a plotted layout. The various chart types available allow the monitoring of parameter values against time or against other parameters and offer a broad range of options.

How do I...

Open a GRD

  1. Push the GRD button inside the Telemetry Displays panel of the Displays ribbon bar category.
    This opens the GRD Open Dialog Box.
  2. Choose between the available display types by checking the appropriate radio button in the dialog box:

    Type

    Description

    Normal

    Lists all GRDs defined in the database.
    Note:
    The parameter arrangement of this type of GRD can be changed interactively. The modifications, however, are temporary only.

    Temporary

    Lists all GRDs created interactively.
    This type of display can be created by typing a unique name into the Display edit field, by choosing the chart type and by populating it with telemetry parameters.
    An existing GRD of this type can always be deleted from the database by pushing the Remove button.
  3. Choose the data mode by either checking the Realtime or History radio button.
    Follow the instructions about how to retrieve with a GRD when requesting historical telemetry data.
  4. Optionally configure the GRD.
  5. Push the OK button.

Configure the Settings

Many aspects of a GRD are configurable. Layout, chart properties, parameters, retrieval source and print jobs can be modified as required. Furthermore, it is also possible to define the way how parameter values should be displayed with respect to validity and out-of-limit status colors. Tool-tips, providing details about parameters and, in particular, their values can be enabled or disabled as well.

  1. Open the GRD Settings Dialog Box if it is not open yet:
    Activate the GRD and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.

  2. Read more about:

    Modifying the Layout

    Specifying the Appearance:

    1. Select the Layout page.

    2. Creating a new Column

      1. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Name combo box.
      2. Click at the position on the column items bar below the Name combo box where the selected column should be inserted.
        A '*' indicating the selected column appears.
      3. Push the Insert Before or Insert After button.

      Deleting a Column

      1. Select the column to be deleted by clicking on it in the column items bar below the Name combo box.
        A '*' indicating the selected column appears.
      2. Push the Remove button.
        Note:
        The ID column cannot be deleted.

      Changing the Out-of-Limit Colors

      1. Select the out-of-limit status of which the color association should be changed in the For values with status combo box.
      2. Choose the desired color in the ...use color combo box.
      Note:

      Any changes to the out-of-limit status color association apply to the complete graphic display and not to a specific parameter only.

      Changing the Fonts

      1. Inside the Fonts group box select the font that should be changed in the For the area combo box.
      2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.

    Controlling the Operating Flags:

    1. Select the Layout page.

    2. Invalid Data Flag

      This flag controls the visibility behavior of the telemetry parameter values with respect to their validity.

      • Mark the Invalid Data check box if the parameter values should be displayed even when the associated validity is false.

      Bad Data Flag

      Every telemetry parameter has an associated data quality flag referring to its value. By default, a parameter value with a bad quality flag is not updated.

      • Mark the Bad Data check box if the parameter values should be displayed even if their data quality flag is set to bad.

      Tool-tips

      A small tool-tip containing parameter related details can be displayed on request. When this option is enabled and the cursor is moved over a parameter inside the GRD, a small hand occurs indicating that a tool-tip is available. Press the right mouse button to make it pop-up. The tool-tip disappears automatically when the mouse button is released.

      • Mark the Tooltips check box in order to enable this feature.

    Changing the Chart

    Specifying the Characteristics:

    1. Select the Chart page.

    2. Choosing the Chart Type

      • Select one of the chart types by checking the appropriate radio button:

        Type

        Description

        Normal

        Plots the parameter values against time:
        Each parameter contributing to the plot uses the complete vertical extension of the graph.

        Stripchart

        Plots the parameter values against time:
        Each parameter is plotted relative to an assigned horizontal slot. Clipping is performed so that parameter values outside the assigned slots are not visible.

        Stackchart

        Plots the parameter values against time:
        Each parameter is stacked on top of the one beneath it.

        Diagram

        Plots parameters against each other:
        If more than two parameters are contributing to the plot, the value of the first one of each pair defines the x-value, the second one the y-value.

      Changing the Chart Duration

      • In the Duration group box specify the duration of the graph (i.e. the time until the graph is restarted or overwritten depending on the plotting mode) by filling-in the appropriate numbers into the Days, Hours and Minutes edit fields.
        Specify the update interval of the graph by entering the number of minutes after which the graph should be updated into the Update edit field.
        Note:
        When plotting in the Scroll mode it is possible to specify a visible interval (<= duration of the graph) after which the graph should be scrolled. This can be done by marking the Interval check box and by entering the number of percent (with respect to the duration of the graph) the visible part should consist of. An update interval of zero minutes indicates a default refresh period.

      Choosing the Plot Mode

      • Select one of the plotting modes by checking the appropriate radio button:

        Mode

        Description

        Normal

        Plots the parameter values as lines.

        Area

        Fills the space between the plotted lines and the x-axis with the color assigned to the corresponding parameter.
        Note:
        This option is not available for the Diagram chart type.

      Changing the Plot Behavior

      • Select one of the plotting behaviors by checking the appropriate radio button:

        Behavior

        Description

        Scroll

        Scrolls the contents of the graph when the specified visible interval time is expired. The graph is restarted completely when the duration time (>= visible interval) is expired.
        Note:
        This option is not available for the Diagram chart type.

        Overwrite

        Overwrites the graph when the specified duration time is expired. The current location is indicated by a vertical slider.
        Note:
        This option is not available for the Diagram chart type.

        Restart

        Restarts the graph when the specified duration time is expired.

      Changing the Plot Area

      • The following actions can be performed to change the plot area:

        Action

        How to do it

        Modify the plot area size

        Enter the width and height of the area by pushing the Width and/or Height check box inside the Graph group box and enter the appropriate sizes.
        Note:
        A default width and height is used when no size is specified.

        Modify the plot area grids and ticks

        Specify the number of horizontal and vertical grid lines or ticks respectively by marking the Horizontal and/or Vertical check box. Enter the number of desired grid lines or ticks into the corresponding edit fields. The width of each grid line (or tick) can also be specified by using the spin box on half height between the previously mentioned check boxes.
        The color of the grid lines (ticks) can be chosen with the Color combo box.

        Modify the background color of the plot area

        Choose the color for the background of the plot area by selecting the desired color in the Background combo box inside the Color group box.

        Modify the color of the plot area labels

        Choose the color for the plot area labels by selecting it in the Labels combo box inside the Color group box.

    Changing the Contents:

    1. Select the Parameters page.

    2. Adding a Parameter

      1. Select the parameter to be added by choosing it in the Parameters combo box.
      2. Determine the position where the parameter should be inserted by clicking onto the desired parameter inside the Contents list box after which the new one should be inserted.
      3. Push the Add button.
      4. Determine how the parameter value should be displayed by selecting the appropriate item in the Value combo box inside the Properties group box:

        Coding & Radix

        Description

        Default

        Shows the parameter value in a calibrated form when possible otherwise it is presented in a coded form.

        Decimal

        The raw (not calibrated) value is displayed with a decimal radix.

        Binary

        The raw value is displayed with a binary radix.

        Octal

        The raw value is displayed with an octal radix.

        Hexadecimal

        The raw value is displayed with a hexadecimal radix.

        Inverse

        Displays the one's complement of the parameter value with a binary radix.
      5. If a non-default number of decimal places (for floating-point numbers) should be used to display a parameter's value then mark the Digits check box and enter the required number into the associated spin box.
      6. Mark the Occurrence check box if not all occurrences of a parameter value should be plotted and enter the corresponding number into the related edit field.
      7. Enter a minimum and maximum value to be applied to the plotting of the parameter values into the Minimum and Maximum edit fields.
        Note:
        For parameters with a textual calibration curve all allowed values are listed in the Minimum and Maximum combo boxes.
      8. Type an optional comment related to the parameter into the Comments edit field.
      9. Choose a color for the parameter through the Color combo box.
      10. Specify the characteristics of the line with which the plotted parameter values should be drawn by selecting the appropriate line style, symbol and width in the Style, Symbol and Width combo boxes or edit field respectively.

      Modifying a Parameter

      1. Select the parameter to be modified by choosing it in the Contents combo box.
        Perform the necessary changes.
      2. Push the Modify button.

      Deleting a Parameter

      1. Select the parameter to be removed by choosing it in the Contents combo box.
      2. Push the Remove button in order to remove the selected parameter. Using the Remove All button deletes all parameters.

    Specifying the Retrieval Source

    1. Select the Retrieving page.
    2. Select the retrieval file to become the source for any retrieval requests by highlighting it in the Data Source combo box.
      Push the Inspect button to check a potential retrieval file. The details about the inspected retrieval file are shown in the Source Information group box.
      Note:
      The set of retrieval files listed in the Data Source combo box can be specified in the Retrievals page of the System Settings Dialog Box.

    Using Print Job Profiles

    1. Select the Printing page.

    2. Creating a new Print Job Profile

      1. Type the name of the print job profile to be created into the Name combo box.
        Note:
        The Default print job profile is used when printing with the quick printing method and can only be modified but not removed.
      2. In the Properties group box specify the print profile to be used with this print job profile by selecting the corresponding profile in the Print Profile combo box. The print profile specifies the printer and paper related properties whereas the print job profile adds some, more GRD specific, printing directives.
      3. Mark the Print OOL Legend check box if a small legend should be included on the print-out indicating the out-of-limit status color association.
        Note:
        This option is useful when printing in colors.
      4. Mark the Underline OOLs check box if an out-of-limit value should be underlined.
        Note:
        This option is useful when printing without colors.
      5. Mark the No Colors check box if the print-out should be in monochrome despite the printer's color capability.
      6. Determine the scaling of the print-out by checking either the Scale automatically to paper size or Use a fixed scaling factor of n% check box whereby the factor n needs to be filled into the corresponding edit field. Scaling is always done in an isotropic way.
      7. Type a comment into the Text edit field if additional information should be added to the title of the print-out. Mark the Clear after use check box if this text should be used in a single print-out only.
      8. Push the Add button.

      Modifying a Print Job Profile

      • Select the name of the print job profile to be modified in the Name combo box and perform the necessary changes (see 'Creating a new Print Job Profile').

      Deleting a Print Job Profile

      1. Select the name of the print job profile to be deleted in the Name combo box.
      2. Push the Remove button to remove the selected print job profile. Use the Remove All button to delete all existing profiles.
        Note:
        The Default print job profile cannot be deleted.

    Scheduling Print Jobs

    1. Select the Jobs page.
    2. Select the print job to be scheduled in the Defined list box.
    3. Specify when the print job should be executed by marking one or more of the available options:

      Option

      Description

      Print at YYYY.DDD.hh:mm:ss

      Prints the GRD at a specified time.
      Type the time into the associated time edit field.

      Print at hh:mm:ss every day

      Prints the GRD every day at a specified time.
      Type the time into the associated time edit field.

      Print at name event

      Prints the GRD when the event name is in a signaled state.
      Select the event in the associated combo box.

      Print at expiration time

      Prints the GRD as soon as the chart duration time is reached.
      Mark the Print at expiration time check box.
    4. When printing at a certain time is enabled it is possible to specify how many times the printing should be repeated and with what interval. In the For repetition type combo box select the repetition characteristics for either daily or non-daily printing. In the case of non-daily printing specify the interval in minutes after which another printing should occur by checking the Repeat every n minutes and by typing the interval n into the associated edit field.
      For both kinds of printing stop conditions can be specified by selecting either the Stop after n copies or Stop at time YYYY.DDD.mm:hh:ss radio button and by filling in the appropriate data into the edit fields.
    5. Push the > button to activate the print job. The name of the print job is copied from the Defined list box into the Active one in order to indicate the activation.
      Whenever a print job needs to be deactivated but not deleted select it in the Active list box and then push the < button.
      Note:
      Deactivated print jobs are not scheduled.
  3. Push the OK button.

Perform Retrievals

SatView provides a convenient way to retrieve telemetry data. Navigating through historical telemetry data is a simple process thanks to the various playback modes and high capacity random access storage devices.

  1. Open the GRD Retrievals Dialog Box by selecting the Retrievals... item on the application button's menu. Alternatively, push the Retrievals button on the display's toolbar.
  2. In the Retrieve from Time edit field inside the Historical Data group box specify the time at which the retrieval should start.
    The syntax for the specification of the time is: YYYY.DDD.hh:mm:ss.
    Note:
    Depending on the telemetry data standard used it is not always possible to start the retrieval process exactly at the specified time. This is true for incremental telemetry data streams where fast retrievals can only start at positions containing a summary of all telemetry parameters. Mark the Exact Time check box in order to force the retrieval to start exactly at the specified time. However, this may take some time before the retrieval can take place.
  3. Specify when the retrieval should be stopped by checking one of the following options:

    Option

    Description

    Stop at YYYY.DDD.hh:mm:ss

    Stops the retrieval at the specified time. Type that time into the associated time edit field.

    Stop when no more data is available

    Stops retrieving telemetry data when no more data is available in the retrieval file.

    Switch to realtime

    Automatically switches to real-time when no more telemetry data is available.
  4. Determine the playback mode by checking one of the following possibilities:

    Option

    Description

    Automatic

    Replays the telemetry data with a specified speed.
    Adjust the speed with the Speed slider.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.

    Pseudo Realtime

    The telemetry data is replayed at a speed close to real-time.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.

    Manual

    The telemetry data closest to the specified starting time is monitored and then the retrieval is stopped.
    Note:
    The playback mode may be changed interactively during the retrieval. Read below how to navigate through the historical telemetry data.
  5. Specify an interval in minutes if a sequential retrieval is not required. This option is useful when navigating through the retrieval file and searching for a certain incident.
    Note:
    Depending on the telemetry data standard used it is not always possible to retrieve with the interval specified i.e. the de-facto interval may be smaller or bigger than the one specified.
  6. Push the OK button.
Note:

Push the Realtime Mode check box to terminate any retrievals and to switch back to real-time.

Explore Features

  • Read more about:

    Viewing Parameter Details

    GRDs provide the capability to display additional details about the telemetry parameters they contain.

    1. Open the GRD Parameters Dialog Box by double-clicking on the parameter inside the GRD to be viewed or by selecting the Parameters... item on the application button's menu when the display is activated. Alternatively, push the Telemetry Parameters button on the display's toolbar.
    2. Select the parameter those properties should be shown in the Parameter combo box of the dialog box (if necessary).
    3. Choose the appropriate property page in order to view the desired information.
    4. Push the OK button.
    Note:

    Additional parameter details can also be obtained through the use of tool-tips: Right-click on one of the parameters to view additional column related information.

    Scaling Parameters

    1. Open the GRD Parameters Dialog Box by double-clicking on the parameter inside the GRD to be scaled or by selecting the Parameters... item on the application button's menu when the display is activated. Alternatively, push the Telemetry Parameters button on the display's toolbar.
    2. Select the parameter those properties should be shown in the Parameter combo box of the dialog box (if necessary).
    3. Choose the Scaling property page.
    4. Enter the required minimum and maximum values into the corresponding edit fields located to the right of the current range.
      Note:
      • Push the Refresh button in order to view the current minimum and maximum values of the selected parameter.
      • Push the Reset button in order to restore to the previous range or the Default button to reset the range to the database defaults.
      • Mark the Upper and/or Lower check box and enter a percentage into the associated spin box in order include a margin to the top or bottom of the plotting area.
    5. If a scaling should occur automatically, mark the Autoscaling check box and enter the number of minutes into the associated spin box after which a scaling should occur.
    6. Push the OK button.

    Zooming Parameters

    1. Push the left mouse button and the 'Shift' key at the starting position (on the graph) to be zoomed-in. A ruler will occur.
    2. Move the mouse to the right by keeping the left button and 'Shift' key pressed. A small tool-tip above the ruler will indicate the time range to be activated.
      Note:
      Push the 'Ctrl' key in order to activate the ruler.
    3. Release the mouse button.
    Note:

    The graphic display will be set to hold mode after zooming-in whenever the time range is completely within historical telemetry data. Pushing the Hold button again will restore to normal update mode.

Find Displays

SatView provides a convenient way to search for a GRD containing one or more specified telemetry parameters or for a display which has a title containing a certain word or words. The search criteria can contain boolean operators.

  1. Open the GRD Find Dialog Box if it is not open yet:
    Push the Find... button on the GRD Open Dialog Box.
  2. Select the search key in the Key combo box.
  3. Select a search value expression in the Value combo box or enter one into the same edit field (if no choices are available).
    Note:
    The search values may also contain the boolean operators '+' (AND) and '|' (OR). Whenever one or more of these operators are specified the operands need to be set into double quotes. There is no operator precedence; the search value expression is just evaluated from the left to the right.
    Examples:

    Key

    Search Value

    Action

    Parameter

    "V191" + "V193"

    Searches for all GRDs that contain both telemetry parameters.

    "V191" | "V193" | "V195"

    Searches for all GRDs that contain at least one of the specified telemetry parameters.

    Description

    "AOCS" + "DYNAMICS"

    Searches for all GRDs that contain the words 'AOCS' and 'DYNAMICS' in their title.
  4. Push the Search button.
  5. Select the GRD to be opened in the Results list box and push the Open button or double-click on the display to be opened.
    Note:
    Push the Close button in order to discontinue the search for a GRD.

Link Displays

In some cases it might be of interest to retrieve the same historical telemetry data in multiple display windows simultaneously. This can be achieved by linking them together.

  1. Open the GRD Linkages Dialog Box if it is not open yet:
    Activate the GRD to be linked with other display windows and select Linkages... item on the application button's menu. Alternatively, push the Linkages button on the display's toolbar.
  2. Select the displays to be linked to the current GRD in the Displays list box.
  3. Push the > button to add the selected one to the list of already linked displays.
    Use the < button to remove a single linked display selected in the Links list box or << to remove all of them at once.
    Note:
    The unlinked display windows will be changed to stopped historical mode or will remain in real-time mode.
  4. Push the OK button.
Note:
  • If the current GRD is processing real-time telemetry data, any linked display window will also be opened in that mode. When operating in historical mode any linked one will be opened with telemetry data being processed at the same (historical) time.
  • The retrieval navigation keys are always disabled on a linked display window.
  • More than one display can be linked to a graphic display.
    In order to have several displays monitoring the same historical telemetry data perform the following steps:
    • Start a retrieval request on the graphic display (see 'Retrieving with a GRD').
    • Link one or several display windows to it as described above.
    • Use the retrieval navigation keys on the initial graphic display to stop or continue the monitoring.
  • Initiating a retrieval request on a display window linked to this one will automatically unlink the other one.

  2. Choose one of the following printing methods:

    Method

    Description

    How to do it

    Quick Printing

    This method is the most simple one because it does not need any further interaction of the user in order to print a GRD. The way how the display is printed is defined by the Default print job profile.

    Push the Print button on the display's toolbar.

    Whenever a GRD needs to be printed in a different way than defined by the Default print job profile this method is the one to be used. It allows the modification of the print properties without changing the profile.

    • Change the settings in the Print Dialog Box as required.
    • Push the OK button.

    Scheduled Printing
    Event-driven Printing

    Sometimes it may be advantageous to print a GRD automatically at a certain time or when a predefined telemetry event occurs. This can be achieved by scheduling a print job.
    • Select the Settings... item on the application button's menu when the display is activated or push the Settings button on the display's toolbar.
    • Select the Print page in the GRD Settings Dialog Box.
    • Add a print profile as required.
    • Push the OK button.

Import GRDs

SatView supports the interactive definition of temporary GRDs. These displays are not part of the standard database and hence can be exported separately. Through the implemented import/export interfaces, it is possible to synchronize the temporary display databases on multiple workstations in an easy way.

  1. Open the GRD Import Dialog Box if it is not open yet:
    Open the Import button's menu inside the Telemetry Displays panel of the Displays ribbon bar category and select the GRD... item.
  2. Enter the name of the file that contains the GRDs to be imported into the Filename edit field.
    Use the Browse... button in order to search for the requested file if necessary.
  3. In the Directory combo box select a temporary GRD that should be imported.
  4. Push the Add button. Repeat the previous steps if more than one display should be imported or use the Add All button in order to import all available GRDs.
    Use the Remove or Remove All button in order to delete one or several GRDs from the importation list.
  5. Push the OK button.

Prerequisites:
The privilege 'Importing temporary graphic displays' must be enabled in order to import temporary GRDs.

Export GRDs

SatView supports the interactive definition of temporary GRDs. These displays are not part of the standard database and hence can be exported separately. Through the implemented import/export interfaces, it is possible to synchronize the temporary display databases on multiple workstations in an easy way.

  1. Open the GRD Export Dialog Box if it is not open yet:
    Open the Export button's menu inside the Telemetry Displays panel of the Displays ribbon bar category and select the GRD... item.
  2. In the Directory combo box select a temporary GRD that should be exported.
  3. Push the Add button. Repeat the previous steps if more than one display should be exported or use the Add All button in order to export all available GRDs.
    Use the Remove or Remove All button in order to delete one or several GRDs from the exportation list.
  4. Enter the name of the file that should contain all exported GRDs into the Filename edit field.
    Use the Browse... button in order to search for an appropriate destination file if necessary.
  5. Push the OK button.

Prerequisites:
The privilege 'Exporting temporary graphic displays' must be enabled in order to export temporary GRDs.

Prerequisites:
The privilege 'Using graphic display windows' must be enabled in order to use any GRDs.
