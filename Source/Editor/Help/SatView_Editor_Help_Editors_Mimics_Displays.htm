Mimics Displays

SUMMARY

Saturday, October 10, 2015
Version 1.0

In the context of designing Mimics Displays there is an easy-to-use interface consisting of two panes: The upper one allowing the interactive arrangement of the mimics objects and the lower one hosting the coding area. Special features are also available to test the design before using it in the operational environment.

How do I...

Create a new Mimics Display

  1. Open the New Dialog Box if it is not open yet:
    Select the New... item on the application button's menu.
  2. Select the Mimics Display item in the Type combo box.
  3. Push the OK button.
Note:

The same result can also be achieved by right-clicking onto the Mimics Displays component inside the Database page of the Workspace tool window and selecting the New... item of the floating menu.

Configure the Properties of a Mimics Display

The configurable properties of a mimics display are not limited to its size, margins or background they also consist of elements aiding the design of the display that might not be visible anymore afterwards.

  1. Open the MMD Properties Dialog Box if it is not open yet:
    Push the Properties button inside the Details panel of the View ribbon bar category.
  2. Check either the Default or Fixed radio button depending on the requirement to have an image with a custom size (no automatic adjustment) or not.
    For an image with a custom size type the desired width and height (in pixels) into the corresponding edit fields.
  3. Enter the size (in pixels) of a required margin into the Left, Right, Top and/or Bottom edit field.
    Note:
    An image with an automatic size adjustment cannot have a left or top margin.
  4. Choose the color to be the background of the display in the Background combo box.
  5. Check either the Points or Lines radio button depending on whether points or lines should be added to the background. If this option is enabled enter the horizontal and vertical interval (in pixels) between them into the Horizontal Interval or Vertical Interval edit field and choose their color in the Color combo box.
    Note:
    This option is available primarily for design purposes. By marking the Show always button the grid can be made visible even after that time.
  6. Push the OK button.

Design & Edit a Mimics Display

Opening a Mimics Display

  • Create or open the mimics display to be edited:
    Select the New... item on the application button's menu, choose the Mimics Display item in the Type combo box and design the mimics display as required.
    Select the Open è Mimics Display... item on the application button's menu and choose the display to be opened in the MMD Open Dialog Box.

Saving a Mimics Display

  • Select the Save item on the application button's menu to save an existing one.
    Use the Save As... item to save a new one or an existing one with a different name.

Working with Mimics Objects

Mimics Objects:
Mimics objects are the basic components of an image. They can be moved, sized and rotated interactively. In addition to that many other visual aspects of them can be changed. The complete functionality is also available programmatically through the Mimics Description Language (MDL) library. Each mimics object can have an associated tool tip providing more detailed information about it. Common actions are:

Inserting...

Shows how to add a new mimics object to the image.

  1. Open the Insert button's menu by pushing that button inside the Editing panel of the Mimics è Objects ribbon bar category and select the requested mimics object type to be inserted.
  2. Move the cursor to the location on the image where the object should be placed at and push the left mouse button.
Note:

The selected mimics object will occur in a default configuration at the desired location.

Selecting...

Shows how to select multiple mimics objects at once.

  1. Move the cursor to the left and upper most position covering all mimics objects to be selected.
  2. Hold down the Shift key and press the left mouse button at the same time.
  3. Drag and move the cursor until all mimics objects to be selected are within (or at least partly covered) by the dragging rectangle. Release the Shift key and mouse button.
Note:

The selected collection of mimics objects can now be treated like a single one i.e. most of the actions now apply to all selected objects.

Moving...

Shows how to move a mimics object around.

  1. Activate the mimics object to be moved by clicking onto it. Object frame thumbs appear to indicate the activation.
  2. Move to the desired location while keeping down the left mouse button and release the button or, as an alternative, use the arrow keys to move the object.

Sizing...

Shows how to change a mimics object in size.

  1. Activate the mimics object to be sized by clicking onto it. Object frame thumbs appear to indicate the activation.
  2. Move the cursor on one of the frame thumbs. It will change its appearance in order to indicate the direction of the size operation.
  3. Push the left mouse button and size the object accordingly. Release the button when the sizing is complete.

Aligning...

Shows how to align multiple mimics objects either horizontally or vertically.

  1. Activate the mimics object to be used as a reference for the alignment by clicking onto it. Object frame thumbs appear to indicate the activation.
  2. Hold down the Shift key and activate all other objects to be aligned with the first activated one. A diagonal grid covering the objects indicates the multiple activation.
  3. Push the Align Horizontally or Align Vertically button inside the Editing panel of the Mimics è Objects ribbon bar category depending on whether the objects should be aligned horizontally or vertically.
Note:

The alignment of mimics objects is always performed relative to their center points.

Flipping...

Shows how to flip a mimics object horizontally or vertically with respect to its center point.

  1. Activate the mimics object to be flipped by clicking onto it. Object frame thumbs appear to indicate the activation.
  2. Push the Flip Horizontally button inside the Editing panel of the Mimics è Objects ribbon bar category to mirror it versus a vertical axis.
    Use the Flip Vertically button to mirror it versus a horizontal axis.
Note:

Flipping mimics objects that are symmetrical to the flipping axis will not change.

Turning & Rotating...

Shows how to rotate a mimics object around its center point.

  1. Activate the mimics object to be rotated by clicking onto it. Object frame thumbs appear to indicate the activation.
  2. Push the Turn Mode button inside the Editing panel of the Mimics è Objects ribbon bar category. The mimics object can now be turned manually around its center. Push the Turn Mode button again to exit this mode.
    In order to rotate by a 'pre-defined' angle, click on the down arrow of the Turn Mode button and select the Rotate... menu item. Then the Mimics Object Rotate Dialog Box will appear. Specify the rotation angle (in degrees) that the object should be turned around its center in a counter-clockwise direction.
  3. Push the OK button.

Changing the Z-Order...

Shows how to change the visibility of a mimics object.

  1. Activate the mimics object those z-order needs to be changed by clicking onto it. Object frame thumbs appear to indicate the activation.
  2. Push the Move To Top button inside the Editing panel of the Mimics è Objects ribbon bar category to move the selected mimics object on top of all other overlapping objects.
    Use the Move To Bottom button to move the mimics object below all other overlapping objects, the Move Up button to move the mimics object before the next overlapping object or the Move Down button to move it behind the next underlying object.

Showing/Hiding...

Shows how to hide and show a mimics object.

  1. Activate the mimics object those visibility needs to be changed by clicking onto it. Object frame thumbs appear to indicate the activation.
  2. Push the Show/Hide button inside the Editing panel of the Mimics è Objects ribbon bar category to toggle the visibility of the selected mimics object.
Note:

Mimics objects should be hidden with care only. This is because hidden objects are hard to find again (a changing cursor is the only indication for an invisible underlying object).

Linking...

Shows how to link a mimics object with another display.

  1. Activate the mimics object to be linked with another display by clicking onto it. Object frame thumbs appear to indicate the activation.
  2. Push the Links button inside the Details panel of the View ribbon bar category. This opens the Mimics Object Link Information Dialog Box.
  3. Choose the display that should be linked to the activated mimics object in the Display list box.
  4. Push the OK button.
Note:

A small red hand will occur whenever the cursor is over a mimics object that has a valid display link. Double-clicking on it will open the linked display with the identical telemetry data being processed.

Copying...

Shows how to copy a mimics object to the clipboard.

  1. Activate the mimics object that needs to be copied to the clipboard by clicking onto it. Object frame thumbs appear to indicate the activation.
  2. Push the Copy button inside the Clipboard panel of the Home ribbon bar category.
Note:

The mimics object will be copied to the clipboard in its current configuration but without name.

Cutting...

Shows how to delete a mimics object by moving it to the clipboard.

  1. Activate the mimics object that needs to be deleted by clicking onto it. Object frame thumbs appear to indicate the activation.
  2. Push the Cut button inside the Clipboard panel of the Home ribbon bar category.

Pasting...

Shows how to paste a mimics object from the clipboard.

  • Push the paste button inside the Clipboard panel of the Home ribbon bar category.
Note:

The mimics object is pasted to the upper left corner of the image.

Deleting...

Shows how to delete a mimics object from an image.

  1. Activate the mimics object that needs to be deleted by clicking onto it. Object frame thumbs appear to indicate the activation.
  2. Push the Delete button inside the Clipboard panel of the Home ribbon bar category.

Mimics Groups:
Mimics groups are a collection of mimics objects composing a more complex one. Once existing as a group they can be handled like normal mimics objects i.e. most actions applicable to mimics objects are available for groups as well.
Hence, the points below describe the operations specific to mimics groups:

Creating...

Shows how to create a group composed of multiple mimics objects or other groups.

  1. Open the Mimics Group New Dialog Box if it is not open yet:
    Open the Groups button's menu located inside the Editing panel of the Mimics è Objects ribbon bar category by pushing that button and select the New... menu item.
  2. Type a unique name for the new mimics group into the Name edit field.
  3. Select one of the mimics objects to get a member of the group by selecting it in the Objects list box inside the Members group box and by pressing the < button. Repeat this step as many times as objects should be added to the group.
    Removing an object that is already part of the group can be achieved by selecting it in the Collection list box and using the > button. All members of a group can be deleted in one step by using the >> button.
    Note:
    The Preview image illustrates the mimics group in its current form.
  4. Push the OK button.
Note:

Only mimics objects (or groups) that are part of the image can get members of a group.

Deleting...

Shows how to remove a group from the library.

  1. Open the Mimics Group Delete Dialog Box if it is not open yet:
    Open the Groups button's menu located inside the Editing panel of the Mimics è Objects ribbon bar category by pushing that button and select the Delete... menu item.
  2. Select the mimics group to be deleted in the Groups list box.
  3. Push the Delete button. Use the Delete All button to remove all available mimics groups from the library.
  4. Push the OK button.

Importing...

Shows how to import mimics groups from other sources.

  1. Open the Mimics Group Import Dialog Box if it is not open yet:
    Open the Groups button's menu located inside the Editing panel of the Mimics è Objects ribbon bar category by pushing that button and select the Import... menu item.
  2. Specify the name of the file containing the mimics group to import by typing it into the Filename edit field. Eventually use the Browse... button in order to search for the requested file.
  3. Type the desired name of the mimics group into the Import as edit field.
    Note:
    Use the Groups list box in order to make sure that a unique mimics group name is entered.
  4. Push the OK button.

Exporting...

Shows how to export mimics groups.

  1. Open the Mimics Group Export Dialog Box if it is not open yet:
    Open the Groups button's menu located inside the Editing panel of the Mimics è Objects ribbon bar category by pushing that button and select the Export... menu item.
  2. Select the mimics group to be exported in the Groups list box.
  3. Choose the format with which the group should be exported in the Format combo box.
    Note:
    Export a mimics group in the standard file format if it should be imported on another workstation.
  4. Type the name of the resulting file into the Filename edit field. If necessary, use the Browse... button in order to search for an existing file.
  5. Push the OK button.

Tips and Tricks:

  • Use the TAB key to activate the mimics object following the currently active one. Holding down the Shift key and then pressing TAB activates the previous one.
  • In order to view the source code associated with a mimics object press the right mouse button on that object and select the View Code item in the appearing floating menu. The focus will then jump to the corresponding code.
  • Finding a mimics object associated with a certain source code can be done by pressing the right mouse button on the name of that mimics object procedure and then selecting the View Object item in the appearing floating menu. The corresponding mimics object will then be activated.
  • A mimics object can be deactivated by activating another one or by clicking on an area of the image that does not contain any objects.

Besides the standard edit shortcut keys the following text manipulations are supported:

    The search always starts with the first or last line available depending on the search direction if the Search all check box is marked.
  4. Choose one or more of the available search options:

    Option

    Description

    Search all

    Searches all source code. Starts the search either from the first or last line (depending on the search direction) but not from the current one.

    Match whole word

    Searches for the specified word which may not be part of another word.

  2. Enter the text to be replaced into the Find Text edit field.
    Note:

  3. Type the replacement text x into the Replace with x edit field.
    Note:

    Previous replacement text can be found in the associated combo box.

  4. Determine the direction the text replacement should take place by checking either the Up or Down radio button.
    Note:
    The search always starts with the first or last line available depending on the search direction if the Search all check box is marked.
  5. Choose one or more of the available search options:

    Option

    Description

    Search all

    Searches all source code. Starts the search either from the first or last line (depending on the search direction) but not from the current one.

    Match case

    Searches for a word or phrase with matching case.

    Match whole word

    Searches for the specified word which may not be part of another word.
  6. Take one of the following actions by pushing the corresponding button:

    Action

    Description

    Jumps to the next (or previous) occurrence of the text and replaces it immediately.

    Replaces all occurrences of the searched text immediately.
  7. Push the Close button to terminate the text replacement.

Using the Tools

Some simple but useful tools exist to ease the design of mimics displays. The floating toolbar, for example, allows a quick selection of mimics objects and the zoom window is a feature increasing the design accuracy.

Mimics Toolbar

The mimics toolbar provides an easy-to-use interface simplifying the selection of one of the predefined mimics objects. It is a more handy solution compared to the selection via the ribbon bar.

  • Opening the Mimics Toolbar

    • Push the Mimics Toolbar button inside the Tools panel of the View ribbon bar category.

    Choosing a Mimics Object from the Toolbar

    1. Push the tool button associated with the mimics object to be selected.
    2. Move the cursor to the point where the mimics object should be inserted and press the left mouse button.
    Note:

    The buttons of the mimics toolbar are only selectable when a mimics display is the active display window.

    Closing the Mimics Toolbar

    • Push the Close button on the toolbar or push again the Mimics Toolbar button inside the Tools panel of the View ribbon bar category.

Zoom Window

The zoom window is a helper tool available to improve the design accuracy of mimics displays. It provides a variable zoom factor and a hot spot (area to be enlarged) that can be configured easily.

  • Opening the Zoom Window

    • Push the Zoom Window button inside the Tools panel of the View ribbon bar category.

    Changing the Zoom Factor

    • Use the scroll bar in order to change the zoom factor: Scrolling downwards increases the zoom factor, upwards results in the opposite.

    Moving the Zoom Hotspot

    • Move the cursor inside the client area of the zoom window. A small hand occurs. Press the left mouse button and move the appearing rectangle onto the area of the image to be enlarged. Release the mouse button as soon as the hotspot is correctly positioned.

    Closing the Zoom Window

    • Push the Close button on the tool window or push again the Zoom Window button inside the Tools panel of the View ribbon bar category.

Coding Guidelines

The source code of a mimics display always consists of one or more procedures, each of them associated to a mimics object. The procedures are executed automatically whenever one of the parameters used inside is updated. It is also possible to control the update behavior programmatically.

The syntax looks as follows:

MIMICS OBJECT 'Ni'

[PARAMETERS Pi...Pj]

[VOLATILE Pk...Pl]

[AUTOTRIGGER | NOAUTOTRIGGER]

BEGIN

MDL Code

END

Notation:

 

Ni

The name of the mimics object procedure.

Pi

The tag of a telemetry parameter.

[x]

x occurs zero times or once.

{x}

x occurs zero or more times.

By using the VOLATILE keyword the subsequent enumerated telemetry parameters are not checked whether or not they are initialized (i.e. do have a value). Each of the specified parameters must be contained in the list declared with the PARAMETERS keyword and may have a random value. By default, the procedure is not executed whenever one of the parameters Pi...Pj has no value.
With the VOLATILE keyword specified, the calculation in the procedure is performed although one or more of the parameters Pk...Pl have no (or a random) value. The keywords AUTOTRIGGER or NOAUTOTRIGGER control the execution of the procedure. The first one checks the parameters used in the procedure in order to decide whether an update should take place or not whereas the second one forces a recalculation at the occurrence of every telemetry packet of format.

Compile & Test a Mimics Display

Mimics displays can be tested before they are used in an operational environment. This is achieved by predefining telemetry conditions to be tested and then executing the associated source code.

  1. Create or open the mimics display to be compiled and tested:
    Select the New... item on the application button's menu, choose the Mimics Display item in the Type combo box of the dialog box and design the display.
    Select the Open è Mimics Display... item on the application button's menu and choose the display to be opened in the MMD Open Dialog Box.
  2. Push the Compile button inside the Compilation panel of the Build ribbon bar category.
    Note:
    All output generated by the compiler is displayed in the Build page of the Output tool window.
  3. Correct any reported errors or warnings by double-clicking onto a message in the Build page of the Output tool window. This highlights the line in the source code where the problem is located.
    If required, some help can eventually be obtained by reading more about:
    • Mimics Description Language (MDL)
    • Compiler Basics

Testing the compiled Code

The mimics display testing facility allows the verification of the compiled code against any possible telemetry condition. It includes the capability to create telemetry parameter value sets which can be used within test scripts.

  1. Open the MMD Test Dialog Box if it is not open yet:
    Push the Test button inside the Debugging panel of the Build ribbon bar category.
  2. Select one of the telemetry parameters used to control the behavior of one or more mimics objects in the Parameter combo box.
  3. Choose the appropriate calibration table for the telemetry parameter (if any) in the Table combo box.
    Note:
    Some of the telemetry parameters may have more than one calibration table associated with it. Select the correct one in order to guarantee an expected mapping from raw to calibrated values.
  4. Enter or select a raw value for the current parameter in the Value combo box.
    Note:
    Telemetry parameters with an associated textual calibration table list all available values in the Value combo box.
  5. Select the required occurrence of the telemetry parameter to be assigned with the previously specified value.
    Note:
    Past values cannot be specified.
  6. Specify the out-of-limit condition in which the telemetry parameter should be by checking either the No OOL, Hard OOL, Soft OOL, Delta OOL or Consistency OOL radio button.
  7. Repeat the same with the data quality by marking the Invalid Data and/or Bad Data check box.
    Note:
    Indicating invalid data implies a false validity. Bad data on the other hand indicates that its value might be corrupted.
  8. Push the Set Value button in order to initialize the telemetry parameter with the specified value, data quality and out-of-limit condition.
    Repeat the previous steps (except for the first one) if more than one telemetry parameter should be associated with a value.
  9. Push the Test button as soon as all involved telemetry parameters are initialized with a value and watch the changes on the image of the mimics display.
    Note:
    Use the Stop button in order to discontinue any blinking or other automatic (periodic) behavior or the Reset button to revert the display to its original state.
  10. Push the OK button.

Creating a new Test Profile

The test profiles contain one or more telemetry parameters initialized with a certain value. They are used to simulate certain telemetry conditions and can also be part of a test script.

  1. Open the MMD Test Dialog Box if it is not open yet:
    Push the Test button inside the Debugging panel of the Build ribbon bar category.
  2. Enter the name of the profile to be created into the Name edit field of the Profiles group box.
  3. Initialize a telemetry parameter with a value, data quality and out-of-limit condition by making the appropriate selections and push the Set Value button.
  4. Push the << button in order to add the current telemetry parameter to the profile. Repeat the last two steps as many times as parameters should be inserted into the profile.
    Use the >> button in order to remove one from the profile.
  5. Push the Add button. This stores the test profile.
  6. Push the OK button.

Using Test Profiles

  1. Open the MMD Test Dialog Box if it is not open yet:
    Push the Test button inside the Debugging panel of the Build ribbon bar category.
  2. Select the profile to be applied in the Name combo box.
  3. Push the Test button. This will first initialize all telemetry parameters that are part of the profile with those values, then execute the code and render the image accordingly.
    Repeat the previous two steps if more than one profile should be applied.
  4. Push the OK button.

Deleting a Test Profile

  1. Open the MMD Test Dialog Box if it is not open yet:
    Push the Test button inside the Debugging panel of the Build ribbon bar category.
  2. Select the profile to be deleted in the Name combo box.
  3. Push the Remove button.
  4. Push the OK button.

Using Test Scripts

Test scripts are way to replay predefined telemetry conditions. They can consist of initialized telemetry parameters or test profiles separated by a definable time interval.

  1. Open the MMD Test Dialog Box if it is not open yet:
    Push the Test button inside the Debugging panel of the Build ribbon bar category.
  2. Select the telemetry parameter to be inserted into the script in the Parameter combo box. Initialize it and push the Set Value button. Use the << button to finally insert it into the script.
    If a test profile should be inserted instead of a telemetry parameter, choose its name in the Name combo box inside the Profiles group box and push the < button.
    Repeat the previous step as many times as telemetry parameters or test profiles should be added to the script.
  3. Adjust the delay to be inserted between the various steps by filling the correct number of milliseconds into the Delay edit field. Use the > button to remove a selected entry from the script.
  4. Push the Run button to start executing the script.
    Note:
    Use the Stop button in order to discontinue the execution of the script. The Reset button is available to revert the image to its original state which might be useful when it is saved afterwards.
  5. Push the OK button.

Import Mimics Displays

Mimics displays can be shared between distributed databases through importing and exporting.

  1. Open the MMDs Import Dialog Box if it is not open yet:
    Select the Import è Mimics Display... item on the application button's menu.
  2. Enter the name of the file that contains the mimics displays to be imported into the Filename edit field.
    Use the Browse... button in order to search for the file.
  3. Select the items to be imported inside the list box and specify how to import them by either checking the Default or Custom radio button. By using the Custom radio button it is possible to import a single display with another name and description then when it was exported.
    Note:
    Importing a mimics display with the same name than an existing one will overwrite it.
  4. Push the OK button.
Note:

The same result can also be achieved by right-clicking onto the Mimics Displays component inside the Database page of the Workspace tool window and selecting the Import... item of the floating menu.

Export Mimics Displays

Mimics displays can be shared between distributed databases through importing and exporting.

  1. Open the MMDs Export Dialog Box if it is not open yet:
    Select the Export è Mimics Display... item on the application button's menu.
  2. Choose the mimics displays to be exported in the Directory list box.
  3. Specify how to export the mimics displays by either checking the Default, Custom or Source Code radio button. By using the Custom radio button it is possible to export a display with a different name and description. The Source Code option writes the source code only to the specified file.
    Note:
    The Custom and Source Code options are available only when a single mimics display is selected for export.
  4. Enter the name of the file to contain the mimics displays to be exported into the Filename edit field.
    Use the Browse... button in order to find an existing file.
  5. Push the OK button.
Note:

The same result can also be achieved by right-clicking onto the Mimics Displays component inside the Database page of the Workspace tool window and selecting the Export... item of the floating menu.
