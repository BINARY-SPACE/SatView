Table Forms

SUMMARY

Saturday, October 10, 2015
Version 1.0

Table Forms are the interface available to create, modify, delete or simply browse database component items hosted by the (relational) DBMS. They present the information within tables that are directly modifiable. In addition, the tables are arranged in a way showing the relationship between them which significantly simplifies the interpretation of the data.

Edit a Database Item

The core DBMS components can be browsed and edited within forms showing all database tables relevant to the selected component. Within each form there is always a top-level table owning zero or more parental or non-parental sub-tables connected in a way showing their relationship. Some of the (top-level) tables can contain sub-tables that require the relevant data to be filled into their sub-tables to make them complete. There is also a difference in manipulating database items in a top-level table compared to the same actions in a parental sub-table: Actions like saving, printing or deleting that need to be applied to a top-level table are performed through the application button's menu whereas the Database è Tables ribbon bar category (or the sub-table floating menu) is available for manipulating the sub-tables.

Database Component Items sometimes have associated information like, for example, telemetry parameters that have calibration tables attached. This information is presented in, so called, parental sub-tables. As mentioned above, these tables are not manipulated through the application button's menu; use the Database è Tables ribbon bar category instead. All actions available for these sub-tables can also be initiated through a floating-point menu by right-clicking onto the title row of those tables.
Some common actions related to such sub-tables are:

Action

How to do it

Create a new Database Component Sub-Item
  1. Open an empty table form hosting the required sub-table(s).
  2. Enter all information into the related sub-table and all its child tables.
  3. Select the Database è Tables ribbon bar category and push the Save button inside the Transactions panel. The same action can also be achieved by using the sub-table floating menu.

Choose a different Database Component Sub-Item
  1. Activate the table field hosting the name of the Database Component Sub-Item by double-clicking onto that field.
  2. Choose the desired Database Component Sub-Item.

All information related to the selected Database Component Sub-Item is then automatically displayed.

Print a Database Component Sub-Item
  1. Open an empty table form hosting the required sub-table(s).
  2. Select the Database Component Sub-Item to be printed in the sub-table.
  3. Select the Database è Tables ribbon bar category and push the Print button. The same action can also be achieved by using the sub-table floating menu.
Note:

All items of the selected database sub-component type can be printed by pushing the Print All button inside the Printing panel of the Database è Tables ribbon bar category or by using the sub-table floating menu.

Save a Database Component Sub-Item
  1. Open an empty table form hosting the required sub-table(s).
  2. Create or modify the desired Database Component Sub-Item in the corresponding sub-table.
  3. Select the Database è Tables ribbon bar category and push the Save button. The same action can also be achieved by using the sub-table floating menu.

Delete a Database Component Sub-Item
  1. Open an empty table form hosting the required sub-table(s).
  2. Select the Database Component Sub-Item to be deleted in the sub-table.
  3. Select the Database è Tables ribbon bar category and push the Delete button. The same action can also be achieved by using the sub-table floating menu.

Import Database Component Sub-Items
  1. Open an empty table form hosting the required sub-table(s).
  2. Right-click onto the title row of the sub-table hosting the type of Database Component Sub-Items to be imported.
  3. Select the Database è Tables ribbon bar category and push the Import button. The same action can also be achieved by using the sub-table floating menu.

Export Database Component Sub-Items
  1. Open an empty table form hosting the required sub-table(s).
  2. Right-click onto the title row of the sub-table hosting the Database Component Sub-Items to be exported.
  3. Select the Database è Tables ribbon bar category and push the Export button. The same action can also be achieved by using the sub-table floating menu.

Consider the following points when working with forms:

  • The outline of the tables can be modified by pushing the Properties button inside the Details panel of the View ribbon bar category.
    This opens the Table Form Properties Dialog Box:

    Changing the Fonts

    1. Check either the Title or Items radio button depending on which font of the tables should be changed.
    2. Choose the desired font face in the Name combo box and then complement the properties by selecting the style, size and color in the corresponding Style, Size and Color combo boxes. A sample of the chosen font is always displayed in the Sample area.
    3. Push the OK button.

    Changing the Background Colors

    1. The background color of all table title bars can be changed in the Title combo box.
    2. The same can be applied for any key or value table columns by selecting the desired color in the Keys combo box.
    3. Push the OK button.
  • Text or values can be entered directly into the corresponding fields of the tables by double-clicking onto them. Depending on the type of the input required for a field, a list box, combo box, edit field or other control will appear.
  • Use the following shortcuts to navigate through the modifiable fields:

    Shortcut

    Description

    TAB

    Jumps to the next table if a row or column is selected or moves from the current active field (owning the focus) to the next one.

    Shift+TAB

    Jumps to the previous table if a row or column is selected or moves from the current active field (owning the focus) to the previous one.

    Arrow Up

    Selects the row above the current one.
    Note:
    This shortcut is only accessible when a row is selected and no field is active.

    Arrow Down

    Selects the row below the current one.
    Note:
    This shortcut is only accessible when a row is selected and no field is active.

    Page Up

    Selects the last row on the previous page and scrolls up there.
    Note:
    This shortcut is only accessible when a row is selected and no field is active.

    Page Down

    Selects the first row on the next page and scrolls down there.
    Note:
    This shortcut is only accessible when a row is selected and no field is active.

    Arrow Right

    Selects the next column on the current row.
    Note:
    This shortcut is only accessible when a column is selected and no field is active.

    Arrow Left

    Selects the previous column on the current row.
    Note:
    This shortcut is only accessible when a column is selected and no field is active.

    End

    Selects the last available row or column depending on whether a row or column is currently selected and scrolls there.
    Note:
    This shortcut is only accessible when a row or column is selected and no field is active.

    Ctrl+C

    Copies the content of the active field into the clipboard.

    Ctrl+V

    Pastes the content of the clipboard into the active field.
  • A sub-table floating menu is available and can be made visible by right-clicking onto the title row of the table to which the menu's items should apply to.
  • The following operations can be applied to tables:

    How to do it

    4. Push the Paste button.
    Note:

    This command is also available through the sub-table floating menu.

    Deleting a row
    1. Select the row to be deleted by left-clicking onto it.
    2. Push the Delete button inside the Rows panel of the Database è Tables ribbon bar category.
    Note:

    This command is also available through the sub-table floating menu.

    Inserting a new column
    1. Select the column before or after which the new one should be inserted by left-clicking onto it.
    2. Push the Insert Before or Insert After button inside the Columns panel of the Database è Tables ribbon bar category depending on whether the new column should be inserted before or after the currently selected one.
    Note:

    This command is also available through the sub-table floating menu.

    Deleting a column
    1. Select the row to be deleted by left-clicking onto it.
    2. Push the Delete button inside the Columns panel of the Database è Tables ribbon bar category.
    Note:

    This command is also available through the sub-table floating menu.

    Discarding all modifications for a table
    1. Push the Discard button inside the Modifications button of the Database è Tables ribbon bar category.

    This command is also available through the sub-table floating menu.

    • Push the Reset button inside the Modifications button of the Database è Tables ribbon bar category.
    Note:

    This command is also available through the sub-table floating menu.

Print a Database Item

  1. Open the database component to which the item to be printed belongs to by selecting the Open è Database Component... item on the application button's menu. A dialog box will appear allowing the selection of the requested database item. The same result can be achieved by expanding the corresponding component tree in the Database page of the Workspace pane and double-clicking onto the desired entry.
  2. Select the Print... item on the application button's menu to print the database item.
    Choose the preferred print options in the appearing Print Dialog Box.
  3. Push the OK button.

Delete a Database Item

  1. Open the database component to which the item to be deleted belongs to by selecting the Open è Database Component... item on the application button's menu. A dialog box will appear allowing the selection of the requested database item. The same result can be achieved by expanding the corresponding component tree in the Database page of the Workspace pane and double-clicking onto the desired entry.
  2. Select the Delete item on the application button's menu to remove the database item.

Import Database Items

The capability to synchronize distributed databases is an important feature and also results in an interface to import or export individual database items.

  1. Open the Database Component Import Dialog Box if it is not open yet:
    Select the Import è Database Component... item on the application button's menu.
  2. Enter the name of the file that contains the database component items to be imported into the Filename edit field.
    Use the Browse... button in order to search for the file.
  3. Select the items to be imported inside the list box and specify how to import them by either checking the Default or Custom radio button. By using the Custom radio button it is possible to import a single database item with another name and description then when it was exported.
    Note:
    Importing a database item with the same name than an existing one will overwrite it.
  4. Push the OK button.

Export Database Items

The capability to synchronize distributed databases is an important feature and also results in an interface to import or export individual database items.

  1. Open the Database Component Export Dialog Box if it is not open yet:
    Select the Export è Database Component... item on the application button's menu.
  2. Choose the database component items to be exported in the Directory list box.
  3. Enter the name of the file to contain the database items to be exported into the Filename edit field.
    Use the Browse... button in order to find an existing file.
  4. Specify how to export the items by either checking the Default or Custom radio button. By using the Custom radio button it is possible to export a database single item with a different name and description.
  5. Push the OK button.

Prerequisites:
The corresponding privilege must be enabled to create, modify or delete a database component item.
