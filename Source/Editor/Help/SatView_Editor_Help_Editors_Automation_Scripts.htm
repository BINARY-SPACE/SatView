Automation Scripts

SUMMARY

Saturday, October 10, 2015
Version 1.0

The development of Automation Scripts happens in an integrated way as well: Editing, compiling and testing an automation script takes place within the same interface. It is divided into two panes: The upper one contains a coding area and the lower one renders the output of the tested script.

How do I...

Create a new Automation Script

  1. Open the New Dialog Box if it is not open yet:
    Select the New... item on the application button's menu.
  2. Select the Automation Script item in the Type combo box.
  3. Push the OK button.
Note:

The same result can also be achieved by right-clicking onto the Automation Scripts component inside the Database page of the Workspace tool window and selecting the New... item of the floating menu.

Configure the Properties of an Automation Script

The properties of an automation script describe the type of application to be automated and also the input file and output file involved. Because some scripts may require access to telemetry parameters it is also possible, for testing purposes, to associate values and other attributes with telemetry parameters.

  1. Open the Script Properties Dialog Box if it is not open yet:
    Push the Properties button inside the Details panel of the View ribbon bar category.
  2. Check either the Excel, Word, Access, Outlook or PowerPoint radio button depending on the application to be automated.
    Mark the Other button if another (not listed) application is automated.
  3. Enter the file name of the input file (if any) into the Input File edit field.
    Note:

    This file name will be assigned to the variable InputFile (see Automation Scripts for more information) and can be used within the automation code.

  4. Repeat the same for an eventual output file (if any) by typing its path name into the Output File edit field.
    Note:

    This file name will be assigned to the variable OutputFile. Furthermore, this file is considered to be the resulting file of the automation process and is displayed in the lower pane of the display window when testing the script.
  5. Push the OK button.

Simulating Telemetry Conditions

When testing an automation script it sometimes might be necessary to simulate certain 'health conditions' which can be reflected by telemetry data. This can be achieved by initializing the affected telemetry parameters in the required way.

  1. Open the Script Test Parameters Dialog Box if it is not open yet:
    Push the Properties button inside the Details panel of the View ribbon bar category and use the Parameters... button on the dialog box.
  2. Select a telemetry parameter to be associated with a value in the Tag combo box.
  3. Enter the desired value into the Value edit field.
    Notes:
    • For all telemetry parameters having a textual calibration table attached the possible values are listed in the combo box part of the Value edit field.
    • Double-clicking onto a telemetry parameter with an attached calibration table in the List list box displays the calibration table through the Script Test Parameter Details Dialog Box turning-up.
    • The calibration curve used can be selected in the Table combo box.
    • A non-calibrated value can be entered by marking the Raw check box.
    • Adjust the number in the Sample spin box accordingly.
    • Mark the Past check box in order to indicate that the current value sample is a historical one.
  4. Specify the out-of-limit status, the validity and the data quality of the supplied value by checking the associated radio buttons.
  5. Push the Add button.
    Repeat the previous steps (except for the first one) as many times as telemetry parameters need to be initialized before the testing takes place.
    Note:

    Use the Set button in order to modify a telemetry parameter value. In addition, the Remove or Remove All buttons can be used to delete a selected telemetry parameter (or all) from the list of initializations.

  6. Push the OK button.

Design & Edit an Automation Script

Opening an Automation Script

  • Create or open the automation script to be edited:
    Select the New... item in the application button's menu, choose the è Automation Script item in the Type combo box of the dialog box and write the required code.
    Select the Open è Automation Script... item on the application button's menu and choose the script to be opened in the Script Open Dialog Box.

Saving an Automation Script

  • Select the Save item on the application button's menu to save an existing one.
    Use the Save As... item to save a new one or an existing one with a different name.

Working with Code

Besides the standard edit shortcut keys the following text manipulations are supported:

Goto Line...

  1. Move the focus to the code pane and push the Go To button inside the Editing panel of the Home ribbon bar category. The Goto Dialog Box appears.
  2. Choose one of the available options:

    Option

    Description

    Number: n

    Jumps to line n and highlights it.

    Jump down: n Lines

    Jumps n lines down (if possible) and highlights that one.

    Jump up: n Lines

    Jumps n lines up (if possible) and highlights that one.
  3. Push the Go button. This performs the required jump.
  4. Push the OK button.

Find Text...

  1. Move the focus to the code pane and push the Find button inside the Editing panel of the Home ribbon bar category. The Find Dialog Box appears.
  2. Enter the searched value into the Find edit field or select it inside the combo box (for predefined or recent values).
    Note:
    Some search keys require different ways to specify a searched value.
  3. Determine the direction the search should take place by checking either the Up or Down radio button.
    Note:
    The search always starts with the first or last line available depending on the search direction if the Search all check box is marked.
  4. Choose one or more of the available search options:

    Option

    Description

    Search all

    Searches all source code. Starts the search either from the first or last line (depending on the search direction) but not from the current one.

    Match case

    Searches for a word or phrase with matching case.

    Match whole word

    Searches for the specified word which may not be part of another word.
  5. Push the Find Next button to start or continue the search.
  6. Push the Close button to terminate the search.

Replace Text...

  1. Move the focus to the code pane and push the Replace button inside the Editing panel of the Home ribbon bar category. The Replace Dialog Box appears.
  2. Enter the text to be replaced into the Find Text edit field.
    Note:

    Use the associated combo box in order to re-use previous searched words.

  3. Type the replacement text x into the Replace with x edit field.
    Note:

    Previous replacement text can be found in the associated combo box.

  4. Determine the direction the text replacement should take place by checking either the Up or Down radio button.
    Note:
    The search always starts with the first or last line available depending on the search direction if the Search all check box is marked.
  5. Choose one or more of the available search options:

    Option

    Description

    Search all

    Searches all source code. Starts the search either from the first or last line (depending on the search direction) but not from the current one.

    Match case

    Searches for a word or phrase with matching case.

    Match whole word

    Searches for the specified word which may not be part of another word.
  6. Take one of the following actions by pushing the corresponding button:

    Action

    Description

    Find Next

    Jumps to the next (or previous) occurrence of the text and highlights it but does not replace it.

    Replace

    Jumps to the next (or previous) occurrence of the text and replaces it immediately.

    Replace All

    Replaces all occurrences of the searched text immediately.
  7. Push the Close button to terminate the text replacement.

Coding Guidelines

The following points must be considered when writing an automation script:

  • Good knowledge about coding practices with JScript .NET®.
  • Always use a try/catch exception handler in order to handle any faults:
    try
    {
        JScript .NET® code
    }
    catch (exception)
    {
        Cleanup code
    }
    Multiple try/catch blocks are allowed but must be nested in a way that only one remains at the outermost level.
  • Add the following code segment at the beginning of the last (outermost) catch block:
    catch (exception)
    {
        // Provide an error reason
        Error.Insert(0,exception.message);
        Cleanup code
    }
    This code returns an error reason and hence simplifies the code debugging.
  • Close all open items (like files, workbooks etc.) in the catch block.
  • It is advisable to prevent any required user interaction while an automation script is executing.
    For automating Microsoft® Excel®, for example, check for an existing copy of the output file first before saving it:
        // Sample for Microsoft® Excel®
        if (File.Exists(OutputFile.ToString())) Book.Save();
        else Book.SaveAs(OutputFile.ToString());
    This prevents any undesired input prompt asking if the file should be overwritten.
Note:

Consult the chapter about Automation Scipts in order to get more in-depth information.

Compile & Test an Automation Script

Automation scripts can be tested before they are used in an operational environment. This is achieved by executing the script and verifying its output in the lower pane.

  1. Create or open the automation script to be compiled and tested:
    Select the New... item on the application button's menu, choose the Automation Script item in the Type combo box of the dialog box and write the required code.
    Select the Open è Automation Script... item on the application button's menu and choose the script to be opened in the Script Open Dialog Box.
  2. Push the Compile button inside the Compilation panel of the Build ribbon bar category.
    Note:
    All output generated by the compiler is displayed in the Build page of the Output tool window.
  3. Correct any reported errors or warnings by double-clicking onto a message in the Build page of the Output tool window. This highlights the line in the source code where the problem is located.
  4. Test the compiled automation script by pushing the Test button inside the Debugging panel of the Build ribbon bar category.
    Note:
    The output (if any) is displayed in the lower pane below the source code.

Print an Automation Script

  1. Open the automation script to be printed: Select the Open è Automation Script... item on the application button's menu and choose the script to be opened in the Script Open Dialog Box.
  2. Choose the desired topic:

    Printing the Source Code

    1. Select the Print Source Code... item on the application button's menu.
      Choose the preferred print options in the appearing Print Dialog Box.
    2. Push the OK button.

    Printing the resulting Output

    1. Select the Print... item on the application button's menu.
      Choose the preferred print options in the appearing Print Dialog Box.
    2. Push the OK button.

Delete an Automation Script

  1. Open the automation script to be deleted: Select the Open è Automation Script... item on the application button's menu and choose the script to be opened in the Script Open Dialog Box.
  2. Select the Delete item on the application button's menu.
Note:

The same result can also be achieved by opening the Automation Scripts component tree inside the Database page of the Workspace tool window, right-clicking onto the script to be deleted and selecting the Delete item of the floating menu.

Import Automation Scripts

Automation scripts can be shared between distributed databases through importing and exporting.

  1. Open the Scripts Import Dialog Box if it is not open yet:
    Select the Import è Automation Script... item on the application button's menu.
  2. Enter the name of the file that contains the automation scripts to be imported into the Filename edit field.
    Use the Browse... button in order to search for the file.
  3. Select the items to be imported inside the list box and specify how to import them by either checking the Default or Custom radio button. By using the Custom radio button it is possible to import a single script with another name and description then when it was exported.
    Note:
    Importing an automation script with the same name than an existing one will overwrite it.
  4. Push the OK button.
Note:

The same result can also be achieved by right-clicking onto the Automation Scripts component inside the Database page of the Workspace tool window and selecting the Import... item of the floating menu.

Export Automation Scripts

Automation scripts can be shared between distributed databases through importing and exporting.

  1. Open the Scripts Export Dialog Box if it is not open yet:
    Select the Export è Automation Script... item on the application button's menu.
  2. Choose the automation scripts to be exported in the Directory list box.
  3. Specify how to export the automation scripts by either checking the Default, Custom or Source Code radio button. By using the Custom radio button it is possible to export a script with a different name and description. The Source Code option writes the source code only to the specified file.
    Note:
    The Custom and Source Code options are available only when a single automation script is selected for export.
  4. Enter the name of the file to contain the automation scripts to be exported into the Filename edit field.
    Use the Browse... button in order to find an existing file.
  5. Push the OK button.
Note:

The same result can also be achieved by right-clicking onto the Automation Scripts component inside the Database page of the Workspace tool window and selecting the Export... item of the floating menu.

Prerequisites:
The privilege 'Creating and modifying automation scripts' must be enabled to create, modify or delete automation scripts.
